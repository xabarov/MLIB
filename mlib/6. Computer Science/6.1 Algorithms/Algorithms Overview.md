# Алгоритмы — обзор тем

**Контекст (EN)**: Classic algorithms to know for ML interviews.

## Ответ

### Сортировка

| Алгоритм    | Среднее    | Худшее | Память | Примечание |
|-------------|------------|--------|--------|------------|
| Quicksort   | O(n log n) | O(n²)  | O(log n) | In-place; pivot важен |
| Merge sort  | O(n log n) | O(n log n) | O(n) | Стабильный |
| Radix sort  | O(nk)      | O(nk)  | O(n+k) | Для целых с k цифрами |
| Heapsort    | O(n log n) | O(n log n) | O(1) | In-place, нестабильный |

### Поиск

- **Binary Search** — O(log n); требует отсортированного массива. Ищет позицию, где `arr[mid] == target`; при промахе сужает [lo, hi]. Используется для оптимизации LIS до O(n log n), медианы двух массивов и пр.
  ```python
  def binary_search(arr, target):
      lo, hi = 0, len(arr) - 1
      while lo <= hi:
          mid = (lo + hi) // 2
          if arr[mid] == target: return mid
          elif arr[mid] < target: lo = mid + 1
          else: hi = mid - 1
      return -1
  ```

### Графы и пути

- **BFS (Breadth-First Search)** — обход графа уровнями; структура: очередь; O(V+E). Находит кратчайший путь по числу рёбер.
- **DFS (Depth-First Search)** — обход вглубь; структура: стек / рекурсия; O(V+E). Используется для топологической сортировки, поиска компонент связности.
- **Dijkstra** — кратчайшие пути от одной вершины; неотрицательные веса; O((V+E) log V) с бинарной кучей.
- **A\*** — эвристический поиск; f(n) = g(n) + h(n); оптимален при admissible heuristic (h ≤ реальное расстояние).

### Деревья

- **Pre-order:** корень → левое → правое.
- **In-order:** левое → корень → правое (для BST даёт отсортированную последовательность).
- **Post-order:** левое → правое → корень.

### Задачи классической комбинаторной оптимизации

- **Stable Marriage Problem (алгоритм Гэйла–Шепли)** — дано n мужчин и n женщин с упорядоченными предпочтениями; найти стабильное паросочетание (нет пары, которой было бы выгоднее поменяться партнёрами). Решение: O(n²) через алгоритм предложений. Применяется в задачах назначения (распределение ординаторов по больницам, студентов по вузам).
- **Traveling Salesman Problem (TSP)** — обойти n городов ровно по одному разу с минимальной суммарной стоимостью. NP-трудная задача. Точные методы: динамическое программирование Хелда–Карпа O(2ⁿ · n²). На практике используют эвристики: greedy nearest-neighbor, 2-opt, алгоритм Christofides (гарантия 3/2 от оптимума).

### Техники проектирования алгоритмов

| Техника | Идея | Пример |
|---------|------|--------|
| **Dynamic programming** | Хранить ответы подзадач; оптимальная подструктура | LCS, LIS, 0-1 Knapsack |
| **Greedy** | Локально оптимальный выбор → глобальный оптимум | Dijkstra, Kruskal MST, Huffman |
| **Backtracking** | Перебор с откатом при нарушении ограничений | Sudoku, N-Queens |
| **Divide & Conquer** | Делить задачу, решать рекурсивно, объединять | Merge sort, Quicksort, FFT |
| **Two pointers** | Два указателя с разных концов / с разной скоростью | Subarray sum, Palindrome |
| **Sliding window** | Окно фиксированного или переменного размера | Max sum subarray, Longest unique substring |
| **Prefix sum** | Накопленная сумма для O(1)-запросов отрезков | Subarray sum = k |

### Работа со строками

- **KMP (Knuth–Morris–Pratt)** — поиск паттерна за O(n+m); использует failure function.
- **Rabin-Karp** — поиск с rolling hash; O(n+m) в среднем.
- **Trie (префиксное дерево)** — хранение и поиск строк с общими префиксами; O(k) на операцию (k — длина строки). Полезно для autocomplete.
- **Regular expressions** — реализуются через NFA/DFA; backtracking при наивной реализации может быть экспоненциальным.
