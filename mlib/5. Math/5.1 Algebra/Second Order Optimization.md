# Оптимизация второго порядка

**Вопрос (EN)**: How can we use second-order derivatives for training? Pros and cons? Why don't we see more second-order optimization in practice?

## Ответ

### Использование вторых производных

**Метод Ньютона:** \(w_{t+1} = w_t - H^{-1} \nabla L\), где \(H\) — гессиан. Учитывает кривизну; сходимость квадратичная вблизи минимума.

**Quasi-Newton (BFGS, L-BFGS):** аппроксимация \(H^{-1}\) без явного вычисления гессиана; обновление по градиентам.

### Плюсы и минусы

**Плюсы:**
- Быстрая сходимость (квадратичная для Ньютона).
- Не нужен подбор learning rate.
- Лучше ведут себя при плохой обусловленности.

**Минусы:**
- Вычисление и инверсия гессиана — \(O(n^3)\) по числу параметров \(n\).
- Хранение \(O(n^2)\).
- Неприменимы напрямую для очень больших моделей (миллионы параметров).

### Почему редко в practice

Глубокие сети имеют миллионы параметров; гессиан \(10^6 \times 10^6\) невычислим и нехраним. Квази-Ньютоновские методы (L-BFGS) используют для задач с умеренным числом параметров. В DL обычно применяют методы первого порядка (SGD, Adam) с адаптивным шагом.
