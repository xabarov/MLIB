# Снижение размерности и PCA

**Вопрос (EN)**: Why dimensionality reduction? PCA on features in different ranges (0-1 vs 10-1000)? SVD vs eigendecomposition? PCA vs SVD? How does t-SNE work?

## Ответ

### Зачем снижать размерность

- Уменьшение шума и overfitting.
- Ускорение обучения и инференса.
- Визуализация (2D/3D).
- Удаление коррелированных признаков.

### Разные диапазоны и PCA

PCA чувствителен к масштабу. Признак в диапазоне 10–1000 доминирует над 0–1. Перед PCA нужна стандартизация (например, StandardScaler).

### Eigendecomposition vs SVD

- **Eigendecomposition** — для квадратных матриц; нужна симметричность для ортонормированного базиса.
- **SVD** — для произвольных матриц \(A = U\Sigma V^T\); более общий инструмент.

Связь: для симметричной \(A\) SVD совпадает с eigendecomposition. Для PCA по ковариационной матрице \(X^TX\) используется либо eigendecomposition \(X^TX\), либо SVD \(X\).

**Единственность eigendecomposition:** если все собственные значения различны — собственные векторы определены с точностью до знака; при кратных \(\lambda\) любой ортонормированный базис соответствующего собственного подпространства подходит.

**Численная устойчивость:** SVD применяется к \(X\) напрямую; eigendecomposition \(X^TX\) возводит числа в квадрат, ухудшая число обусловленности (\(\kappa(X^TX) = \kappa(X)^2\)). На практике SVD предпочтительнее для PCA.

### PCA и SVD

PCA можно вычислить через SVD матрицы центрированных данных: главные компоненты — столбцы \(V\), сингулярные значения задают долю объяснённой дисперсии.

### t-SNE

t-SNE сохраняет локальную структуру: похожие точки сближаются, далёкие — отдаляются. Использует student-t распределение для низкоразмерного пространства, чтобы избежать «сжатия» в одной точке. Подходит для визуализации; не сохраняет глобальную структуру и не подходит для feature extraction.
