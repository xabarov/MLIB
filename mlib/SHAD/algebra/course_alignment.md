# Согласование курса алгебры

Этот файл фиксирует редакторские соглашения для лекций и задач по алгебре. Его цель — держать курс связным: одинаковые обозначения, понятный порядок тем, покрытие программы и осмысленную графику.

## Порядок тем

Порядок папок соответствует программе для поступающих в ШАД:

1. `1_Substitutions` — вводная тема вне основного списка программы, полезна для перестановок и знаков в определителях.
2. `2_Complex numbers` — комплексные числа.
3. `3_Linear_equations` — системы линейных уравнений и метод Гаусса.
4. `4_Rank` — линейная зависимость, ранг, критерий Кронекера--Капелли.
5. `5_Det` — определители.
6. `6_Matrices` — операции с матрицами, обратимость, ранг произведения.
7. `7_Vector_spaces` — пространства, базисы, координаты, подпространства.
8. `8_Linear_maps` — линейные отображения, ядро/образ, подобие.
9. `9_Hamilton_Cayley_Jordan` — Гамильтон--Кэли, нильпотентные операторы, жорданова форма.
10. `10_Bilinear_quadratic_forms` — билинейные и квадратичные формы.
11. `11_Euclidean_spaces` — евклидовы пространства.
12. `12_Eigenvectors_eigenvalues` — собственные значения и диагонализируемость.
13. `13_Complex_spaces` — комплексные пространства, эрмитовы формы, спектральная теорема, SVD.

Важно: тема собственных значений появляется дважды. В лекции 9 она дана как минимальный язык, необходимый для жордановой формы. В лекции 12 она систематизируется отдельно: характеристический многочлен, собственные подпространства, кратности и критерии диагонализируемости.

## Общие обозначения

| Объект | Соглашение |
|---|---|
| Вектор и координаты | `x` — вектор, `X` — столбец координат в выбранном базисе, если нужно различать объект и запись |
| Матрица перехода | `C_{e\leftarrow f}` переводит координаты из базиса `f` в координаты базиса `e` |
| Оператор при замене базиса | `B=C^{-1}AC` |
| Билинейная форма при замене базиса | `A'=C^TAC` |
| Эрмитова форма | В лекции 13 используется соглашение: линейность по первому аргументу, сопряжённая линейность по второму |
| Матрица эрмитовой формы | При этом соглашении `H(x,y)=y^*Ax`, где `A^*=A` |
| Тождественный оператор | В абстрактной записи допустимо `E`, в матричной записи чаще `I`; в пределах одного доказательства не смешивать без пояснения |
| Собственное подпространство | `E_\lambda=\ker(A-\lambda I)` для матрицы, `E_\lambda=\ker(\varphi-\lambda E)` для оператора |
| Прямая сумма | `U\oplus W` только при единственности разложения, для двух подпространств эквивалентно `U\cap W=\{0\}` |

## Проверка QA

QA должны использовать ровно те соглашения, которые приняты в соответствующей лекции.

- `9_Hamilton_Cayley_Jordan/qa.md`: первые задачи используют минимальный язык собственных значений; это явно отмечено во введении.
- `12_Eigenvectors_eigenvalues/qa.md`: лист систематизирует собственные значения после минимального использования в лекции 9; это явно отмечено во введении.
- `3_Linear_equations/qa.md`: задача про условие `\rho(B)<1` поддержана кратким объяснением спектрального радиуса в лекции.
- `8_Linear_maps/qa.md`: направление матрицы перехода `C=C_{e\leftarrow f}` явно указано в задаче про нестандартный базис.
- `11_Euclidean_spaces/qa.md`: задача на проверку скалярного произведения через критерий Сильвестра поддержана напоминанием критерия в лекции.
- `13_Complex_spaces/qa.md`: матрица эрмитовой формы согласована с лекцией: `H(x,y)=y^*Ax`.

Полный проход по QA выполнен: у всех задач есть блоки `Решение` и `Ответ`; найденные расхождения с лекциями закрыты локальными вставками.

## Баланс задач

Количество задач не обязано быть одинаковым во всех темах:

- 10 задач — нормальный базовый лист для стандартной темы;
- 13--15 задач — допустимо для тем с несколькими независимыми техниками или важными ловушками;
- 25 задач — допустимо для вводных/разгонных тем, если они служат банком тренировки.

При добавлении новых задач лучше сохранять структуру: сначала вычисления, затем параметры/ловушки, затем доказательства или задачи на понимание.

## Чеклист перед публикацией

Перед публикацией или крупным коммитом по алгебре выполнить:

1. Проверить ссылки на изображения во всех `lesson.md`.
2. Проверить валидность `assets/manifest.json` и `lecture_llm_images.json`.
3. Проверить, что jobs в JSON не ссылаются на удалённые ассеты.
4. Проверить QA: каждая задача должна иметь блоки `Решение` и `Ответ`.
5. Запустить `git diff --check -- SHAD/algebra`.
6. Поискать устаревшие соглашения:
   - `x^*Ay` в лекции 13 допустимо только как пример другого соглашения;
   - `C^{-1}AC` должно относиться к операторам;
   - `C^TAC` должно относиться к билинейным формам;
   - `C_{e\leftarrow f}` должно переводить координаты из `f` в `e`.
7. Поискать слабые визуальные остатки:
   - ссылки на уже удалённые файлы;
   - картинки с посторонними английскими подписями;
   - иллюстрации, которые украшают простой абзац вместо поддержки сложного места.
8. Просмотреть фактический markdown-рендер:
   - короткие display-formula блоки;
   - таблицы с формулами;
   - длинные `aligned`/`cases`;
   - GIF и PNG в целевом просмотрщике.
9. Убедиться, что каждая картинка поддерживает сложное место, а не украшает простой абзац.

Минимальные команды для технической проверки:

```bash
python3 - <<'PY'
from pathlib import Path
import re
failed = False
for md in sorted(Path('SHAD/algebra').glob('*/lesson.md')):
    text = md.read_text(encoding='utf-8')
    refs = re.findall(r'!\[[^\]]*\]\(([^)]+)\)', text)
    missing = [ref for ref in refs if not (md.parent / ref).exists()]
    print(f'{md.parent.name}: lesson images={len(refs)}, missing={missing}')
    failed = failed or bool(missing)
raise SystemExit(1 if failed else 0)
PY

git diff --check -- SHAD/algebra

python3 - <<'PY'
from pathlib import Path

failed = False
for qa in sorted(Path('SHAD/algebra').glob('*/qa.md')):
    text = qa.read_text(encoding='utf-8')
    tasks = text.count('## Задача')
    solutions = text.count('### Решение')
    answers = text.count('### Ответ')
    print(f'{qa.parent.name}: tasks={tasks}, solutions={solutions}, answers={answers}')
    failed = failed or tasks != solutions or tasks != answers
raise SystemExit(1 if failed else 0)
PY
```
