# SHAD Interactive: план следующего этапа

Этот этап отвечает на вопрос: превращается ли `SHAD/interactive` в полезный
учебный продукт, а не просто в красивый каталог математических игрушек.

Короткий вывод: направление удачное и уже полезное, но польза пока локальная.
Три миссии дают телесное ощущение ядра, определителя и матрицы как образов
базиса. Этого достаточно, чтобы оправдать проект. Но пока не хватает
дидактической связности: пользователь решает уровни, получает ключи, но еще не
всегда понимает, какую мысль он должен унести обратно в лекцию.

## Что уже получается хорошо

- **Есть рабочий игровой язык.** Миссии имеют цель, состояние, проверку,
  прогресс и короткую реакцию Меби.
- **Интерактив действительно добавляет смысл.** Определитель как площадь и
  знак, ядро как направление решений, матрица как пара образов базиса - это
  лучше трогать, чем только читать.
- **Архитектура стала нормальной основой.** Есть runtime, feedback policy,
  model modules, Vitest и Playwright smoke; новые миссии можно добавлять без
  копирования всего компонента.
- **Меби работает.** Он уже не просто картинка: персонаж помогает удерживать
  эмоциональный тон и обратную связь.
- **Mobile стал жизнеспособным.** Drawer вместо каталога первым экраном
  меняет восприятие: пользователь попадает в миссию, а не в меню.

## Чего не хватает

### 1. Учебного вывода после действия

Сейчас приложение хорошо говорит "ты попал". Ему нужно лучше говорить "что это
значит".

Нужен короткий итог уровня:

- какую идею мы увидели;
- какую формулу это объясняет;
- где это место в лекции;
- что изменится в следующем уровне.

Важно: это не лекционный текст внутри приложения, а 1-2 короткие строки после
успеха.

### 2. Карты курса

Пока миссии существуют как отдельные точки. Нужно показать учебный маршрут:

```text
линейные уравнения -> ранг -> определитель -> матрицы -> пространства ->
линейные отображения
```

Пользователь должен видеть, что миссии соединены в цепочку, а не разбросаны по
sidebar.

### 3. Разных механик

Сейчас три миссии в основном геометрические. Это хороший старт, но если
продолжать только в этом стиле, приложение станет "геометрической песочницей"
и плохо покроет программирование, алгоритмы и анализ данных.

Следующий этап должен добавить хотя бы одну дискретную/пошаговую механику:

- перестановки;
- ранг через операции;
- алгоритмический trace;
- графовый обход.

### 4. Сильнее игрового feedback

Инвариант-ключи есть, но момент награды пока слабый. Нужны:

- короткая анимация ключа;
- явное открытие следующего уровня;
- финальный summary миссии;
- маленький "что ты теперь умеешь".

### 5. Единых ссылок из лекций

Ссылки сейчас неоднородны: часть ведет на `interactive/dist/index.html#...`,
часть на `interactive/#...`. Нужно выбрать один формат для локальной работы и
один формат для опубликованной сборки.

## Продуктовая цель этапа

Сделать из набора миссий **учебный маршрут по линейной алгебре**:

- пользователь видит карту тем;
- каждая миссия заканчивается смысловым выводом;
- текущие 3 миссии полируются до "показывать другому человеку не стыдно";
- добавляется одна новая не-геометрическая миссия;
- приложение получает основу для programming/data-analysis механик.

## Приоритеты

### P0. Закрыть дидактический цикл текущих миссий

Добавить в `MissionLevel` или соседнюю структуру:

```ts
takeaway: string
lectureAnchor?: string
nextPrompt?: string
```

Использовать это в `MissionShell` после успеха:

- `successText` - эмоциональная реакция;
- `takeaway` - математический вывод;
- `lectureAnchor` - ссылка/подпись к месту в лекции;
- `nextPrompt` - зачем идти дальше.

Примеры:

- Kernel Hunt: "Ядро - не случайный набор решений, а подпространство."
- Determinant Forge: "Модуль det - площадь, знак det - ориентация."
- Matrix Machine: "Столбцы матрицы - это образы базисных векторов."

Готово, когда после прохождения уровня можно вслух сказать, чему он научил.

### P0. Полировать current three

Kernel Hunt:

- сделать 3D drag понятнее: подсветить плоскость drag и показать, что `z`
  меняется slider'ом;
- добавить visual distance-to-kernel indicator;
- сделать уровень rank-nullity не просто "еще раз выставь вектор", а сборку
  равенства `3 = 2 + 1`.

Determinant Forge:

- увеличить handles и hit area;
- добавить явный target ghost для площади/ориентации;
- сделать вырожденность заметной как "форма схлопнулась".

Matrix Machine:

- добавить ghost-матрицу цели рядом с текущей;
- добавить reset current level;
- сделать уровни визуально разными, а не только разными координатами.

Готово, когда каждая миссия дает визуально разный опыт, а не один и тот же
паттерн "подтяни ручку к числу".

### P0. Согласовать route links из лекций

Сделать одну политику:

- в исходных лекциях использовать dev-friendly hash route:
  `../../interactive/#/algebra/...`;
- для публикации иметь documented replacement на
  `../../interactive/dist/index.html#/algebra/...`.

Или наоборот, но единообразно.

Проверить и поправить:

- `SHAD/algebra/5_Det/lesson.md`;
- `SHAD/algebra/6_Matrices/lesson.md`;
- `SHAD/algebra/8_Linear_maps/lesson.md`.

Готово, когда `rg "interactive" SHAD/algebra -g lesson.md` не показывает
смешанный стиль ссылок.

### P1. Добавить миссию "Цех перестановок"

Почему именно она:

- это первая дискретная механика, не похожая на геометрические плоскости;
- материал уже есть в `SHAD/algebra/1_Substitutions/`;
- хорошо работает как игра: поменяй элементы, собери циклы, сохрани/измени
  четность;
- вводит механику, полезную для алгоритмов: состояние, операция, стоимость.

Механика:

- есть ряд из 5-7 элементов;
- пользователь делает транспозиции;
- приложение показывает циклы, знак, число ходов;
- цель уровня: собрать заданную перестановку, разложить на циклы, получить
  четную/нечетную перестановку, исправить ошибку минимальным числом swaps.

Файлы:

- `src/visualizations/substitutions/substitutionWorkshopModel.ts`;
- `src/visualizations/substitutions/SubstitutionWorkshopMission.tsx`;
- test file для parity/cycles/composition;
- запись в `missions.ts`, `missionRegistry.ts`, `routeLoaders.ts`,
  `navigation.ts`;
- ссылка из `SHAD/algebra/1_Substitutions/lesson.md`.

Готово, когда новая миссия использует общий shell, но ощущается не как
геометрическая сцена.

### P1. Карта линейной алгебры

Добавить отдельный route или sidebar mode:

```text
Линейная алгебра
1. Уравнения
2. Ранг
3. Определитель
4. Матрицы
5. Пространства
6. Отображения
```

На первом шаге это может быть не новая страница, а улучшенная структура
навигации:

- группировать playable missions и planned missions;
- показывать "следующую рекомендуемую";
- показывать, какие лекции связаны с миссией.

Готово, когда пользователь понимает, какую миссию проходить после текущей.

### P1. Mission completion summary

После прохождения всех уровней миссии показать компактный финал:

- 3-4 собранных вывода;
- сколько ключей получено;
- ссылка на связанную лекцию;
- следующая рекомендуемая миссия.

Без модального окна на каждый уровень. Лучше небольшая финальная панель в
правой колонке/под сценой.

### P2. Подготовить механику для programming/data-analysis

Не реализовывать большой редактор кода. Сначала добавить типы и один
прототип данных:

```ts
type TraceStep = {
  id: string
  label: string
  state: Record<string, unknown>
  cost?: number
  invariantOk?: boolean
}
```

Кандидаты:

- `CodeTraceRunner` для инварианта цикла;
- `ComplexityRace` для сравнений сортировок;
- `SamplerLab` для выборки и разброса статистик.

Готово, когда есть design/data contract, но без преждевременного code editor.

## Порядок работ

1. Обновить `MissionLevel` и `MissionShell`: `takeaway`, `lectureAnchor`,
   `nextPrompt`.
2. Дописать takeaways для Kernel Hunt, Determinant Forge, Matrix Machine.
3. Добавить финальный mission summary.
4. Полировать UX трех текущих миссий.
5. Согласовать ссылки из лекций на interactive.
6. Добавить "Цех перестановок" как первую дискретную миссию.
7. Добавить tests для перестановок и расширить Playwright smoke.
8. Обновить README/gameplay roadmap.
9. Прогнать `make interactive-quality`.

## Definition of Done

- У каждой текущей миссии есть level takeaways и финальный summary.
- Ссылки из лекций на interactive единообразны.
- Добавлена playable mission "Цех перестановок".
- Новая миссия покрыта unit tests и Playwright happy path.
- Mobile screenshots не имеют горизонтального overflow.
- `make interactive-quality` проходит.

## Статус выполнения

Этап реализован как первый содержательный слой поверх игрового каркаса:

- `MissionLevel` получил `takeaway`, `lectureAnchor`, `nextPrompt`.
- `MissionShell` показывает смысловой вывод уровня и финальный summary миссии.
- Kernel Hunt, Determinant Forge и Matrix Machine получили takeaways, summaries
  и следующую рекомендуемую миссию.
- Kernel Hunt стал понятнее за счет drag-диска и индикатора ошибки до ядра.
- Determinant Forge получил более крупные handles, hit area и target hint.
- Matrix Machine получил target-матрицу, reset и числовые inputs как
  устойчивый способ пройти уровни на desktop/mobile.
- Ссылки из лекций приведены к одному формату `../../interactive/#/...`.
- Добавлена playable mission "Цех перестановок" с чистой model-логикой,
  unit tests, маршрутом, навигацией и ссылкой из лекции про подстановки.
- Playwright smoke расширен desktop/mobile скриншотами и happy path для
  четырех миссий.
- Для будущих programming/data-analysis миссий добавлен минимальный
  `TraceStep` / `MissionDataContract`.

Осознанно оставлено на следующий этап:

- отдельная визуальная карта курса вместо улучшенной структуры навигации;
- более выразительная reward-анимация инвариант-ключей;
- замена устаревшего `THREE.Clock` на `THREE.Timer`, если предупреждение
  начнет мешать или three обновит поведение;
- первый настоящий алгоритмический trace, например BFS/DFS или инвариант цикла.

## Честные риски

- Если добавить много миссий без takeaways, приложение станет приятной
  песочницей, но слабым учебным инструментом.
- Если слишком рано делать универсальный движок, мы потратим время на
  абстракции вместо учебной ценности.
- Если оставить только geometry-lab механику, программирование и анализ данных
  потом придется втискивать в неподходящий интерфейс.
