# SHAD Interactive: accessibility baseline

Этот baseline обязателен для новых миссий. Он не заменяет полноценный аудит, но
задает минимальный уровень, ниже которого интерактив не должен опускаться.

## Основные правила

- Основное действие миссии должно иметь keyboard path или input fallback.
- Drag-heavy сцены обязаны иметь числовой/кнопочный fallback для прохождения.
- Интерактивные SVG/DOM элементы должны иметь понятный `aria-label`.
- Focus state должен быть видимым.
- Цвет не должен быть единственным носителем смысла: нужны подписи, форма или
  текстовое состояние.
- Mobile tap targets должны быть достаточно крупными для уверенного нажатия.
- Анимации должны уважать `prefers-reduced-motion`.

## Canvas и 3D

Для canvas/Three.js сцен:

- дать альтернативный способ ввода через controls;
- не полагаться только на вращение камеры;
- показывать численное состояние рядом с canvas;
- использовать stable `data-testid` для Playwright.

## SVG

Для SVG-сцен:

- интерактивные группы получают `role="button"` и `tabIndex={0}`;
- Enter/Space повторяют click-действие;
- `aria-label` должен называть объект действия;
- visual state должен отличаться не только цветом.

## Проверка перед завершением

- пройти миссию мышью;
- пройти mission happy path Playwright;
- проверить mobile screenshot;
- проверить, есть ли fallback для drag;
- убедиться, что reduced motion не ломает смысл reward/feedback.

## Фокус и типографика (2026-06-09)

- Видимый фокус клавиатуры: глобальное правило `:focus-visible` в `index.css`
  даёт оранжевый outline + offset на все интерактивные элементы (ссылки,
  кнопки, inputs, SVG-группы с `role="button"`). Раньше видимого фокуса не было.
- Шрифты подобраны под читаемость кириллицы: body — Golos Text (humanist sans,
  родная кириллица), заголовки — Literata (антиква с кириллицей). Оба
  self-hosted, `font-display: swap`.
- `card-rise`-анимация входа карты добавлена в `prefers-reduced-motion`-блок.

Текущий реализованный baseline: reward-анимация и card-rise отключаются через
`prefers-reduced-motion`, графовая/эйлерова миссии имеют keyboard path для
SVG-вершин, а клавиатурный фокус виден на всех интерактивных элементах.
