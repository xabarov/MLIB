# MLIB / SHAD: визуальный стиль иллюстраций для лекций (LLM)

Документ фиксирует **единый визуальный язык** для картинок к лекциям: сгенерированных через `generate_images.py` (OpenRouter), а также как ориентир для ручных иллюстраций и постобработки.

Основа палитры — **спокойный “editorial / research” вид**, сознательно близкий к публично описываемой эстетике продуктов вроде Claude (теплый бумажный фон, приглушенные акценты). Это **внутренний учебный стиль проекта**, а не официальный брендбук сторонней компании.

Все вспомогательные файлы для LLM-визуалов лежат в каталоге **`SHAD/lecture_visual_generation/`** (этот файл, суффикс промпта, пример JSON).

---

## Цели

- Картинки **читаются рядом с текстом** в Markdown (lesson / qa): не перетягивают внимание, не спорят по цвету с типовым светлым фоном редактора.
- **Одинаковая “температура”** кадра от лекции к лекции: студент не получает визуальный зоопарк.
- Допускаются как **чисто декоративные** обложки/герои, так и **схемы** — но схемы должны оставаться честными (без выдуманных чисел и “фейковых дашбордов”).

---

## Палитра (токены)

| Роль | HEX | Применение |
|------|-----|------------|
| Фон (светлый) | `#faf9f5` | Основной фон слайда / иллюстрации |
| Текст / контур | `#141413` | Главные линии, крупные формы, “чернила” |
| Вторичный серый | `#b0aea5` | Вспомогательные линии, подписи-намеки, “воздух” |
| Панели / карточки | `#e8e6dc` | Мягкие блоки, разделители |
| Акцент 1 (теплый) | `#d97757` | Фокус, важный узел, “тепло” |
| Акцент 2 (холодный) | `#6a9bcc` | Второй акцент, связь, “структура” |
| Акцент 3 (природный) | `#788c5d` | Третий уровень акцента, “валидность / рост” |

Правило: **в одном кадре 1–2 акцента**, остальное — нейтраль; не смешивать все три акцентных цвета в равных долях без причины.

## Типографика (если на картинке есть текст)

- Предпочтительно **без длинного текста** на изображении: заголовки и объяснения живут в Markdown.
- Если подписи необходимы: **короткие лейблы** (слово–три), нейтральный гуманистический или аккуратный sans, высокий контраст к фону.
- Избегать: мелкий бинарный шрифт, “техно” глитч, водяные знаки, логотипы вымышленных продуктов.

*(Для слайдов в коде/HTML можно ориентироваться на пары вроде Poppins / Lora — для LLM-рендера шрифт задаётся описательно в промпте.)*

---

## Композиция и свет

- **Много воздуха**, явный фокус (одна главная идея на картинку).
- Мягкий **диффузный свет**, лёгкая глубина (слои карточек/плоскостей), без агрессивных неоновых градиентов.
- Текстуры: **лёгкая бумажная зернистость** или очень слабый виньет — допустимы, если не превращаются в шум.

---

## Запреты (anti-patterns)

- Фейковые метрики, оси с выдуманными числами, “живые” дашборды.
- Клише “AI”: голографические мозги, неоновые сети без смысла, stock-роботы.
- Перегруженный “product mockup” интерфейса с несуществующим продуктом.
- Перегруз текста на изображении (абзацы, длинные формулировки).

---

## Соотношение сторон

- По умолчанию **16:9** для героев и обложек к лекциям (удобно в презентациях и широких layout’ах).
- Для врезок в узкие колонки можно явно запросить в промпте **4:3** или **1:1**.

---

## Графики и анимации из кода

Скрипты вида `*/generate_visuals.py` (matplotlib / imageio) остаются основным способом для **точных** диаграмм. LLM-иллюстрации — для **метафор, обложек, неточных схем**, где важна атмосфера и единый стиль. При желании выровнять matplotlib под эту систему — использовать те же HEX для линий и заливок.

## Машиночитаемый суффикс промпта

Короткий блок, который по умолчанию дописывается к каждому запросу в `generate_images.py`, хранится в:

`SHAD/lecture_visual_generation/lecture_visual_prompt_suffix.txt`

Его можно редактировать без правки кода. В `generate_images.py` можно отключить суффикс флагом или указать другой файл.

---

## Связанные файлы

| Файл | Назначение |
|------|------------|
| `generate_images.py` (в корне `mlib/`) | CLI: одиночный промпт или пакет из JSON |
| `SHAD/lecture_visual_generation/lecture_visual_prompt_suffix.txt` | Стиль по умолчанию для модели |
| `SHAD/lecture_visual_generation/lecture_images.example.json` | Пример списка заданий |

## Переменные окружения (`generate_images.py`)

Задаются в `mlib/.env` или в оболочке перед запуском:

| Переменная | Назначение |
|------------|------------|
| `LLM_API_KEY` | Ключ OpenRouter |
| `LLM_BASE_URL` | База API, например `https://openrouter.ai/api/v1` |
| `LLM_VISUAL_GENERATION_MODEL` | Модель с выдачей изображения (например `google/gemini-3.1-flash-image-preview`) |
| `LLM_HTTP_REFERER` | (опционально) заголовок `HTTP-Referer` для OpenRouter |
| `LLM_APP_TITLE` | (опционально) заголовок `X-Title` |

Примеры запуска из каталога `mlib`:

```bash
# Просмотр итогового промпта без запроса к API
.venv/bin/python generate_images.py --dry-run --jobs SHAD/lecture_visual_generation/lecture_images.example.json

# Один кадр
.venv/bin/python generate_images.py --slug my_topic --prompt-file path/to/prompt.txt

# Пакет
.venv/bin/python generate_images.py --jobs SHAD/my_lecture_images.json --out-dir SHAD/algebra/7_Vector_spaces/assets
```

Каталог вывода по умолчанию: `SHAD/_generated_lecture_images` (можно переопределить `--out-dir`).

**Безопасность:** не коммитьте `.env` с ключами; при утечке ключа перевыпустите его в кабинете провайдера.
