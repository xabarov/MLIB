## Задача 1. Трассировка бинарного поиска

Дан отсортированный массив `a = [2, 5, 8, 12, 16, 23, 38, 56, 72, 91]`. Применить бинарный поиск для нахождения элемента `23`. Показать значения lo, hi, mid на каждой итерации.

### Решение

Размер массива: $n = 10$. Индексы: 0..9.

**Итерация 1:**
- $lo = 0$, $hi = 10$
- $mid = 0 + (10 - 0) / 2 = 5$
- $a[5] = 23 = \text{target}$ → возвращаем 5.

Бинарный поиск нашёл элемент за одну итерацию. Повезло — `mid` сразу совпал с ответом.

Разберём более длинный пример: ищем `38`.

**Итерация 1:**
- $lo = 0$, $hi = 10$, $mid = 5$, $a[5] = 23$
- $23 < 38$ → $lo = 6$

**Итерация 2:**
- $lo = 6$, $hi = 10$, $mid = 6 + (10-6)/2 = 8$, $a[8] = 72$
- $72 > 38$ → $hi = 8$

**Итерация 3:**
- $lo = 6$, $hi = 8$, $mid = 7$, $a[7] = 56$
- $56 > 38$ → $hi = 7$

**Итерация 4:**
- $lo = 6$, $hi = 7$, $mid = 6$, $a[6] = 38$
- $38 = 38$ → return 6.

### Ответ

Элемент `23` находится на позиции 5 (за 1 итерацию). Элемент `38` — на позиции 6 (за 4 итерации).

## Задача 2. lower\_bound вручную

Дан массив `a = [1, 3, 3, 5, 5, 5, 7, 9]`. Найти вручную (трассировкой) `lower_bound(6)` и `upper_bound(5)`.

### Решение

**lower\_bound(6): первый индекс i, где a[i] >= 6.**

$n = 8$, $lo = 0$, $hi = 8$.

| Шаг | lo | hi | mid | a[mid] | Условие a[mid] < 6 | Действие |
|-----|----|----|-----|--------|--------------------|----------|
| 1   | 0  | 8  | 4   | 5      | Да (5 < 6)         | lo = 5   |
| 2   | 5  | 8  | 6   | 7      | Нет (7 ≥ 6)        | hi = 6   |
| 3   | 5  | 6  | 5   | 5      | Да (5 < 6)         | lo = 6   |
| 4   | lo = hi = 6 | | | |                    | return 6 |

`a[6] = 7` — первый элемент ≥ 6. Корректно.

**upper\_bound(5): первый индекс i, где a[i] > 5.**

$lo = 0$, $hi = 8$. Условие: `a[mid] <= 5` → lo = mid + 1.

| Шаг | lo | hi | mid | a[mid] | Условие a[mid] <= 5 | Действие |
|-----|----|----|-----|--------|---------------------|----------|
| 1   | 0  | 8  | 4   | 5      | Да                  | lo = 5   |
| 2   | 5  | 8  | 6   | 7      | Нет                 | hi = 6   |
| 3   | 5  | 6  | 5   | 5      | Да                  | lo = 6   |
| 4   | lo = hi = 6 | | | |                     | return 6 |

`a[6] = 7` — первый элемент > 5. Корректно. Число вхождений 5: `upper_bound(5) - lower_bound(5) = 6 - 3 = 3`.

### Ответ

`lower_bound(6) = 6` (a[6] = 7). `upper_bound(5) = 6` (a[6] = 7). Число вхождений 5 равно 3.

## Задача 3. Целочисленный квадратный корень

Найти минимальное целое $x \geq 0$ такое, что $x^2 \geq 17$, методом бинарного поиска по ответу.

### Решение

Предикат: $f(x) = (x^2 \geq 17)$. Он монотонен: False при $x \leq 3$ (так как $3^2 = 9 < 17$), True при $x \geq 5$ (так как $5^2 = 25 \geq 17$), $4^2 = 16 < 17$, $5^2 = 25 \geq 17$.

Начальный диапазон: $lo = 0$, $hi = 17$ (заведомо достаточно, т.к. $17^2 \geq 17$).

| Шаг | lo | hi | mid | f(mid): mid²≥17? | Действие |
|-----|----|----|-----|-----------------|----------|
| 1   | 0  | 17 | 8   | 64 ≥ 17: True   | hi = 8   |
| 2   | 0  | 8  | 4   | 16 ≥ 17: False  | lo = 5   |
| 3   | 5  | 8  | 6   | 36 ≥ 17: True   | hi = 6   |
| 4   | 5  | 6  | 5   | 25 ≥ 17: True   | hi = 5   |
| 5   | lo = hi = 5 | | | |              | return 5 |

Проверка: $5^2 = 25 \geq 17$ ✓, $4^2 = 16 < 17$ ✓. Значит $\lceil\sqrt{17}\rceil = 5$.

### Ответ

Минимальное $x$ такое, что $x^2 \geq 17$: $x = 5$.

## Задача 4. Анализ варианта с ошибкой

Рассмотрим следующую реализацию бинарного поиска:

```cpp
int search(const int* a, int n, int target) {
    int lo = 0, hi = n - 1;
    while (lo <= hi) {
        int mid = (lo + hi) / 2;
        if (a[mid] == target) return mid;
        if (a[mid] < target) lo = mid + 1;
        else hi = mid - 1;
    }
    return -1;
}
```

(а) Есть ли здесь ошибка? (б) Что произойдёт при вызове `search(a, 0, 5)` на пустом массиве?

### Решение

**(а) Анализ корректности.**

Реализация использует замкнутый интервал $[lo, hi]$ вместо полуоткрытого $[lo, hi)$. Это тоже допустимо, если обновления согласованы:
- $a[mid] < target$: $lo = mid + 1$ — правильно, исключаем mid.
- $a[mid] > target$: $hi = mid - 1$ — правильно, исключаем mid.

При $n > 0$ алгоритм корректен.

Единственная **потенциальная ошибка** — переполнение: `mid = (lo + hi) / 2`. Если `lo` и `hi` оба близки к `INT_MAX`, сумма переполнится. На практике при поиске в массиве $n \leq 10^9$ индексы влезают в int, но безопаснее `lo + (hi - lo) / 2`.

**(б) Пустой массив: `n = 0`.**

При $n = 0$: $lo = 0$, $hi = -1$. Условие цикла `lo <= hi` сразу ложно. Функция вернёт -1. Корректно.

Для реализации с `hi = n` (полуоткрытый): $lo = 0$, $hi = 0$, `lo < hi` тоже ложно сразу. Тоже корректно.

### Ответ

Ошибки нет, кроме потенциального переполнения при вычислении `mid`. На пустом массиве функция корректно возвращает -1.

## Задача 5. Количество итераций

Дан отсортированный массив из $n = 10^6$ элементов. (а) Сколько итераций в худшем случае выполнит бинарный поиск? (б) Во сколько раз быстрее он работает по сравнению с линейным поиском в худшем случае?

### Решение

**(а) Число итераций бинарного поиска.**

На каждой итерации размер области $[lo, hi)$ уменьшается как минимум вдвое. Начальный размер — $n = 10^6$.

Процесс: $10^6 \to 5 \cdot 10^5 \to \ldots \to 1 \to 0$.

Число итераций: $\lceil \log_2 n \rceil = \lceil \log_2 10^6 \rceil$.

$$
\log_2 10^6 = 6 \log_2 10 \approx 6 \times 3.3219 \approx 19.93
$$

Значит, $\lceil \log_2 10^6 \rceil = 20$ итераций.

**(б) Ускорение.**

Линейный поиск в худшем случае: $n = 10^6$ сравнений.
Бинарный поиск в худшем случае: $20$ сравнений.

Ускорение: $\dfrac{10^6}{20} = 50\,000$ раз.

### Ответ

Бинарный поиск выполнит не более 20 итераций. Это в 50 000 раз меньше, чем у линейного поиска.

## Задача 6. Поиск граничного элемента

Дан отсортированный массив из $n$ элементов. Требуется найти количество элементов, строго меньших $x$, за O(log n). Напишите функцию и обоснуйте корректность.

### Решение

Количество элементов, строго меньших $x$, равно `lower_bound(x)` — индексу первого элемента $\geq x$. Все элементы до этого индекса строго меньше $x$.

```cpp
// Возвращает количество элементов строго меньше x в отсортированном a[0..n-1].
int count_less(const int* a, int n, int x) {
    int lo = 0, hi = n;
    // Инвариант: a[lo-1] < x  и  a[hi] >= x (граничные условия)
    while (lo < hi) {
        int mid = lo + (hi - lo) / 2;
        if (a[mid] < x) lo = mid + 1;
        else            hi = mid;
    }
    return lo;  // = количество элементов < x
}
```

**Корректность:**

Инвариант: $a[lo - 1] < x \leq a[hi]$ (при соглашении $a[-1] = -\infty$, $a[n] = +\infty$).

- Инициализация: $lo = 0$, $hi = n$. Тривиально: нет элементов левее lo, все правее hi.
- Шаг: если $a[mid] < x$, смещаем lo = mid + 1 (a[mid] < x исключает mid); иначе hi = mid (a[mid] >= x сохраняет инвариант).
- Завершение: lo = hi, возвращаем lo. Все индексы $< lo$ содержат элементы $< x$, все индексы $\geq lo$ — элементы $\geq x$.

Сложность: O(log n).

### Ответ

Функция `count_less` возвращает правильный ответ за O(log n). Она эквивалентна `std::lower_bound(a, a+n, x) - a`.

## Задача 7. Нижняя оценка

Докажите, что любой алгоритм, основанный только на сравнениях, требует в худшем случае $\lceil \log_2(n+1) \rceil$ сравнений для поиска одного элемента в отсортированном массиве из $n$ элементов.

### Решение

**Модель.** Алгоритм — детерминированная программа, использующая только сравнения вида $a[i] \mathbin{?} t$. Представим его как бинарное дерево решений: каждый внутренний узел — одно сравнение, два исхода (меньше / больше-или-равно), каждый лист — ответ.

**Число возможных ответов.** При поиске в массиве размера $n$ ответ может быть одним из $n+1$ значений: найден элемент на позиции $0, 1, \ldots, n-1$ или «не найден».

**Нижняя оценка.** Число листьев дерева $\geq n + 1$. Бинарное дерево высоты $h$ имеет не более $2^h$ листьев. Значит:

$$
2^h \geq n + 1 \implies h \geq \log_2(n+1) \implies h \geq \lceil \log_2(n+1) \rceil
$$

Высота дерева — максимальная длина пути из корня в лист — равна максимальному числу сравнений на любом входе. Следовательно, любой алгоритм делает не менее $\lceil \log_2(n+1) \rceil$ сравнений в худшем случае.

**Достижимость.** Бинарный поиск делает ровно $\lceil \log_2(n+1) \rceil$ сравнений — следовательно, оценка точная.

### Ответ

Нижняя оценка: $\lceil \log_2(n+1) \rceil$ сравнений. Доказывается через дерево решений с $n+1$ листом. Бинарный поиск оптимален.

## Задача 8. Поиск пика

Дан массив $a[0..n-1]$, у которого существует единственный пик: $a[0] < a[1] < \ldots < a[k] > a[k+1] > \ldots > a[n-1]$. Найти индекс пика за O(log n).

### Решение

**Ключевое наблюдение.** Пик — это индекс $k$, где $a[k] > a[k-1]$ и $a[k] > a[k+1]$. Если $a[mid] < a[mid+1]$, то пик находится правее $mid$ (массив ещё возрастает). Если $a[mid] > a[mid+1]$, то пик — это $mid$ или левее.

**Инвариант:** пик лежит в $[lo, hi]$.

```cpp
// Предусловие: a имеет единственный пик, n >= 1.
int find_peak(const int* a, int n) {
    int lo = 0, hi = n - 1;
    // Инвариант: пик ∈ [lo, hi]
    while (lo < hi) {
        int mid = lo + (hi - lo) / 2;
        if (a[mid] < a[mid + 1])
            lo = mid + 1;  // пик правее mid
        else
            hi = mid;      // mid — кандидат, пик в [lo, mid]
    }
    return lo;  // lo == hi — пик
}
```

**Корректность инварианта:**
- Если $a[mid] < a[mid+1]$: массив возрастает в точке $mid$, значит пик правее → $lo = mid + 1$ сохраняет инвариант.
- Если $a[mid] \geq a[mid+1]$: $mid$ может быть пиком или пик левее → $hi = mid$ сохраняет инвариант.

Завершение: $hi - lo$ убывает ⟹ цикл конечен.

**Пример.** $a = [1, 3, 7, 5, 2]$, $n = 5$.

| lo | hi | mid | a[mid] vs a[mid+1] | Действие |
|----|----|----|---------------------|----------|
| 0  | 4  | 2  | 7 > 5               | hi = 2   |
| 0  | 2  | 1  | 3 < 7               | lo = 2   |
| lo = hi = 2 — return 2 | | | | |

$a[2] = 7$ — пик. Верно.

### Ответ

Алгоритм находит пик за $O(\log n)$ итераций, поддерживая инвариант «пик лежит в $[lo, hi]$».

## Задача 9. Минимум в ротированном массиве

Дан массив, полученный ротацией отсортированного: $a = [5, 7, 9, 1, 3]$ (или $[1, 3, 5, 7, 9]$). Найти минимум за O(log n).

### Решение

**Ключевое наблюдение.** Ротация создаёт один «излом»: минимум — это начало правой (меньшей) части. Если $a[mid] > a[hi - 1]$, то излом находится справа от mid → минимум в правой половине. Иначе минимум в левой половине (включая mid).

**Инвариант:** минимум в $[lo, hi)$.

```cpp
// Найти индекс минимума в ротированном отсортированном массиве без дубликатов.
int find_min_rotated(const int* a, int n) {
    int lo = 0, hi = n - 1;
    // Инвариант: минимум ∈ [lo, hi]
    while (lo < hi) {
        int mid = lo + (hi - lo) / 2;
        if (a[mid] > a[hi])
            lo = mid + 1;  // минимум правее mid
        else
            hi = mid;      // минимум в [lo, mid]
    }
    return lo;
}
```

**Трассировка для [5, 7, 9, 1, 3], n = 5:**

| lo | hi | mid | a[mid] | a[hi] | a[mid]>a[hi]? | Действие |
|----|----|----|--------|-------|---------------|----------|
| 0  | 4  | 2  | 9      | 3     | Да            | lo = 3   |
| 3  | 4  | 3  | 1      | 3     | Нет           | hi = 3   |
| lo = hi = 3 — return 3 | | | | | | |

$a[3] = 1$ — минимум. Верно.

**Случай без ротации:** [1, 3, 5, 7, 9]: $a[mid] \leq a[hi]$ всегда → hi уменьшается до 0 → $a[0] = 1$.

### Ответ

Алгоритм находит минимум в ротированном отсортированном массиве (без дубликатов) за O(log n).

## Задача 10. Синтез: кабели и мастера

**Условие.** Есть $n$ кабелей длиной $l_1, l_2, \ldots, l_n$ (целые числа). Нужно нарезать из них не менее $k$ кусков одинаковой длины $x$ (каждый кусок целой длины). Найти максимально возможное $x$.

Ограничения: $n \leq 10^4$, $l_i \leq 10^9$, $k \leq 10^7$.

### Решение

**Шаг 1: Формулируем предикат.**

$f(x)$ = «можно нарезать не менее $k$ кусков длины $x$».

Из кабеля длиной $l_i$ можно нарезать $\lfloor l_i / x \rfloor$ кусков. Суммарно:

$$
f(x) = \left(\sum_{i=1}^{n} \lfloor l_i / x \rfloor \geq k\right)
$$

**Шаг 2: Монотонность.**

При увеличении $x$ количество кусков из каждого кабеля не растёт ($\lfloor l_i / x \rfloor$ убывает), значит $f$ убывает: существует порог $x^*$ такой, что $f(x) = \text{True}$ при $x \leq x^*$ и $f(x) = \text{False}$ при $x > x^*$.

Нам нужен максимальный $x$ с $f(x) = \text{True}$ — это эквивалентно минимальному $x$ с $f(x) = \text{False}$ минус один.

**Шаг 3: Граница поиска.**

- $lo = 1$ (минимальная длина куска).
- $hi = \max(l_i)$ (нельзя нарезать куски длиннее самого длинного кабеля).
- При $x = 1$ всегда $f(1) = \text{True}$ (если $\sum l_i \geq k$), поэтому $lo$ — допустимое значение.

**Шаг 4: Код.**

```cpp
#include <iostream>
#include <vector>
#include <algorithm>

// f(x): можно нарезать >= k кусков длиной x
bool can_cut(const std::vector<long long>& l, long long k, long long x) {
    long long total = 0;
    for (long long li : l) {
        total += li / x;
        if (total >= k) return true;  // ранний выход
    }
    return total >= k;
}

long long max_piece(const std::vector<long long>& l, long long k) {
    // Ищем максимальный x с f(x) = true.
    // Эквивалентно: минимальный x с f(x) = false, затем x - 1.
    long long lo = 1, hi = *std::max_element(l.begin(), l.end());
    if (!can_cut(l, k, lo)) return 0;  // невозможно даже для x=1

    // Ищем последнее True: двигаем lo к первому False минус 1.
    // Перепишем: бинарный поиск на максимум True.
    // lo всегда True, hi+1 всегда False.
    // Инвариант: f(lo) = true, f(hi+1) = false
    long long ans = lo;
    while (lo <= hi) {
        long long mid = lo + (hi - lo) / 2;
        if (can_cut(l, k, mid)) {
            ans = mid;   // mid подходит, пробуем больше
            lo = mid + 1;
        } else {
            hi = mid - 1;  // mid не подходит
        }
    }
    return ans;
}

int main() {
    std::vector<long long> l = {10, 10, 10, 10};
    std::cout << max_piece(l, 11) << "\n";  // 3
    std::cout << max_piece(l, 9)  << "\n";  // 4
    std::cout << max_piece(l, 4)  << "\n";  // 10
}
```

**Трассировка для l = [10,10,10,10], k = 11:**

$lo = 1$, $hi = 10$. $\sum \lfloor 10/x \rfloor = 40/x$ кусков.

| lo | hi | mid | Кусков (40/mid) | ≥ 11? | ans  | Действие |
|----|----|----|-----------------|-------|------|----------|
| 1  | 10 | 5  | 8               | Нет   | —    | hi = 4   |
| 1  | 4  | 2  | 20              | Да    | 2    | lo = 3   |
| 3  | 4  | 3  | 13              | Да    | 3    | lo = 4   |
| 4  | 4  | 4  | 10              | Нет   | —    | hi = 3   |
| lo > hi — return ans = 3 | | | | | | |

Ответ: 3. Проверка: $4 \times \lfloor 10/3 \rfloor = 4 \times 3 = 12 \geq 11$ ✓. $4 \times \lfloor 10/4 \rfloor = 4 \times 2 = 8 < 11$ ✓.

**Сложность:** O(n log(max_l)) — логарифм от максимальной длины кабеля.

### Ответ

Бинарный поиск по ответу с предикатом $f(x) = (\sum \lfloor l_i/x \rfloor \geq k)$ находит максимальную длину куска за O(n log(max_l)).
