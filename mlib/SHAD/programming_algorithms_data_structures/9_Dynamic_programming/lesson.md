# Лекция 9: Динамическое программирование

![Динамическое программирование](assets/hero_dp.png)

Динамическое программирование (ДП) — один из центральных алгоритмических методов: он превращает экспоненциальный перебор в полиномиальный, сохраняя результаты подзадач, которые встречаются снова и снова. Метод появился в 1950-х годах в работах Ричарда Беллмана и сегодня лежит в основе алгоритмов сжатия текста, биоинформатики, задач планирования и машинного обучения. Для поступления в ШАД ДП — обязательная тема: задачи на редакционное расстояние и рюкзак встречаются на каждом вступительном экзамене. Лекция строится на базовых знаниях рекурсии и массивов.

Главная линия лекции:

$$
\text{Рекурсия с повторениями} \;\to\; \text{Мемоизация (top-down)} \;\to\; \text{Таблица (bottom-up)}
$$

**Как читать эту лекцию:**
- Раздел 1 объясняет два ключевых свойства задач ДП. Читайте внимательно — без них остальное непонятно.
- Раздел 2 (числа Фибоначчи) — простейший разогрев. Выпишите дерево вызовов для F(5) на бумаге.
- Разделы 3 и 4 — два главных учебных примера. Трассируйте таблицы вручную.
- Раздел 5 (восстановление ответа) — техника, без которой половина задач не решается.
- Раздел 6 — шпаргалка по признакам ДП-задач.

## 1. Идея ДП: оптимальная подструктура и перекрывающиеся подзадачи

### Два ключевых свойства

Задача допускает решение методом ДП тогда и только тогда, когда выполнены оба условия.

**Оптимальная подструктура** (Optimal Substructure): оптимальное решение задачи строится из оптимальных решений её подзадач. Иначе говоря, если мы уже знаем оптимальный ответ для всех меньших подзадач, мы можем за один шаг получить оптимальный ответ для большей задачи.

**Перекрывающиеся подзадачи** (Overlapping Subproblems): при рекурсивном разложении одни и те же подзадачи встречаются многократно. Именно здесь ДП выигрывает: вместо того чтобы решать одну подзадачу снова и снова, мы решаем её один раз и сохраняем результат.

### Сравнение с «разделяй и властвуй»

В методе «разделяй и властвуй» (например, сортировка слиянием) подзадачи **не пересекаются** — каждая решается ровно один раз, поэтому кэширование не нужно. В ДП подзадачи пересекаются, и без кэширования сложность взрывается экспоненциально.

| Метод | Подзадачи пересекаются? | Пример |
|---|---|---|
| Разделяй и властвуй | Нет | Сортировка слиянием |
| ДП | Да | Числа Фибоначчи, Edit Distance |

### Два способа реализации

**Top-down (рекурсия + мемоизация):** решаем задачу рекурсивно, но перед вычислением проверяем, не посчитан ли уже ответ. Если посчитан — возвращаем его из таблицы. Естественно выражает рекуррентное соотношение.

**Bottom-up (итеративная таблица):** заполняем таблицу в порядке возрастания размера подзадач. Нет накладных расходов на стек рекурсии, проще контролировать порядок заполнения.

$$
\text{Рекуррентность} + \text{Мемоизация} = \text{Top-down ДП}
$$

$$
\text{Порядок заполнения таблицы} + \text{База} = \text{Bottom-up ДП}
$$

## 2. Числа Фибоначчи: мотивирующий пример

### Задача

Числа Фибоначчи определяются рекуррентностью:

$$
F(0) = 0,\quad F(1) = 1,\quad F(n) = F(n-1) + F(n-2) \text{ при } n \ge 2
$$

### Наивная рекурсия: O(2ⁿ)

Каждый вызов порождает два новых. Для $F(5)$ дерево вызовов выглядит так (дублирующиеся вызовы показаны курсивом):

```
F(5)
├── F(4)
│   ├── F(3)
│   │   ├── F(2)  ← дубль
│   │   └── F(1)
│   └── F(2)      ← дубль
│       ├── F(1)
│       └── F(0)
└── F(3)          ← дубль
    ├── F(2)      ← дубль
    └── F(1)
```

$F(3)$ вычисляется 2 раза, $F(2)$ — 3 раза. Для $F(50)$ наивный алгоритм делает более $10^{10}$ вызовов.

```cpp
// Наивная рекурсия — O(2^n) время
long long fib_naive(int n) {
    if (n <= 1) return n;
    return fib_naive(n - 1) + fib_naive(n - 2);
}
```

### Top-down с мемоизацией: O(n) время, O(n) память

```cpp
#include <vector>
#include <iostream>

// Top-down: рекурсия + мемоизация
long long fib_memo(int n, std::vector<long long>& memo) {
    if (n <= 1) return n;
    if (memo[n] != -1) return memo[n];   // уже посчитано
    memo[n] = fib_memo(n - 1, memo) + fib_memo(n - 2, memo);
    return memo[n];
}

int main() {
    int n = 10;
    std::vector<long long> memo(n + 1, -1);
    std::cout << fib_memo(n, memo) << "\n";  // 55
    return 0;
}
```

Каждая подзадача $F(k)$ вычисляется ровно один раз — итого $n$ вычислений.

### Bottom-up итеративно: O(n) время, O(1) память

```cpp
// Bottom-up: итерация по таблице
long long fib_iter(int n) {
    if (n <= 1) return n;
    long long prev2 = 0, prev1 = 1;
    for (int i = 2; i <= n; i++) {
        long long cur = prev1 + prev2;
        prev2 = prev1;
        prev1 = cur;
    }
    return prev1;
}
```

Полная таблица не нужна — достаточно двух переменных, потому что $F(n)$ зависит только от двух предыдущих значений.

| Метод | Время | Память |
|---|---|---|
| Наивная рекурсия | $O(2^n)$ | $O(n)$ (стек) |
| Мемоизация | $O(n)$ | $O(n)$ |
| Итерация | $O(n)$ | $O(1)$ |

## 3. Редакционное расстояние (Edit Distance / Levenshtein)

### Постановка

Даны строки $s$ длины $m$ и $t$ длины $n$. Разрешены три операции, каждая стоимостью 1:
- **Вставка** символа в $s$
- **Удаление** символа из $s$
- **Замена** символа $s[i]$ на другой

Найти минимальное число операций для превращения $s$ в $t$.

### Рекуррентность

Определим $\text{dp}[i][j]$ — редакционное расстояние между $s[0..i{-}1]$ и $t[0..j{-}1]$.

**База:**

$$
\text{dp}[i][0] = i \quad (\text{удалить } i \text{ символов из } s)
$$

$$
\text{dp}[0][j] = j \quad (\text{вставить } j \text{ символов})
$$

**Рекуррентность:**

$$
\text{dp}[i][j] = \begin{cases}
\text{dp}[i{-}1][j{-}1] & \text{если } s[i{-}1] = t[j{-}1] \\
1 + \min\!\left(\text{dp}[i{-}1][j],\; \text{dp}[i][j{-}1],\; \text{dp}[i{-}1][j{-}1]\right) & \text{иначе}
\end{cases}
$$

Три варианта в случае несовпадения:
- $\text{dp}[i{-}1][j] + 1$: удалить $s[i{-}1]$
- $\text{dp}[i][j{-}1] + 1$: вставить $t[j{-}1]$
- $\text{dp}[i{-}1][j{-}1] + 1$: заменить $s[i{-}1]$ на $t[j{-}1]$

### Пример: "cat" → "cut"

|     |   | c | u | t |
|-----|---|---|---|---|
|     | 0 | 1 | 2 | 3 |
| **c** | 1 | 0 | 1 | 2 |
| **a** | 2 | 1 | 1 | 2 |
| **t** | 3 | 2 | 2 | 1 |

Ответ: $\text{dp}[3][3] = 1$ (заменить 'a' на 'u').

### Классический пример: "kitten" → "sitting" = 3 операции

1. kitten → sitten (замена 'k' на 's')
2. sitten → sittin (замена 'e' на 'i')
3. sittin → sitting (вставка 'g')

### Реализация

```cpp
#include <vector>
#include <string>
#include <algorithm>
#include <iostream>

// O(mn) время и O(mn) память
int edit_distance(const std::string& s, const std::string& t) {
    int m = s.size(), n = t.size();
    // dp[i][j] = edit distance между s[0..i-1] и t[0..j-1]
    std::vector<std::vector<int>> dp(m + 1, std::vector<int>(n + 1, 0));

    // База
    for (int i = 0; i <= m; i++) dp[i][0] = i;
    for (int j = 0; j <= n; j++) dp[0][j] = j;

    // Заполнение таблицы
    for (int i = 1; i <= m; i++) {
        for (int j = 1; j <= n; j++) {
            if (s[i - 1] == t[j - 1]) {
                dp[i][j] = dp[i - 1][j - 1];
            } else {
                dp[i][j] = 1 + std::min({dp[i - 1][j],    // удаление
                                          dp[i][j - 1],    // вставка
                                          dp[i - 1][j - 1] // замена
                                         });
            }
        }
    }
    return dp[m][n];
}

int main() {
    std::cout << edit_distance("kitten", "sitting") << "\n"; // 3
    std::cout << edit_distance("cat", "cut") << "\n";        // 1
    return 0;
}
```

### Оптимизация памяти: скользящий массив O(min(m,n))

Для вычисления ячейки $\text{dp}[i][j]$ нужны только строки $i{-}1$ и $i$. Храним две строки:

```cpp
int edit_distance_opt(const std::string& s, const std::string& t) {
    int m = s.size(), n = t.size();
    std::vector<int> prev(n + 1), curr(n + 1);
    for (int j = 0; j <= n; j++) prev[j] = j;

    for (int i = 1; i <= m; i++) {
        curr[0] = i;
        for (int j = 1; j <= n; j++) {
            if (s[i - 1] == t[j - 1]) {
                curr[j] = prev[j - 1];
            } else {
                curr[j] = 1 + std::min({prev[j], curr[j - 1], prev[j - 1]});
            }
        }
        std::swap(prev, curr);
    }
    return prev[n];
}
```

## 4. Задача о рюкзаке (0/1 Knapsack)

### Постановка

Дано $n$ предметов. Предмет $i$ имеет вес $w_i$ и ценность $v_i$. Рюкзак вмещает суммарный вес не более $W$. Найти набор предметов с максимальной суммарной ценностью при условии, что каждый предмет берётся не более одного раза.

### Рекуррентность

Определим $\text{dp}[i][w]$ — максимальная ценность, достижимая при использовании первых $i$ предметов и бюджете веса $w$.

**База:** $\text{dp}[0][w] = 0$ для всех $w$ (нет предметов — нет ценности).

**Рекуррентность:**

$$
\text{dp}[i][w] = \begin{cases}
\text{dp}[i{-}1][w] & \text{если } w_i > w \text{ (предмет не помещается)} \\
\max\!\left(\text{dp}[i{-}1][w],\;\text{dp}[i{-}1][w - w_i] + v_i\right) & \text{иначе}
\end{cases}
$$

Два варианта: **не брать** предмет $i$ или **взять** его (тогда используем оставшийся бюджет $w - w_i$).

### Пример: 3 предмета, W = 5

| i | $w_i$ | $v_i$ |
|---|--------|--------|
| 1 |   2    |   3    |
| 2 |   3    |   4    |
| 3 |   4    |   5    |

Таблица $\text{dp}[i][w]$ (строки = предметы 0..3, столбцы = вес 0..5):

|       | w=0 | w=1 | w=2 | w=3 | w=4 | w=5 |
|-------|-----|-----|-----|-----|-----|-----|
| i=0   |  0  |  0  |  0  |  0  |  0  |  0  |
| i=1   |  0  |  0  |  3  |  3  |  3  |  3  |
| i=2   |  0  |  0  |  3  |  4  |  4  |  7  |
| i=3   |  0  |  0  |  3  |  4  |  5  |  7  |

Ответ: $\text{dp}[3][5] = 7$ — взять предметы 1 и 2 (вес $2 + 3 = 5$, ценность $3 + 4 = 7$).

### Реализация: O(nW) время и O(nW) память

```cpp
#include <vector>
#include <algorithm>
#include <iostream>

int knapsack(const std::vector<int>& w, const std::vector<int>& v, int W) {
    int n = w.size();
    // dp[i][cap] = max value using items 0..i-1 with weight budget cap
    std::vector<std::vector<int>> dp(n + 1, std::vector<int>(W + 1, 0));

    for (int i = 1; i <= n; i++) {
        for (int cap = 0; cap <= W; cap++) {
            dp[i][cap] = dp[i - 1][cap]; // не берём предмет i
            if (w[i - 1] <= cap) {
                dp[i][cap] = std::max(dp[i][cap],
                                      dp[i - 1][cap - w[i - 1]] + v[i - 1]);
            }
        }
    }
    return dp[n][W];
}

int main() {
    std::vector<int> weights = {2, 3, 4};
    std::vector<int> values  = {3, 4, 5};
    int W = 5;
    std::cout << knapsack(weights, values, W) << "\n"; // 7
    return 0;
}
```

### Оптимизация памяти: O(W) — скользящий массив

Важно: итерировать по $w$ нужно **от W до $w_i$**, иначе один предмет может быть взят дважды.

```cpp
int knapsack_1d(const std::vector<int>& w, const std::vector<int>& v, int W) {
    int n = w.size();
    std::vector<int> dp(W + 1, 0);

    for (int i = 0; i < n; i++) {
        // КРИТИЧНО: итерируем справа налево!
        for (int cap = W; cap >= w[i]; cap--) {
            dp[cap] = std::max(dp[cap], dp[cap - w[i]] + v[i]);
        }
    }
    return dp[W];
}
```

## 5. Восстановление ответа (Traceback)

Вычислить оптимальное значение — половина задачи. Часто нужно восстановить **сам ответ**: последовательность операций или набор предметов.

### Traceback для редакционного расстояния

Идём из ячейки $\text{dp}[m][n]$ обратно к $\text{dp}[0][0]$, на каждом шаге определяя, откуда пришли:

```cpp
#include <vector>
#include <string>
#include <algorithm>
#include <iostream>

void print_edit_ops(const std::string& s, const std::string& t) {
    int m = s.size(), n = t.size();
    std::vector<std::vector<int>> dp(m + 1, std::vector<int>(n + 1));
    for (int i = 0; i <= m; i++) dp[i][0] = i;
    for (int j = 0; j <= n; j++) dp[0][j] = j;
    for (int i = 1; i <= m; i++)
        for (int j = 1; j <= n; j++) {
            if (s[i-1] == t[j-1]) dp[i][j] = dp[i-1][j-1];
            else dp[i][j] = 1 + std::min({dp[i-1][j], dp[i][j-1], dp[i-1][j-1]});
        }

    // Traceback
    int i = m, j = n;
    std::string ops;
    while (i > 0 || j > 0) {
        if (i > 0 && j > 0 && s[i-1] == t[j-1]) {
            ops += 'M';  // Match
            i--; j--;
        } else if (i > 0 && j > 0 && dp[i][j] == dp[i-1][j-1] + 1) {
            ops += 'R';  // Replace
            i--; j--;
        } else if (i > 0 && dp[i][j] == dp[i-1][j] + 1) {
            ops += 'D';  // Delete
            i--;
        } else {
            ops += 'I';  // Insert
            j--;
        }
    }
    std::reverse(ops.begin(), ops.end());
    std::cout << ops << "\n";
}
```

### Traceback для рюкзака

```cpp
std::vector<int> knapsack_items(const std::vector<int>& w,
                                const std::vector<int>& v, int W) {
    int n = w.size();
    std::vector<std::vector<int>> dp(n + 1, std::vector<int>(W + 1, 0));
    for (int i = 1; i <= n; i++)
        for (int cap = 0; cap <= W; cap++) {
            dp[i][cap] = dp[i-1][cap];
            if (w[i-1] <= cap)
                dp[i][cap] = std::max(dp[i][cap], dp[i-1][cap-w[i-1]] + v[i-1]);
        }

    // Восстановление: идём от dp[n][W] к dp[0][0]
    std::vector<int> chosen;
    int cap = W;
    for (int i = n; i >= 1; i--) {
        if (dp[i][cap] != dp[i-1][cap]) {
            chosen.push_back(i - 1); // предмет с индексом i-1 взят
            cap -= w[i - 1];
        }
    }
    return chosen;
}
```

### Общий паттерн traceback

1. Вычислить таблицу ДП целиком (или сохранить указатели выбора).
2. Начать с ячейки финального ответа.
3. На каждом шаге определить, из какой соседней ячейки пришло значение.
4. Двигаться в эту ячейку и записать сделанный выбор.
5. Остановиться при достижении базового случая.

## 6. Признаки ДП-задач

Задача, скорее всего, решается ДП, если в ней присутствует один из следующих признаков:

- **«Минимальный/максимальный»** — «найти кратчайший путь», «минимум операций», «максимальная ценность».
- **«Посчитать количество способов»** — «сколькими способами можно достичь...».
- **«Возможно ли»** — «можно ли набрать сумму W», «существует ли разбиение».
- **Принцип Беллмана** — состояние должно содержать достаточно информации для принятия следующего решения без оглядки на историю.

### Дополнительный пример: наибольшая общая подпоследовательность (LCS)

Определим $\text{lcs}[i][j]$ — длина LCS строк $s[0..i{-}1]$ и $t[0..j{-}1]$.

$$
\text{lcs}[i][j] = \begin{cases}
\text{lcs}[i{-}1][j{-}1] + 1 & \text{если } s[i{-}1] = t[j{-}1] \\
\max(\text{lcs}[i{-}1][j],\;\text{lcs}[i][j{-}1]) & \text{иначе}
\end{cases}
$$

База: $\text{lcs}[0][j] = \text{lcs}[i][0] = 0$.

Это та же структура, что и у Edit Distance, — таблица $O(mn)$ с рекуррентностью, зависящей от трёх соседних ячеек.

```cpp
int lcs(const std::string& s, const std::string& t) {
    int m = s.size(), n = t.size();
    std::vector<std::vector<int>> dp(m + 1, std::vector<int>(n + 1, 0));
    for (int i = 1; i <= m; i++)
        for (int j = 1; j <= n; j++) {
            if (s[i-1] == t[j-1]) dp[i][j] = dp[i-1][j-1] + 1;
            else                   dp[i][j] = std::max(dp[i-1][j], dp[i][j-1]);
        }
    return dp[m][n];
}
```

## 7. Типичные ошибки

**Ошибка 1. Неверный порядок заполнения таблицы.**
Каждая ячейка должна заполняться после всех ячеек, от которых она зависит. Если $\text{dp}[i][j]$ зависит от $\text{dp}[i{-}1][j]$, строки нужно заполнять сверху вниз, не наоборот. Перепутанный порядок даёт неверные (или неинициализированные) значения.

**Ошибка 2. Направление обхода при оптимизации памяти (рюкзак).**
При использовании одномерного массива в задаче о рюкзаке нужно итерировать $w$ от $W$ до $w_i$. Если итерировать слева направо, предмет $i$ может быть взят несколько раз — задача превращается в unbounded knapsack.

**Ошибка 3. Индексация "с единицы" vs "с нуля".**
Таблица $\text{dp}[i][j]$ обычно индексируется с 1, а строки $s$ и $t$ — с 0. Смешение этих соглашений даёт выход за границу массива или неверные базовые случаи. Всегда проверяйте: $s[i{-}1]$, а не $s[i]$.

**Ошибка 4. Забытые базовые случаи.**
Для Edit Distance строка $\text{dp}[i][0] = i$ и $\text{dp}[0][j] = j$ критична. Без неё все значения первого столбца/строки будут неправильными, и ошибка каскадно распространится по всей таблице.

**Ошибка 5. Неверное восстановление ответа при оптимизации памяти.**
Если вы оптимизировали память до $O(W)$ (одномерный массив), восстановить набор предметов по нему нельзя — нужна полная таблица $O(nW)$ или явные указатели выбора. Не выбрасывайте таблицу до завершения traceback.

**Ошибка 6. Переполнение при инициализации "бесконечности".**
Если $\text{dp}[i][j]$ инициализируется как $10^9$ и к нему прибавляется 1, получается переполнение `int`. Используйте `INT_MAX/2` или `1e9` как безопасный sentinel.

- Чётко понимать **порядок заполнения** таблицы для конкретной задачи.

## 9. Итог

Динамическое программирование — это систематический способ устранить экспоненциальный взрыв рекурсии, кэшируя результаты подзадач. Ключ к методу — два свойства: оптимальная подструктура и перекрывающиеся подзадачи. Top-down реализация сохраняет интуицию рекурсии, bottom-up — позволяет тонко управлять памятью. Редакционное расстояние и задача о рюкзаке — два канонических примера, покрывающие большинство приёмов ДП: двумерные таблицы, скользящие массивы, traceback. Выучив эти два примера досконально, вы получаете шаблон для решения сотен задач.
