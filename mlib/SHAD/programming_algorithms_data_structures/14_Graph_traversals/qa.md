## Задача 1. BFS-расстояния вручную

Дан неориентированный граф с 6 вершинами (0–5) и рёбрами: 0-1, 0-2, 1-3, 2-3, 3-4, 4-5. Запустите BFS от вершины 0. Восстановите массив `dist[]` и порядок обработки вершин.

### Решение

Строим список смежности:
```
adj[0] = {1, 2}
adj[1] = {0, 3}
adj[2] = {0, 3}
adj[3] = {1, 2, 4}
adj[4] = {3, 5}
adj[5] = {4}
```

Запускаем BFS:

| Шаг | Очередь (начало) | Обрабатываем | Открываем (dist) |
|-----|-----------------|-------------|-----------------|
| 1 | [0] | 0 | 1 (d=1), 2 (d=1) |
| 2 | [1, 2] | 1 | 3 (d=2) |
| 3 | [2, 3] | 2 | — (3 уже открыта) |
| 4 | [3] | 3 | 4 (d=3) |
| 5 | [4] | 4 | 5 (d=4) |
| 6 | [5] | 5 | — |

Результат: `dist = [0, 1, 1, 2, 3, 4]`.

Порядок обработки: 0, 1, 2, 3, 4, 5.

### Ответ

`dist = [0, 1, 1, 2, 3, 4]`. Кратчайший путь от 0 до 5 имеет длину 4: 0→2→3→4→5 (или 0→1→3→4→5).

## Задача 2. Компоненты связности

Дан неориентированный граф с 8 вершинами (0–7) и рёбрами: 0-1, 0-2, 3-4, 5-6, 5-7. Вершина 6 изолирована... Нет: рёбра именно такие. Найдите все компоненты связности и их размеры.

### Решение

Список смежности:
```
adj[0] = {1, 2}
adj[1] = {0}
adj[2] = {0}
adj[3] = {4}
adj[4] = {3}
adj[5] = {6, 7}
adj[6] = {5}
adj[7] = {5}
```

Запускаем обход (BFS/DFS) от каждой непосещённой вершины:

- **Старт с 0:** BFS открывает {0, 1, 2}. Компонента 1, размер 3.
- **Старт с 3:** BFS открывает {3, 4}. Компонента 2, размер 2.
- **Старт с 5:** BFS открывает {5, 6, 7}. Компонента 3, размер 3.

Все вершины 0–7 посещены.

```cpp
int count_components(const vector<vector<int>>& adj) {
    int n = adj.size();
    vector<bool> vis(n, false);
    int cnt = 0;
    for (int s = 0; s < n; ++s) {
        if (!vis[s]) {
            ++cnt;
            queue<int> q;
            vis[s] = true;
            q.push(s);
            while (!q.empty()) {
                int u = q.front(); q.pop();
                for (int v : adj[u])
                    if (!vis[v]) { vis[v] = true; q.push(v); }
            }
        }
    }
    return cnt;
}
```

### Ответ

3 компоненты: {0,1,2}, {3,4}, {5,6,7}.

## Задача 3. Проверка двудольности

Проверьте, является ли граф с вершинами {0,1,2,3,4} и рёбрами 0-1, 1-2, 2-3, 3-4, 4-0 двудольным. Обоснуйте с помощью BFS-2-раскраски.

### Решение

Граф — пятицикл $C_5$ (нечётный цикл). Запустим BFS-2-раскраску от вершины 0:

| Вершина | color |
|---------|-------|
| 0 (старт) | 0 |
| 1 (сосед 0) | 1 |
| 4 (сосед 0) | 1 |
| 2 (сосед 1) | 0 |
| 3 (сосед 4 и 2) | ? |

При обработке вершины 4 открываем вершину 3 → color[3] = 0.
При обработке вершины 2 пытаемся покрасить вершину 3 → color[3] = 1.

Конфликт: color[3] = 0 (от вершины 4), но сосед 2 требует color[3] = 1.

Альтернативно: в пятицикле длина нечётная (5), значит нечётный цикл → граф не двудольный по критерию Кёнига.

```cpp
bool is_bipartite(const vector<vector<int>>& adj) {
    int n = adj.size();
    vector<int> color(n, -1);
    for (int s = 0; s < n; ++s) {
        if (color[s] != -1) continue;
        queue<int> q;
        color[s] = 0;
        q.push(s);
        while (!q.empty()) {
            int u = q.front(); q.pop();
            for (int v : adj[u]) {
                if (color[v] == -1) { color[v] = 1 - color[u]; q.push(v); }
                else if (color[v] == color[u]) return false;
            }
        }
    }
    return true;
}
```

### Ответ

Граф **не двудольный**: содержит нечётный цикл длины 5.

## Задача 4. Метки времени DFS

Дан граф с 5 вершинами (0–4) и рёбрами: 0-1, 0-4, 1-2, 1-3, 2-3. Запустите рекурсивный DFS от вершины 0 (соседей обходить в порядке возрастания номеров). Найдите `tin[]` и `tout[]` для всех вершин (счётчик начинается с 0 и инкрементируется при каждом событии).

```
adj[0] = {1, 4}

adj[2] = {1, 3}
adj[3] = {1, 2}
adj[4] = {0}

Трассируем DFS (visited — false для всех):

```
enter 0: tin[0]=0

    enter 2: tin[2]=2
      сосед 1: посещён (пропуск)
      enter 3: tin[3]=3

exit 0: tout[0]=9

| Вершина | tin | tout |
|---------|-----|------|
| 0 | 0 | 9 |
| 1 | 1 | 6 |

| 3 | 3 | 4 |
| 4 | 7 | 8 |

## Задача 5. Обнаружение цикла в орграфе

Дан ориентированный граф с 5 вершинами (0–4) и рёбрами: 0→1, 1→2, 2→3, 3→4, 4→2. Есть ли в нём цикл? Если да — найдите его. Реализуйте алгоритм через трёхцветный DFS.

### Решение

Трёхцветный DFS: 0=белый (не посещён), 1=серый (в стеке), 2=чёрный (завершён).

```
DFS(0): color[0]=1
  DFS(1): color[1]=1
    DFS(2): color[2]=1
      DFS(3): color[3]=1
        DFS(4): color[4]=1
          ребро 4→2: color[2]=1 (серый!) → ЦИКЛ!
```

Цикл: 2 → 3 → 4 → 2.

```cpp
bool has_cycle_directed(const vector<vector<int>>& adj, int u, vector<int>& color) {
    color[u] = 1;
    for (int v : adj[u]) {
        if (color[v] == 1) return true;   // back edge
        if (color[v] == 0 && has_cycle_directed(adj, v, color)) return true;
    }
    color[u] = 2;
    return false;
}
```

### Ответ

Цикл существует: **2 → 3 → 4 → 2**. Ребро 4→2 является обратным (back edge).

## Задача 6. Восстановление кратчайшего пути

В неориентированном невзвешенном графе найдите кратчайший путь от вершины 0 до вершины 7. Граф задан рёбрами: 0-1, 0-3, 1-2, 1-4, 3-4, 4-5, 2-6, 5-6, 6-7.

### Решение

Запускаем BFS от 0 с массивом `parent[]`:

```
Слой 0: {0}, dist[0]=0, parent[0]=-1
Слой 1: {1,3}, dist[1]=dist[3]=1, parent[1]=0, parent[3]=0
Слой 2: {2,4}, dist[2]=2 (от 1), dist[4]=2 (от 1 или 3 — первый встреченный)
        parent[2]=1, parent[4]=1
Слой 3: {6,5}, dist[6]=3 (от 2), dist[5]=3 (от 4)
        parent[6]=2, parent[5]=4
Слой 4: {7}, dist[7]=4 (от 6)
        parent[7]=6
```

Восстанавливаем путь от 7 обратно по `parent[]`:
7 ← 6 ← 2 ← 1 ← 0.

Путь: **0 → 1 → 2 → 6 → 7**, длина 4.

Проверим другие возможные пути:
- 0→3→4→5→6→7: длина 5. Длиннее.
- 0→1→4→5→6→7: длина 5. Длиннее.

### Ответ

Кратчайший путь: **0 → 1 → 2 → 6 → 7**, длина 4.

## Задача 7. Топологическая сортировка через DFS

Дан DAG с 6 вершинами (0–5) и рёбрами: 5→2, 5→0, 4→0, 4→1, 2→3, 3→1. Найдите топологический порядок через DFS.

### Решение

Запускаем DFS. Добавляем вершину в стек при **выходе** (tout). Потом разворачиваем стек.

Начинаем с непосещённых вершин в порядке 0,1,2,...:

```
DFS(0): visited[0], нет исходящих → push(0). Стек: [0]
DFS(1): visited[1], нет исходящих → push(1). Стек: [0,1]
DFS(2): visited[2], сосед 3:
  DFS(3): visited[3], сосед 1: уже посещён. → push(3). Стек: [0,1,3]
  → push(2). Стек: [0,1,3,2]
DFS(3): уже посещена.
DFS(4): visited[4], сосед 0: посещён, сосед 1: посещён. → push(4). Стек: [0,1,3,2,4]
DFS(5): visited[5], сосед 2: посещён, сосед 0: посещён. → push(5). Стек: [0,1,3,2,4,5]
```

Разворачиваем: **5, 4, 2, 3, 1, 0**.

Проверка: каждое ребро идёт слева направо:
- 5→2: 5 до 2 ✓
- 5→0: 5 до 0 ✓
- 4→0: 4 до 0 ✓
- 4→1: 4 до 1 ✓
- 2→3: 2 до 3 ✓
- 3→1: 3 до 1 ✓

### Ответ

Топологический порядок: **5, 4, 2, 3, 1, 0** (один из корректных ответов).

## Задача 8. Классификация рёбер орграфа

Дан орграф с вершинами {0,1,2,3} и рёбрами: 0→1, 0→2, 1→3, 2→1, 3→2. После DFS от вершины 0 получены метки: tin=[0,1,2,7], tout=[9,6,8,3] (нумерация 0..3). Классифицируйте каждое ребро.

### Решение

Для каждого ребра $(u,v)$ проверяем состояние $v$ в момент обработки ребра и соотношение меток:

**Ребро 0→1:** tin[0]=0, tin[1]=1, tout[1]=6, tout[0]=9.
- $[1,6] \subset [0,9]$: 1 — потомок 0. Ребро использовалось для открытия 1. → **Древесное**.

**Ребро 0→2:** tin[0]=0, tin[2]=2, tout[2]=8, tout[0]=9.
- $[2,8] \subset [0,9]$: 2 — потомок 0. → **Древесное** (или прямое, если 2 открыта позже через другой путь).

Уточнение: DFS(0)→DFS(1)→DFS(3)→ребро 3→2 (tin[2]=?). Судя по tout[3]=3, DFS(3) завершился раньше чем открылась 2. Значит 2 открылась при DFS(0) после DFS(1). → ребро 0→2 **древесное**.

**Ребро 1→3:** tin[1]=1, tin[3]=2, tout[3]=3, tout[1]=6.
- $[2,3] \subset [1,6]$: 3 — потомок 1. → **Древесное**.

**Ребро 2→1:** tin[2]=7 (по условию... ждём). Но tin[1]=1, tout[1]=6, tin[2]=7. Значит tout[1]=6 < tin[2]=7: интервалы не пересекаются, 1 завершена раньше 2. Ребро 2→1: v=1 завершена (черная). tin[1] < tin[2] → 1 не потомок 2. → **Перекрёстное**.

**Ребро 3→2:** tin[3]=2, tout[3]=3. Но tin[2] нужно уточнить. По условию tin=[0,1,2,7]: вершина 2 имеет tin[2]=7? Но тогда DFS(3) входит при tin=2, выходит при tout=3, а вершина 2 ещё не посещена. При DFS(3) ребро 3→2: v=2 не посещена → **Древесное** (3→2 древесное, 2 открывается из 3). Тогда tin[2] должно быть между 2 и 3. Противоречие с условием tin[2]=2.

Переклассифицируем с tin=[0,1,1,2] — нет, используем данные условия буквально:

При tin=[0,1,2,7], tout=[9,6,8,3] (индексы 0..3):
- Вершина 0: [0,9]; вершина 1: [1,6]; вершина 2: [7,8] (tin[2]=7, tout[2]=8); вершина 3: [2,3].

**0→1:** 1∈[0,9], 1 открыта как потомок 0. → **Древесное**.
**0→2:** tin[2]=7 > tout[1]=6. Вершина 2 открыта после завершения поддерева 1. → **Прямое** (forward edge, 2 — потомок 0 в другой ветке, но уже закрыт? tout[2]=8 < tout[0]=9 → 2 ещё потомок 0). → **Прямое**.
**1→3:** 3∈[1,6] → **Древесное**.
**2→1:** 1 закрыта (tout[1]=6 < tin[2]=7) и tin[1]=1 < tin[2]=7. → **Перекрёстное**.
**3→2:** tin[3]=2, at обработке ребра 3→2: tin[2]=7 > tout[3]=3, то есть 2 ещё не открыта? Но tout[3]=3 и tin[2]=7 — значит 2 открылась после 3 закрылась. Ребро 3→2 ведёт в ещё не посещённую вершину → **Древесное**. Но тогда tin[2] должно быть 3, а не 7. Скорее всего DFS: 0→1→3→(3 закрыта)→(1 закрыта)→0→2. Тогда tin[2]=7, tout[2]=8 корректны, а ребро 3→2: v=2 ещё не посещена при tin[3]=2. Но DFS(3) должен был бы уйти в 2, установить tin[2]=3, tout[2]=4... Противоречие: условие задачи содержит искусственные метки. Примем, что порядок DFS: 0→1→3→закрыт→закрыт→2→закрыт→закрыт. Ребро 3→2 при обработке указывает на уже открытую 2? Нет. Ребро ведёт к 2, которая не посещена в момент выхода из 3 (tout[3]=3), а tin[2]=7 > tout[3]=3. Значит при DFS(3) ребро 3→2 обработано, но 2 была уже посещена и закрыта (из другой ветки)... Это невозможно при tin[2]=7 > tout[3]=3. Примем ребро 3→2 как **перекрёстное** (cross edge): v=2 закрыта, tin[2]>tin[3]).

### Ответ

| Ребро | Тип |
|-------|-----|
| 0→1 | Древесное |
| 0→2 | Прямое (forward) |
| 1→3 | Древесное |
| 2→1 | Перекрёстное (cross) |
| 3→2 | Перекрёстное (cross) |

## Задача 9. Кратчайший путь в сетке (BFS на неявном графе)

Дана сетка $N \times M$, клетки: '.' — пустые, '#' — стены. Найдите минимальное количество шагов от клетки (0,0) до клетки (N-1,M-1). Шаги — в 4 направлениях (горизонталь/вертикаль).

```
. . . #
# # . #
. . . .
. # # .
. . . .
```
(N=5, M=4, старт=(0,0), финиш=(4,3))

### Решение

Моделируем граф: вершины = свободные клетки, рёбра = 4-соседство. BFS от (0,0):

```cpp
#include <bits/stdc++.h>
using namespace std;

int bfs_grid(vector<string>& grid) {
    int n = grid.size(), m = grid[0].size();
    vector<vector<int>> dist(n, vector<int>(m, -1));
    int dx[] = {0,0,1,-1};
    int dy[] = {1,-1,0,0};

    queue<pair<int,int>> q;
    dist[0][0] = 0;
    q.push({0, 0});

    while (!q.empty()) {
        auto [x, y] = q.front(); q.pop();
        for (int d = 0; d < 4; ++d) {
            int nx = x + dx[d], ny = y + dy[d];
            if (nx >= 0 && nx < n && ny >= 0 && ny < m
                && grid[nx][ny] == '.' && dist[nx][ny] == -1) {
                dist[nx][ny] = dist[x][y] + 1;
                q.push({nx, ny});
            }
        }
    }
    return dist[n-1][m-1];
}
```

Трассируем BFS на сетке (координаты (строка, столбец)):

- Слой 0: (0,0)
- Слой 1: (0,1)
- Слой 2: (0,2)
- Слой 3: (1,2)
- Слой 4: (2,2), (2,1), (2,0)
- Слой 5: (2,3), (3,0)
- Слой 6: (4,3)←(2,3)? Нет: (3,3) с dist=6 от (2,3), (4,0) от (3,0)
- Дальше: (4,3) достигается через (3,3) слой 7.

Итого кратчайший путь: длина **7**.

### Ответ

Минимальное количество шагов: **7**. Путь: (0,0)→(0,1)→(0,2)→(1,2)→(2,2)→(2,3)→(3,3)→(4,3).

## Задача 10. Синтез: двудольность + BFS-расстояния + DFS-цикл

Дан неориентированный граф с $n$ вершинами и $m$ рёбрами. Ответьте на три вопроса:
1. Является ли граф двудольным?
2. Каково минимальное расстояние от вершины 0 до вершины $n-1$?
3. Есть ли в графе цикл нечётной длины?

**Вход:** n=6, m=7, рёбра: 0-1, 0-2, 1-3, 2-3, 3-4, 4-5, 1-5.

**Важно:** вопрос 3 решите без отдельного алгоритма — используйте ответ на вопрос 1.

### Решение

**Шаг 1. BFS-2-раскраска (отвечает на вопросы 1 и 3).**

```
color[0]=0
BFS: 0 → {1(c=1), 2(c=1)}
     1 → {0(c=0, ok), 3(c=0), 5(c=0)}
     2 → {0(c=0, ok), 3(c=0, но уже color[3]=0: ок)}
     3 → {1(c=1,ok), 2(c=1,ok), 4(c=1)}
     5 → {4(c=1,ok уже?), 1(c=1,ok)}
```

При обработке вершины 1 открываем 5 с color[5]=0.
При обработке вершины 3 открываем 4 с color[4]=1.
При обработке вершины 5 (color=0): сосед 4 имеет color[4]=1 ≠ color[5]=0 → ok.
Сосед 1: color[1]=1 ≠ color[5]=0 → ok.

Теперь ребро 4-5: color[4]=1, color[5]=0 → соседи разных цветов → ok.

Все рёбра между разными цветами. **Граф двудольный**.

Части: $L = \{0, 3\}$... Подождите: color[0]=0, color[1]=1, color[2]=1, color[3]=0, color[4]=1, color[5]=0.
$L = \{0, 3, 5\}$, $R = \{1, 2, 4\}$.

Проверка ребра 2-3: 2∈R, 3∈L → ✓. Ребра 1-5: 1∈R, 5∈L → ✓.

Граф двудольный → **нечётных циклов нет**.

**Шаг 2. BFS от вершины 0 для нахождения dist[5].**

```
dist[0]=0, открываем {1,2}: dist[1]=dist[2]=1
От 1: {3,5}: dist[3]=dist[5]=2
От 2: {3} уже открыта
От 3: {4}: dist[4]=3
От 5: {4} уже открыта
```

Итог: dist[5]=2. Кратчайший путь: 0→1→5, длина 2.

**Итоговые ответы:**

1. Граф **двудольный** ($L=\{0,3,5\}$, $R=\{1,2,4\}$).
2. Расстояние от 0 до 5 (=n-1=5): **2** (путь 0→1→5).
3. Нечётных циклов **нет** (следует из двудольности).

```cpp
// Единый проход: BFS-2-раскраска + dist[]
#include <bits/stdc++.h>
using namespace std;

pair<bool, vector<int>> solve(const vector<vector<int>>& adj, int src) {
    int n = adj.size();
    vector<int> dist(n, -1);
    vector<int> color(n, -1);
    bool bipartite = true;
    queue<int> q;
    dist[src] = 0;
    color[src] = 0;
    q.push(src);
    while (!q.empty()) {
        int u = q.front(); q.pop();
        for (int v : adj[u]) {
            if (dist[v] == -1) {
                dist[v] = dist[u] + 1;
                color[v] = 1 - color[u];
                q.push(v);
            } else if (color[v] == color[u]) {
                bipartite = false;
            }
        }
    }
    return {bipartite, dist};
}
```

### Ответ

1. Граф двудольный: $L=\{0,3,5\}$, $R=\{1,2,4\}$.
2. Расстояние 0→5 равно **2** (путь: 0→1→5).
3. Нечётных циклов нет (граф двудольный).
