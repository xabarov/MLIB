## Задача 1. Адрес элемента массива

Массив `int a[100]` размещён в памяти, начиная с адреса `0x1000`. Размер `int` равен 4 байтам. Чему равен адрес элемента `a[17]`?

### Решение

Формула адреса элемента с индексом $i$:

$$
\text{addr}(a[i]) = \text{base} + i \cdot \text{sizeof}(T)
$$

Подставляем: $\text{base} = 0x1000$, $i = 17$, $\text{sizeof}(\texttt{int}) = 4$.

$$
\text{addr}(a[17]) = 0x1000 + 17 \cdot 4 = 0x1000 + 68 = 0x1044
$$

Перевод $68$ в шестнадцатеричное: $68 = 4 \cdot 16 + 4 = 0x44$.

Значит, $0x1000 + 0x44 = 0x1044$.

### Ответ

$0x1044$

## Задача 2. Трассировка стека

Выполните трассировку следующей последовательности операций со стеком и укажите содержимое стека после каждой операции:

```
push(3), push(7), push(1), pop(), push(4), top(), pop(), pop()
```

### Решение

Стек изображаем снизу вверх (вершина справа):

| Операция | Стек (снизу→вершина) | Возвращает |
|---|---|---|
| `push(3)` | [3] | — |
| `push(7)` | [3, 7] | — |
| `push(1)` | [3, 7, 1] | — |
| `pop()` | [3, 7] | 1 |
| `push(4)` | [3, 7, 4] | — |
| `top()` | [3, 7, 4] | 4 |
| `pop()` | [3, 7] | 4 |
| `pop()` | [3] | 7 |

### Ответ

После всех операций в стеке остаётся один элемент: `[3]`.

## Задача 3. Вычисление выражения в RPN

Вычислите выражение в обратной польской нотации (RPN): `5 1 2 + 4 * + 3 -`.

### Решение

Алгоритм: числа кладём в стек; оператор снимает два числа (первое снятое — правый операнд, второе — левый), выполняет операцию, кладёт результат.

| Токен | Действие | Стек |
|---|---|---|
| `5` | push(5) | [5] |
| `1` | push(1) | [5, 1] |
| `2` | push(2) | [5, 1, 2] |
| `+` | pop 2, pop 1; push(1+2=3) | [5, 3] |
| `4` | push(4) | [5, 3, 4] |
| `*` | pop 4, pop 3; push(3*4=12) | [5, 12] |
| `+` | pop 12, pop 5; push(5+12=17) | [17] |
| `3` | push(3) | [17, 3] |
| `-` | pop 3, pop 17; push(17-3=14) | [14] |

Исходное выражение соответствует: $5 + (1+2) \times 4 - 3 = 5 + 12 - 3 = 14$.

### Ответ

$14$

## Задача 4. Амортизированный анализ push_back

Докажите, что после $n$ операций `push_back` на изначально пустом `std::vector` (с capacity 1) суммарное число скопированных элементов не превышает $2n$.

### Решение

При удвоении capacity копирование происходит, когда `size == capacity`. Пусть `capacity` росла как $1, 2, 4, 8, \ldots, 2^k$, где $2^k \leq n < 2^{k+1}$.

Суммарное число скопированных элементов при каждом удвоении:

$$
1 + 2 + 4 + \cdots + 2^{k-1} + 2^k
$$

Это геометрическая прогрессия:

$$
S = \sum_{j=0}^{k} 2^j = 2^{k+1} - 1
$$

Поскольку $2^k \leq n$, получаем $2^{k+1} \leq 2n$, значит:

$$
S = 2^{k+1} - 1 \leq 2n - 1 < 2n
$$

Итого суммарное число копирований за $n$ операций: $O(n)$. Амортизированная стоимость одной операции:

$$
\frac{O(n)}{n} = O(1)
$$

### Ответ

Суммарное число копирований $\leq 2n - 1$, поэтому амортизированная стоимость одного `push_back` — $O(1)$.

## Задача 5. Трассировка кольцевого буфера

Кольцевой буфер размером $M = 5$. Начальное состояние: `head = 0`, `tail = 0`, `size = 0`. Выполните операции: `push(A)`, `push(B)`, `push(C)`, `pop()`, `push(D)`, `push(E)`, `pop()`, `push(F)`. Укажите состояние массива и значения `head`, `tail`, `size` после каждой операции.

### Решение

| Операция | Массив [0..4] | head | tail | size |
|---|---|---|---|---|
| Начало | [_, _, _, _, _] | 0 | 0 | 0 |
| `push(A)` | [A, _, _, _, _] | 0 | 1 | 1 |
| `push(B)` | [A, B, _, _, _] | 0 | 2 | 2 |
| `push(C)` | [A, B, C, _, _] | 0 | 3 | 3 |
| `pop()` → A | [A, B, C, _, _] | 1 | 3 | 2 |
| `push(D)` | [A, B, C, D, _] | 1 | 4 | 3 |
| `push(E)` | [A, B, C, D, E] | 1 | 0 | 4 |
| `pop()` → B | [A, B, C, D, E] | 2 | 0 | 3 |
| `push(F)` | [F, B, C, D, E] | 2 | 1 | 4 |

Заметим: при `push(E)` `tail = (4+1) % 5 = 0` — буфер «обернулся». При `push(F)` данные записались в слот 0 (поверх устаревшего A).

### Ответ

После всех операций: массив `[F, B, C, D, E]`, `head = 2`, `tail = 1`, `size = 4`. В очереди элементы: B, C, D, F (в порядке FIFO, начиная с `head = 2`).

## Задача 6. Скобочная последовательность

Является ли последовательность `{[()][]}` правильной скобочной последовательностью? Трассируйте алгоритм на основе стека.

### Решение

Алгоритм: открывающие скобки кладём в стек; для закрывающей проверяем соответствие вершине.

| Символ | Действие | Стек |
|---|---|---|
| `{` | push | [{] |
| `[` | push | [{, [] |
| `(` | push | [{, [, (] |
| `)` | top = `(`, совпадает — pop | [{, [] |
| `]` | top = `[`, совпадает — pop | [{] |
| `[` | push | [{, [] |
| `]` | top = `[`, совпадает — pop | [{] |
| `}` | top = `{`, совпадает — pop | [] |

Конец строки, стек пуст → последовательность правильная.

### Ответ

Да, `{[()][]}` — правильная скобочная последовательность.

## Задача 7. Минимум на скользящем окне

Дан массив $a = [5, 3, 1, 4, 2, 6, 3]$ и размер окна $k = 3$. Найдите минимум в каждом окне за $O(n)$ с помощью монотонного дека. Покажите состояние дека на каждом шаге.

### Решение

Дек хранит **индексы** в порядке возрастания значений (монотонный возрастающий по значениям). Для каждой позиции $i$:
1. Удалить из front индексы $< i - k + 1$ (вышли за окно).
2. Удалить из back индексы с $a[\text{back}] \geq a[i]$ (не могут быть минимумом при наличии $a[i]$).
3. Push_back(i).
4. Если $i \geq k - 1$: ответ для окна — $a[\text{front}]$.

| $i$ | $a[i]$ | Удалить из back | Дек (индексы) | Дек (значения) | Минимум окна |
|---|---|---|---|---|---|
| 0 | 5 | — | [0] | [5] | — |
| 1 | 3 | pop 0 (5≥3) | [1] | [3] | — |
| 2 | 1 | pop 1 (3≥1) | [2] | [1] | 1 |
| 3 | 4 | — | [2, 3] | [1, 4] | 1 |
| 4 | 2 | pop 3 (4≥2) | [2, 4] | [1, 2] | 1 |
| 5 | 6 | — | [2, 4, 5] | [1, 2, 6] | front=2: уходит (2<5-3+1=3), pop front → [4, 5], ответ = $a[4]=2$ |
| 6 | 3 | pop 5 (6≥3) | [4, 6] | [2, 3] | $a[4]=2$ |

Минимумы окон: $1, 1, 1, 2, 2$.

Проверка: окна $[5,3,1]=1$, $[3,1,4]=1$, $[1,4,2]=1$, $[4,2,6]=2$, $[2,6,3]=2$. Верно.

Сложность: каждый индекс добавляется в дек ровно один раз и удаляется ровно один раз → $O(n)$.

### Ответ

Минимумы: $[1, 1, 1, 2, 2]$.

## Задача 8. Удаление узла из односвязного списка за O(1) без предыдущего

Дан указатель на узел $v$ односвязного списка (не на последний элемент). Удалите $v$ из списка за $O(1)$, не имея указателя на предыдущий узел.

### Решение

Классическое удаление требует предыдущего узла: `prev->next = v->next`. Без него — $O(n)$.

**Трюк**: скопировать значение следующего узла в $v$, затем удалить следующий узел.

```cpp
void delete_node(Node* v) {
    // v не является последним узлом
    Node* next = v->next;
    v->val  = next->val;   // скопировать данные
    v->next = next->next;  // обойти next
    delete next;           // удалить следующий узел
}
```

**Пример**: список `1 -> 2 -> 3 -> 4`, удаляем узел со значением 2.

Шаги:
1. `next = ` узел с `val=3`.
2. `v->val = 3` → список выглядит как `1 -> 3 -> 3 -> 4`.
3. `v->next = next->next` → `1 -> 3 -> 4`.
4. `delete next` (бывший узел с `val=3`).

**Ограничение**: метод не работает для последнего узла (нет `next`). В этом случае необходим предыдущий указатель.

### Ответ

Скопировать `v->val = v->next->val`, затем `v->next = v->next->next`, удалить `v->next` (до перезаписи). Сложность: $O(1)$.

## Задача 9. Нижняя граница для операций с очередью на двух стеках

Реализуйте очередь, используя **два стека** (`s_in` и `s_out`). Докажите, что амортизированная стоимость каждой операции — $O(1)$.

### Решение

**Идея**: `push` всегда кладёт в `s_in`. `pop` / `front` берёт из `s_out`; если `s_out` пуст, перекладываем все элементы из `s_in` в `s_out` (порядок инвертируется, превращая LIFO в FIFO).

```cpp
#include <stack>
#include <stdexcept>

struct QueueOnStacks {
    std::stack<int> s_in, s_out;

    void push(int x) {
        s_in.push(x);
    }
    void migrate() {
        if (s_out.empty()) {
            while (!s_in.empty()) {
                s_out.push(s_in.top());
                s_in.pop();
            }
        }
    }
    int front() {
        migrate();
        if (s_out.empty()) throw std::underflow_error("empty");
        return s_out.top();
    }
    void pop() {
        migrate();
        if (s_out.empty()) throw std::underflow_error("empty");
        s_out.pop();
    }
    bool empty() const { return s_in.empty() && s_out.empty(); }
};
```

**Доказательство амортизированности**: каждый элемент проходит путь: `push` в `s_in` ($O(1)$), однократный перенос в `s_out` ($O(1)$), однократный `pop` из `s_out` ($O(1)$). Итого 3 константных операции на элемент за всё его «время жизни». Суммарная стоимость $n$ операций — $O(n)$, амортизированная — $O(1)$ на операцию.

### Ответ

Реализация через два стека обеспечивает $O(1)$ амортизированную стоимость на каждую операцию `push` и `pop`.

## Задача 10. Синтезная задача: валидатор HTML-тегов

Напишите функцию, которая проверяет, что в строке теги HTML правильно вложены. Открывающий тег вида `<tag>` должен закрываться соответствующим `</tag>`. Самозакрывающиеся теги (`<br/>`) игнорируются. Проанализируйте сложность.

### Решение

**Алгоритм**:
1. Найти следующий тег (подстроку между `<` и `>`).
2. Если тег начинается с `/` — это закрывающий. Сравнить с вершиной стека.
3. Если тег заканчивается на `/` — самозакрывающийся, пропустить.
4. Иначе — открывающий, положить в стек.
5. В конце стек должен быть пуст.

```cpp
#include <iostream>
#include <stack>
#include <string>

bool validate_html(const std::string& html) {
    std::stack<std::string> st;
    int n = html.size();
    int i = 0;
    while (i < n) {
        if (html[i] == '<') {
            int j = html.find('>', i + 1);
            if (j == (int)std::string::npos) return false; // незакрытый <
            std::string tag = html.substr(i + 1, j - i - 1);
            i = j + 1;

            if (!tag.empty() && tag.back() == '/') {
                // самозакрывающийся: <br/>, пропускаем
                continue;
            }
            if (!tag.empty() && tag[0] == '/') {
                // закрывающий тег
                std::string name = tag.substr(1);
                if (st.empty() || st.top() != name) return false;
                st.pop();
            } else {
                // открывающий тег
                st.push(tag);
            }
        } else {
            i++;
        }
    }
    return st.empty();
}

int main() {
    std::cout << validate_html("<html><body><p>text</p></body></html>") << "\n"; // 1
    std::cout << validate_html("<div><p></div></p>") << "\n";                    // 0
    std::cout << validate_html("<img/><br/>") << "\n";                           // 1
    std::cout << validate_html("<a><b></a>") << "\n";                            // 0
    return 0;
}
```

**Анализ сложности**:
- Каждый символ строки просматривается не более двух раз (основной цикл + `find`).
- Каждый тег добавляется и удаляется из стека не более одного раза.
- **Время**: $O(|html|)$.
- **Память**: $O(d)$, где $d$ — максимальная глубина вложенности тегов.

**Почему это синтезная задача**: используется стек (LIFO) для отслеживания вложенности; применяется принцип скобочной последовательности к строкам произвольной длины; дополнительно обрабатываются граничные случаи (самозакрывающиеся теги, незакрытые `<`).

### Ответ

Функция `validate_html` работает за $O(|html|)$ по времени и $O(d)$ по памяти, где $d$ — максимальная глубина вложенности.
