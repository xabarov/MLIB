# 10 задач в стиле вступительного экзамена ШАД
по теме: Мосты и точки сочленения в графах

## Задача 1. Граф-дерево

Дано дерево $T$ на $n$ вершинах ($n \geq 2$). Ответьте:
- Сколько мостов содержит $T$?
- Какие вершины являются точками сочленения?

### Решение

**Мосты.** В дереве между любыми двумя вершинами ровно один путь. Следовательно, любое ребро $(u, v)$ единственное, соединяющее свои части дерева. Удаление любого ребра разделяет дерево на два поддерева.

Значит, **каждое из $n-1$ рёбер дерева является мостом**.

**Точки сочленения.** Вершина является точкой сочленения тогда, когда она «держит» несколько поддеревьев. В дереве вершина $v$ имеет степень $\deg(v)$. При удалении $v$ граф распадается на $\deg(v)$ компонент (каждый сосед уходит в своё поддерево), если $\deg(v) \geq 2$. Лист ($\deg(v) = 1$) точкой сочленения не является.

Значит, **точками сочленения являются все вершины со степенью $\geq 2$**, то есть все нелистовые вершины.

Формально: в DFS на дереве $\mathit{low}[v] = \mathit{disc}[v]$ для всех $v$ (нет обратных рёбер). Тогда для каждого ребра $(u, v)$: $\mathit{low}[v] = \mathit{disc}[v] > \mathit{disc}[u]$ — мост. Для каждой внутренней вершины $u$: у неё есть ребёнок $v$ с $\mathit{low}[v] = \mathit{disc}[v] \geq \mathit{disc}[u]$ — точка сочленения.

### Ответ

Дерево на $n$ вершинах содержит $n-1$ мост (все рёбра). Точки сочленения — все вершины со степенью $\geq 2$ (нелистовые).

## Задача 2. Ручная трассировка

Дан граф с вершинами $\{0,1,2,3,4\}$ и рёбрами $\{0\text{-}1,\,0\text{-}2,\,1\text{-}2,\,2\text{-}3,\,3\text{-}4\}$. Прогоните DFS от вершины 0, вычислите $\mathit{disc}[]$ и $\mathit{low}[]$ для каждой вершины, найдите все мосты и точки сочленения.

### Решение

Запустим DFS от 0. Порядок обхода соседей: 1, 2, 3, 4 (естественный).

**Шаг 1.** Входим в 0: $\mathit{disc}[0]=0$, $\mathit{low}[0]=0$.

**Шаг 2.** Входим в 1 (сосед 0): $\mathit{disc}[1]=1$, $\mathit{low}[1]=1$.
- Сосед 2 не посещён: рекурсия.

**Шаг 3.** Входим в 2 (сосед 1): $\mathit{disc}[2]=2$, $\mathit{low}[2]=2$.
- Сосед 0 посещён, $0 \neq \mathit{parent}=1$: $\mathit{low}[2] = \min(2, \mathit{disc}[0]) = 0$.
- Сосед 1 посещён, $1 = \mathit{parent}$: пропуск.
- Сосед 3 не посещён: рекурсия.

**Шаг 4.** Входим в 3 (сосед 2): $\mathit{disc}[3]=3$, $\mathit{low}[3]=3$.
- Сосед 4 не посещён: рекурсия.

**Шаг 5.** Входим в 4 (сосед 3): $\mathit{disc}[4]=4$, $\mathit{low}[4]=4$.
- Нет непосещённых соседей кроме parent=3: $\mathit{low}[4]=4$.

Выход из 4: $\mathit{low}[3] = \min(3, \mathit{low}[4]) = \min(3,4) = 3$.

Выход из 3: $\mathit{low}[2] = \min(0, \mathit{low}[3]) = \min(0,3) = 0$.

Выход из 2: $\mathit{low}[1] = \min(1, \mathit{low}[2]) = \min(1,0) = 0$.

Выход из 1: $\mathit{low}[0] = \min(0, \mathit{low}[1]) = 0$.

**Итоговая таблица:**

| v | disc | low |
|---|------|-----|
| 0 | 0 | 0 |
| 1 | 1 | 0 |
| 2 | 2 | 0 |
| 3 | 3 | 3 |
| 4 | 4 | 4 |

**Мосты** (условие $\mathit{low}[v] > \mathit{disc}[u]$, где $v$ — ребёнок $u$):
- Ребро (0,1): $\mathit{low}[1]=0 \leq \mathit{disc}[0]=0$ — нет.
- Ребро (1,2): $\mathit{low}[2]=0 \leq \mathit{disc}[1]=1$ — нет.
- Ребро (2,3): $\mathit{low}[3]=3 > \mathit{disc}[2]=2$ — **мост**.
- Ребро (3,4): $\mathit{low}[4]=4 > \mathit{disc}[3]=3$ — **мост**.

**Точки сочленения** (условие $\mathit{low}[v] \geq \mathit{disc}[u]$ для не-корней):
- Вершина 2: ребёнок 3, $\mathit{low}[3]=3 \geq \mathit{disc}[2]=2$ — **точка сочленения**.
- Вершина 3: ребёнок 4, $\mathit{low}[4]=4 \geq \mathit{disc}[3]=3$ — **точка сочленения**.
- Корень 0: один ребёнок (1) — не точка сочленения.

### Ответ

Мосты: {2–3}, {3–4}. Точки сочленения: {2, 3}.

## Задача 3. Граф без мостов

Докажите, что в графе, у которого каждая вершина имеет чётную степень, мостов нет.

### Решение

**Лемма об Эйлеровом цикле.** Связный граф, в котором все вершины имеют чётную степень, содержит эйлеров цикл (обходящий каждое ребро ровно один раз).

**Доказательство отсутствия мостов.** Предположим от противного, что ребро $e = (u, v)$ является мостом. Тогда удаление $e$ разделяет граф на два компонента $A$ (содержащий $u$) и $B$ (содержащий $v$).

В компоненте $A$ у каждой вершины чётная степень, кроме вершины $u$: ребро $e$ вносило 1 в степень $u$, после удаления степень $u$ стала нечётной.

Но это противоречит: сумма степеней в $A$ должна быть чётной (равна $2|E_A|$), а если $u$ — единственная вершина нечётной степени, сумма нечётна. Противоречие.

Значит, мостов нет.

**Альтернативная интерпретация через low.** Граф с эйлеровым циклом обладает тем свойством, что каждое ребро лежит на некотором цикле (иначе после обхода цикла мы не смогли бы вернуться через $e$). Ребро, лежащее на цикле, не является мостом (обратное ребро цикла делает $\mathit{low}[v] \leq \mathit{disc}[u]$).

### Ответ

В связном графе, где все степени чётны (существует эйлеров цикл), мостов нет: любое ребро лежит хотя бы на одном цикле.

## Задача 4. Подсчёт мостов в петле

Дан граф $G$: цикл $C_k$ на $k \geq 3$ вершинах плюс одна «хордовая» вершина $h$, соединённая с двумя несмежными вершинами цикла. Сколько мостов в $G$?

### Решение

Обозначим вершины цикла $v_0, v_1, \ldots, v_{k-1}$ с рёбрами $v_i - v_{(i+1)\%k}$. Пусть хорда соединяет $v_0$ и $v_j$ через $h$: рёбра $h - v_0$ и $h - v_j$, $2 \leq j \leq k-2$.

**Анализ.**
- Ребро $h - v_0$ входит в цикл $h - v_0 - v_1 - \ldots - v_j - h$ (через часть $C_k$). Значит, это не мост.
- Ребро $h - v_j$ входит в тот же цикл. Не мост.
- Рёбра цикла между $v_0$ и $v_j$ (их $j$ штук): каждое из них входит хотя бы в цикл через $h$.
- Рёбра цикла между $v_j$ и $v_0$ «в другую сторону» ($k - j$ штук): каждое входит в цикл через $h$.

Все рёбра участвуют в каком-либо цикле, проходящем через $h$.

Таким образом, $G$ — граф с эйлеровым циклом? Нет: вершина $h$ имеет степень 2, а $v_0$ и $v_j$ — степень 3 (нечётные). Но наличие «хорды» не гарантирует мостов.

Проверим явно: рёбро $(v_i, v_{i+1})$ (не между $v_0$ и $v_j$) — все они лежат на цикле, образованном самим $C_k$. Значит, мостов нет.

**Ответ:** мостов 0. Все рёбра лежат на циклах (каждое ребро $C_k$ лежит на $C_k$, а рёбра $h - v_0$ и $h - v_j$ лежат на коротком цикле через $h$).

### Ответ

Мостов в $G$ нет (ноль).

## Задача 5. Минимальный граф с k мостами

Постройте граф с наименьшим числом вершин, содержащий ровно $k$ мостов и не имеющий точек сочленения, кроме концов мостов. Какова его структура?

### Решение

**Идея:** берём цепочку $k+1$ «двусвязных блоков», соединённых мостами.

Простейший двусвязный блок, не являющийся мостом — треугольник $K_3$ (3 вершины, 3 ребра). Вершины сочленения — концы моста между блоками.

Конструкция:
- Блок $B_0$: треугольник на вершинах $\{a_0, b_0, c_0\}$.
- Блок $B_1$: треугольник на вершинах $\{a_1, b_1, c_1\}$.
- ...
- Блок $B_k$: треугольник на вершинах $\{a_k, b_k, c_k\}$.
- Мосты: $c_0 - a_1$, $c_1 - a_2$, ..., $c_{k-1} - a_k$.

Итого: $3(k+1)$ вершин, $3(k+1) + k$ рёбер, ровно $k$ мостов.

Точки сочленения: $c_0, c_1, \ldots, c_{k-1}$ (начала мостов) и $a_1, a_2, \ldots, a_k$ (концы мостов) — всего $2k$ вершин.

Можно ли обойтись меньшим числом вершин? Можно совместить конец одного моста и начало следующего — тогда точка сочленения «между двумя мостами» имеет степень 4 (по 2 ребра в каждый блок + 2 ребра мостов). Это даёт $2k + 2$ вершины при $k+1$ блоках, но блоки тогда — простые рёбра (мосты), что не подходит.

Оптимальная конструкция: $k$ мостов плюс $k+1$ треугольников. Минимум: $3(k+1)$ вершин.

### Ответ

Цепочка из $k+1$ треугольников, соединённых $k$ мостами. Минимальное число вершин: $3(k+1)$.

## Задача 6. Проверка корректности

Рассмотрите следующий «оптимизированный» алгоритм: вместо `low[u] = min(low[u], disc[v])` для обратных рёбер используем `low[u] = min(low[u], low[v])`. Докажите или опровергните: алгоритм всё ещё корректно находит мосты.

### Решение

**Опровержение.** Рассмотрим граф:

```
0 - 1 - 2
    |   |
    +---+
        |
        3 - 4
```

Рёбра: {0-1, 1-2, 2-1 (нет, только 1-2), 2-3, 3-4}. Точнее: {0-1, 1-2, 1-2 — нет дублей, возьмём 0-1, 1-2, 2-3, 3-4, плюс обратное ребро 2-0}.

Для простоты возьмём: вершины {0,1,2,3}, рёбра {0-1, 1-2, 2-0, 2-3}.

Правильный DFS от 0:
- $\mathit{disc}$: 0→0, 1→1, 2→2, 3→3.
- Обратное ребро (2,0): обновляем $\mathit{low}[2] = \min(2, \mathit{disc}[0]) = 0$.
- $\mathit{low}[3] = 3$ (нет обратных рёбер из поддерева 3).
- Ребро (2,3): $\mathit{low}[3]=3 > \mathit{disc}[2]=2$ → мост (правильно).

Теперь применим «оптимизацию» $\mathit{low}[u] = \min(\mathit{low}[u], \mathit{low}[v])$ для уже посещённой $v$:

Когда обрабатываем (2,0) как обратное ребро: $\mathit{low}[2] = \min(2, \mathit{low}[0]) = \min(2, 0) = 0$ — здесь совпадает с правильным, так как $\mathit{low}[0] = 0 = \mathit{disc}[0]$.

Но рассмотрим граф: {0-1, 1-2, 2-3, 3-1, 3-4}.

Правильные значения: $\mathit{disc}$: 0,1,2,3,4. Обратное ребро (3,1): $\mathit{low}[3] = 1$. Тогда $\mathit{low}[2] = \min(2, 1) = 1$, $\mathit{low}[1] = \min(1,1) = 1$.
- Ребро (0,1): $\mathit{low}[1]=1 > \mathit{disc}[0]=0$ → мост (правильно).
- Ребро (3,4): $\mathit{low}[4]=4 > \mathit{disc}[3]=3$ → мост (правильно).

«Оптимизация» в этом случае: при обработке (3,1) используем $\mathit{low}[1]=1 = \mathit{disc}[1]$. Результат тот же.

Контрпример: граф с «мнимой» связностью. Рёбра: {0-1, 1-2, 2-3, 3-0, 1-4, 4-2}. Здесь есть два треугольника через вершины 1 и 2. Если используем $\mathit{low}[v]$ вместо $\mathit{disc}[v]$, значение $\mathit{low}$ может «протечь» через несколько обратных рёбер, занижая $\mathit{low}$ за пределы разрешённой семантики (одно обратное ребро). Это нарушит инвариант: ребро, являющееся мостом, может получить $\mathit{low}[v] \leq \mathit{disc}[u]$ и не будет найдено.

Конкретно: граф {0-1, 1-2, 2-3, 3-2} + ещё вершина, не важно. Если поддерево $v$ имеет своё обратное ребро к далёкому предку, «оптимизация» транзитивно занизит $\mathit{low}$ для промежуточных вершин, нарушая корректность.

### Ответ

Алгоритм некорректен. Для обратных рёбер нужно использовать $\mathit{disc}[v]$, а не $\mathit{low}[v]$: $\mathit{low}$ кодирует «минимум через одно обратное ребро», транзитивное применение ломает семантику.

## Задача 7. Точки сочленения и связность

Граф $G$ не имеет точек сочленения. Докажите, что после удаления любой одной вершины граф остаётся связным.

### Решение

Это прямое следствие определения. Разберём подробно.

**Определение:** $v$ — точка сочленения, если $G - v$ несвязен. Граф без точек сочленения означает: для любой вершины $v$ граф $G - v$ остаётся связным.

**Формальное доказательство:**

Допустим от противного: $G$ не имеет точек сочленения, но существует вершина $v$ такая, что $G - v$ несвязен. Тогда по определению $v$ является точкой сочленения — противоречие с условием.

**Замечание:** граф без точек сочленения называется *двусвязным* (или «вершинно-2-связным», vertex-2-connected). В нём между любыми двумя вершинами $u, w$ существует хотя бы два вершинно-независимых пути. Это гарантирует связность даже при удалении любой одной вершины.

**Следствие:** если $G$ двусвязен и $|V| \geq 3$, то $G - v$ связен для всех $v$.

### Ответ

Если граф не имеет точек сочленения, то по определению удаление любой вершины не нарушает связность — граф двусвязен.

## Задача 8. Количество двусвязных компонент

Дан граф $G$ с $n$ вершинами, $m$ рёбрами, $c$ связными компонентами, $a$ точками сочленения. Выразите количество двусвязных компонент (BCC) через эти величины.

### Решение

**Подход через block-cut дерево.** Block-cut дерево $T$ для каждой связной компоненты — дерево, чьи узлы — BCC и точки сочленения. Для одной связной компоненты с $a_i$ точками сочленения и $b_i$ BCC имеем:

Дерево $T_i$ содержит $a_i + b_i$ узлов и $a_i + b_i - 1$ рёбер. Каждая точка сочленения соединяется с каждым BCC, которому она принадлежит.

**Ключевое соотношение для дерева:** в block-cut дереве ровно $a_i + b_i - 1$ рёбер, и структура чередующаяся (AP — BCC — AP — ...), то есть в дереве не может быть двух соседних узлов одного типа. Из этого следует: $b_i = a_i + 1$ для каждой связной компоненты (из свойства дерева с чередующимися узлами).

Для всего графа с $c$ компонентами:

$$
b = a + c
$$

**Проверка на примере.** Граф {0-1, 1-2, 2-0, 2-3, 3-4, 4-5, 5-3}: $c=1$, $a=2$ (вершины 2,3), $b = 2 + 1 = 3$ (три BCC: треугольник {0,1,2}, мост {2-3}, треугольник {3,4,5}). Верно.

**Проверка на дереве.** $n$ вершин, $m = n-1$ рёбер, $c=1$. Каждое ребро — мост = отдельный BCC, значит $b = n-1$. Точек сочленения: все внутренние вершины, $a = n - (\text{листья})$. Для дерева-пути: $a = n-2$, $b = n-1 = (n-2) + 1$. Формула работает.

### Ответ

Количество двусвязных компонент равно $b = a + c$, где $a$ — число точек сочленения, $c$ — число связных компонент.

## Задача 9. Кратные рёбра

Дан граф с кратными рёбрами: между вершинами 0 и 1 существует два ребра (обозначим их ребро $e_1$ и ребро $e_2$). Остальных рёбер нет (граф из двух вершин и двух рёбер между ними). Что выдаст наивный алгоритм (передающий `parent` как номер вершины)? Что выдаст корректный алгоритм (передающий `parent_edge_id`)?

### Решение

**Граф:** $V = \{0, 1\}$, $E = \{e_1 = (0,1),\, e_2 = (0,1)\}$.

**Наивный алгоритм (parent = номер вершины).**

DFS от 0: $\mathit{disc}[0]=0$, входим в 1 по $e_1$: $\mathit{disc}[1]=1$.
У вершины 1 соседи: 0 (по $e_1$) и 0 (по $e_2$). Parent = 0.

Обработка соседа 0 по $e_1$: $v=0 = parent$ → пропуск (считаем родительским).
Обработка соседа 0 по $e_2$: $v=0 = parent$ → пропуск (опять считаем родительским).

$\mathit{low}[1] = 1$. Выход из 1: $\mathit{low}[0] = \min(0, \mathit{low}[1]) = 0$.

Проверка ребра $(0,1)$: $\mathit{low}[1] = 1 > \mathit{disc}[0] = 0$ → **алгоритм ошибочно объявляет $(0,1)$ мостом**.

Но в действительности удаление $e_1$ не разрывает граф (остаётся $e_2$). Мостов нет.

**Корректный алгоритм (parent_edge_id).**

DFS от 0 по $e_1$ (id=0): $\mathit{disc}[1]=1$.
У вершины 1 соседи: 0 по $e_1$ (id=0), 0 по $e_2$ (id=1). parent_edge = 0.

Обработка $e_1$ (id=0 = parent_edge): пропуск.
Обработка $e_2$ (id=1 ≠ parent_edge): посещена, $\mathit{low}[1] = \min(1, \mathit{disc}[0]) = 0$.

Выход из 1: $\mathit{low}[0] = \min(0, 0) = 0$.

Проверка: $\mathit{low}[1] = 0 \leq \mathit{disc}[0] = 0$ → **не мост**. Верно!

### Ответ

Наивный алгоритм ошибочно объявляет (0,1) мостом. Корректный алгоритм правильно определяет: мостов нет. При кратных рёбрах обязательно передавать индекс ребра, а не номер вершины-родителя.

## Задача 10. Синтез: надёжность сети

Дана компьютерная сеть в виде графа $G$. Требуется найти минимальное количество рёбер, которые нужно добавить в граф, чтобы в нём не осталось мостов (то есть сеть стала «двусвязной по рёбрам»).

Опишите алгоритм и вычислите ответ для графа с рёбрами {0-1, 1-2, 2-3, 3-4}.

### Решение

**Постановка:** требуется минимальное число добавляемых рёбер для устранения всех мостов. Это классическая задача о «рёберной 2-связности» (edge 2-connectivity augmentation).

**Алгоритм (общий случай):**

1. Найти все мосты с помощью алгоритма Тарьяна за $O(V+E)$.
2. Построить *bridge tree* (дерево мостов): стянуть каждую двусвязную компоненту в одну вершину, мосты остаются как рёбра. Получаем дерево $T$.
3. Ответ: $\lceil L/2 \rceil$, где $L$ — количество листьев дерева $T$.

**Интуиция:** в дереве $T$ нужно добавить рёбра, чтобы оно перестало быть деревом (стало 2-связным). Оптимальная стратегия — соединять попарно листья. Если листьев $L$, нужно $\lceil L/2 \rceil$ рёбер.

**Применение к графу {0-1, 1-2, 2-3, 3-4}.**

Граф — путь из 5 вершин. Все 4 ребра — мосты. Каждая вершина — отдельный BCC (из одного ребра).

Bridge tree: дерево-путь $0 - 1 - 2 - 3 - 4$ (где каждый «узел» — вершина исходного графа или BCC).

На самом деле, BCC-узлы дерева мостов для пути: каждое ребро — BCC. Узлы bridge tree — вершины графа (5 штук), ребра — мосты (4 штуки). Получаем путь.

Листья дерева $T$: вершины 0 и 4 (степень 1). $L = 2$.

Ответ: $\lceil 2/2 \rceil = 1$. Достаточно добавить **одно ребро** (0-4), замыкающее путь в цикл.

**Проверка:** граф $\{0-1, 1-2, 2-3, 3-4, 4-0\}$ — цикл $C_5$. В цикле нет мостов. Верно.

**Для пути из $n$ вершин** в общем случае: bridge tree — тот же путь, $L=2$, ответ = 1.

**Для звезды $K_{1,n}$** ($n$ листьев): bridge tree — звезда с $n$ листьями, ответ = $\lceil n/2 \rceil$.

### Ответ

Для графа-пути {0-1, 1-2, 2-3, 3-4}: добавить **1 ребро** (например, 0-4). Общий алгоритм: построить дерево мостов, ответ = $\lceil L/2 \rceil$, где $L$ — число листьев этого дерева.
