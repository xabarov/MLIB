## Задача 1. Сумма на отрезке

**Условие.** Дан массив из $n = 8$ элементов: $A = [3, 1, 4, 1, 5, 9, 2, 6]$. Постройте дерево отрезков для суммы. Нарисуйте дерево (индексы узлов и их значения). Вычислите сумму на отрезке $[2, 5]$.

### Решение

**Шаг 1: Строим дерево.**

$n = 8$ (степень двойки), дерево идеально сбалансировано.

```
Узел 1  [0,7] = 31
  Узел 2  [0,3] = 9       Узел 3  [4,7] = 22
    Узел 4 [0,1]=4          Узел 6 [4,5]=14      Узел 7 [6,7]=8
      Узел 5 [2,3]=5
         8  9              12   13               14  15
        [0] [1]           [2]  [3]
         3   1             4    1

Листья (слева направо): 3,1,4,1,5,9,2,6
tree[8]=3, tree[9]=1, tree[10]=4, tree[11]=1
tree[12]=5, tree[13]=9, tree[14]=2, tree[15]=6
tree[4]=4,  tree[5]=5,  tree[6]=14, tree[7]=8
tree[2]=9,  tree[3]=22, tree[1]=31
```

**Шаг 2: query(1, [0,7], ql=2, qr=5).**

- Узел 1 [0,7]: частичное пересечение → спуск.
- Узел 2 [0,3]: частичное пересечение → спуск.
  - Узел 4 [0,1]: нет пересечения (1 < 2) → return 0.
  - Узел 5 [2,3]: полностью внутри → return 5.
  - return 0 + 5 = 5.
- Узел 3 [4,7]: частичное пересечение → спуск.
  - Узел 6 [4,5]: полностью внутри → return 14.
  - Узел 7 [6,7]: нет пересечения (6 > 5) → return 0.
  - return 14 + 0 = 14.
- Итог: 5 + 14 = **19**.

**Проверка:** $A[2]+A[3]+A[4]+A[5] = 4+1+5+9 = 19$. ✓

### Ответ

Дерево построено, сумма $A[2..5] = 19$.

## Задача 2. Обновление и повторный запрос

**Условие.** Используя дерево из Задачи 1 ($A = [3,1,4,1,5,9,2,6]$), выполните обновление $A[4] = 0$. Какие узлы дерева изменяются? Каково новое значение суммы на $[2, 5]$?

### Решение

**Шаг 1: update(1, [0,7], pos=4, val=0).**

- Узел 1 [0,7]: $4 > \text{mid}=3$ → идём вправо.
- Узел 3 [4,7]: $4 \le \text{mid}=5$ → идём влево.
- Узел 6 [4,5]: $4 \le \text{mid}=4$ → идём влево.
- Узел 12 [4,4]: лист, $l=r=4$. `tree[12] = 0`.
- Возврат: `tree[6] = tree[12]+tree[13] = 0+9 = 9`.
- Возврат: `tree[3] = tree[6]+tree[7] = 9+8 = 17`.
- Возврат: `tree[1] = tree[2]+tree[3] = 9+17 = 26`.

**Изменились узлы:** 12, 6, 3, 1 — ровно путь от корня до листа, $\log_2 8 = 3$ шага.

**Шаг 2: query [2,5] после обновления.**

- Узел 2 [0,3] → ... → узел 5 [2,3] = 5.
- Узел 6 [4,5] = 9 (уже обновлён).
- Итог: $5 + 9 = 14$.

**Проверка:** $4+1+0+9 = 14$. ✓

### Ответ

Изменились 4 узла: листовой [4,4], затем [4,5], [4,7], [0,7]. Новая сумма $[2,5] = 14$.

## Задача 3. Минимум на отрезке

**Условие.** Постройте дерево отрезков для нахождения минимума массива $A = [5, 2, 8, 1, 9, 3, 7, 4]$. Найдите минимум на $[1, 6]$.

### Решение

**Строим дерево (агрегат = min):**

```
Листья: 5,2,8,1,9,3,7,4

tree[4] = min(5,2) = 2
tree[5] = min(8,1) = 1
tree[6] = min(9,3) = 3
tree[7] = min(7,4) = 4
tree[2] = min(2,1) = 1
tree[3] = min(3,4) = 3
tree[1] = min(1,3) = 1
```

**query_min(1, [0,7], ql=1, qr=6):**

- Узел 1 [0,7]: частичное → спуск.
- Узел 2 [0,3]: частичное → спуск.
  - Узел 4 [0,1]: частичное (0 < ql=1) → спуск.
    - Узел 8 [0,0]: нет пересечения → $+\infty$.
    - Узел 9 [1,1]: внутри → return 2.
    - return min($+\infty$, 2) = 2.
  - Узел 5 [2,3]: внутри → return 1.
  - return min(2, 1) = 1.
- Узел 3 [4,7]: частичное (7 > qr=6) → спуск.
  - Узел 6 [4,5]: внутри → return 3.
  - Узел 7 [6,7]: частичное → спуск.
    - Узел 14 [6,6]: внутри → return 7.
    - Узел 15 [7,7]: нет пересечения → $+\infty$.
    - return min(7, $+\infty$) = 7.
  - return min(3, 7) = 3.
- Итог: min(1, 3) = **1**.

**Проверка:** $\min(2,8,1,9,3,7) = 1$. ✓

### Ответ

Минимум на $[1,6] = 1$ (элемент $A[3]$).

## Задача 4. Ленивая пропагация: range add

**Условие.** Дан массив $A = [1, 2, 3, 4]$. Постройте дерево отрезков с ленивой пропагацией для суммы. Выполните операцию: прибавить 5 ко всем элементам $[1, 3]$. Затем запросите сумму $[0, 3]$. Покажите состояние `tree[]` и `lazy[]` на каждом шаге.

### Решение

**Начальное дерево** (узлы 1..7, $n=4$):

```
tree[1]=10, tree[2]=3, tree[3]=7
tree[4]=1,  tree[5]=2, tree[6]=3, tree[7]=4
lazy[1..7] = 0
```

**range_update(1, [0,3], ql=1, qr=3, val=5):**

- Узел 1 [0,3]: частичное → push_down (нечего, lazy=0) → спуск.
- Узел 2 [0,1]: частичное → push_down (нечего) → спуск.
  - Узел 4 [0,0]: нет пересечения (0 < ql=1) → return.
  - Узел 5 [1,1]: полностью внутри:
    - `tree[5] += 5*1 = 7`, `lazy[5] += 5`.
  - Пересчёт: `tree[2] = tree[4] + tree[5] = 1 + 7 = 8`.
- Узел 3 [2,3]: полностью внутри:
  - `tree[3] += 5*2 = 17`, `lazy[3] += 5`.
- Пересчёт: `tree[1] = tree[2]+tree[3] = 8+17 = 25`.

**Состояние после range_update:**

```
tree[1]=25, tree[2]=8, tree[3]=17
tree[4]=1,  tree[5]=7, tree[6]=3 (устарел!), tree[7]=4 (устарел!)
lazy[3]=5, остальные=0
```

**range_query(1, [0,3], ql=0, qr=3):**

- Узел 1 [0,3]: полностью внутри → return `tree[1] = 25`.

**Проверка:** $1 + (2+5) + (3+5) + (4+5) = 1+7+8+9 = 25$. ✓

### Ответ

После прибавления 5 к $[1,3]$ сумма $[0,3] = 25$. Ленивый тег $\text{lazy}[3] = 5$ хранится до следующего спуска в этот узел.

## Задача 5. Анализ сложности

**Условие.** Докажите, что функция `query` дерева отрезков выполняется за $O(\log n)$ вызовов. Сколько узлов максимально посещается на одном уровне дерева?

### Решение

**Ключевое утверждение:** на каждом уровне дерева функция `query` посещает не более 4 узлов.

**Доказательство индукцией по уровням.**

Обозначим запрос $[ql, qr]$.

**База:** На уровне корня посещается 1 узел ≤ 4.

**Переход:** Рассмотрим уровень $k$. Активные (посещённые) узлы этого уровня — те, что «частично» пересекают $[ql, qr]$. «Полностью внутренние» узлы не спускаются ниже — они сразу возвращают значение.

Граничный анализ: на каждой стороне ($ql$ и $qr$) запрос «ломает» не более одного узла на каждом уровне. Таким образом:

- Слева от $ql$: не более 2 активных узлов на уровне (один «частичный», один только что ставший «полным»).
- Справа от $qr$: аналогично не более 2 активных узлов.

Итого: $\le 4$ активных узлов на каждом уровне.

**Итог:** Глубина дерева $= \lceil \log_2 n \rceil$. При не более 4 узлов на уровне:

$$
T(\text{query}) = O(4 \log n) = O(\log n)
$$

Аналогично для `update`: посещается ровно один путь от корня до листа — $O(\log n)$ узлов.

### Ответ

На каждом уровне посещается не более 4 узлов, глубина $O(\log n)$, итоговая сложность $O(\log n)$.

## Задача 6. НОД на отрезке

**Условие.** Постройте дерево отрезков для НОД (наибольшего общего делителя) массива $A = [12, 8, 6, 4, 9, 3]$. Найдите НОД на $[0, 4]$. Какой нейтральный элемент нужен?

### Решение

**Нейтральный элемент для НОД:** 0, поскольку $\gcd(x, 0) = x$ для любого $x > 0$.

**Строим дерево:**

```
Листья: 12, 8, 6, 4, 9, 3

gcd(12,8)=4 → tree[4]=4
gcd(6,4)=2  → tree[5]=2
gcd(9,3)=3  → tree[6]=3

gcd(4,2)=2  → tree[2]=2
             tree[3]=3 (только tree[6], так как [4,5] имеет одного потомка)

gcd(2,3)=1  → tree[1]=1
```

Точнее при $n=6$, дерево несимметрично:
- Узел 1 [0,5], mid=2.
- Узел 2 [0,2], mid=1.
- Узел 3 [3,5], mid=4.
- Узел 4 [0,1]: gcd(12,8)=4.
- Узел 5 [2,2]: 6.
- Узел 6 [3,4]: gcd(4,9)... Нет, $A=[12,8,6,4,9,3]$, $A[3]=4, A[4]=9$ → gcd(4,9)=1.
- Узел 7 [5,5]: 3.

Пересчёт:
- tree[2] = gcd(tree[4], tree[5]) = gcd(4, 6) = 2.
- tree[3] = gcd(tree[6], tree[7]) = gcd(1, 3) = 1.
- tree[1] = gcd(2, 1) = 1.

**query_gcd(1, [0,5], ql=0, qr=4):**

- Узел 1 [0,5]: частичное (5 > 4) → спуск.
- Узел 2 [0,2]: внутри → return tree[2] = 2.
- Узел 3 [3,5]: частичное → спуск.
  - Узел 6 [3,4]: внутри → return tree[6] = 1.
  - Узел 7 [5,5]: нет пересечения → return 0.
  - return gcd(1, 0) = 1.
- return gcd(2, 1) = **1**.

**Проверка:** gcd(12,8,6,4,9) = gcd(gcd(12,8),gcd(6,4,9)) = gcd(4, gcd(2,9)) = gcd(4,1) = 1. ✓

### Ответ

НОД массива $A[0..4] = 1$. Нейтральный элемент — 0.

## Задача 7. Подсчёт инверсий

**Условие.** Дан массив $A = [3, 1, 4, 1, 5]$. Используйте дерево отрезков для подсчёта инверсий (пар $(i, j)$ таких, что $i < j$ и $A[i] > A[j]$). Опишите алгоритм и выполните подсчёт вручную.

### Решение

**Алгоритм:** Заведём дерево отрезков по значениям от 1 до $\max(A)$. Обрабатываем элементы слева направо. Для каждого $a_i$:
1. Запрашиваем sum на $(a_i, \max]$ — количество уже добавленных элементов, которые больше $a_i$ (т.е. инверсии с $a_i$).
2. Прибавляем этот count к ответу.
3. Выполняем point update: увеличиваем позицию $a_i$ на 1.

**Выполнение для $A = [3, 1, 4, 1, 5]$:**

Значения от 1 до 5. BIT/дерево отрезков изначально нулевое.

| Шаг | $a_i$ | Запрос $(a_i, 5]$ | Инверсий добавлено | update($a_i$) |
|---|---|---|---|---|
| 1 | 3 | sum[4,5] = 0 | 0 | freq[3]=1 |
| 2 | 1 | sum[2,5] = 1 (есть 3) | 1 | freq[1]=1 |
| 3 | 4 | sum[5,5] = 0 | 0 | freq[4]=1 |
| 4 | 1 | sum[2,5] = 2 (есть 3,4) | 2 | freq[1]=2 |
| 5 | 5 | sum[6,5] = 0 | 0 | freq[5]=1 |

**Итого инверсий:** $0 + 1 + 0 + 2 + 0 = 3$.

**Проверка вручную:** Инверсии: $(3,1)$, $(3,1)$, $(4,1)$ — действительно 3. ✓

**Сложность:** $O(n \log n)$ — $n$ запросов и обновлений, каждое за $O(\log n)$.

### Ответ

Количество инверсий = 3. Алгоритм: sweep слева направо + дерево отрезков по значениям, $O(n \log n)$.

## Задача 8. Ленивая пропагация: range assign

**Условие.** Модифицируйте дерево отрезков так, чтобы поддерживать операцию «присвоить всем элементам $[l, r]$ значение $v$» (range assign) и запрос «сумма на $[l, r]$». Чем отличается `push_down` от варианта с range add?

### Решение

**Ключевое отличие:** При range add теги «накапливаются» (lazy += val). При range assign новое значение «перекрывает» предыдущее — старый тег теряет смысл. Нужно добавить флаг «есть ли активный тег присваивания».

**Структура:**

```cpp
long long tree[4 * MAXN];
long long lazy[4 * MAXN];   // хранит значение для assign
bool has_lazy[4 * MAXN];    // флаг: есть ли тег

void push_down(int node, int l, int r) {
    if (!has_lazy[node]) return;
    int mid = (l + r) / 2;
    long long val = lazy[node];

    // Левый потомок
    tree[2 * node]     = val * (mid - l + 1);
    lazy[2 * node]     = val;
    has_lazy[2 * node] = true;

    // Правый потомок
    tree[2 * node + 1]     = val * (r - mid);
    lazy[2 * node + 1]     = val;
    has_lazy[2 * node + 1] = true;

    has_lazy[node] = false;  // тег передан
}

void range_assign(int node, int l, int r, int ql, int qr, long long val) {
    if (r < ql || l > qr) return;
    if (ql <= l && r <= qr) {
        tree[node]     = val * (r - l + 1);
        lazy[node]     = val;
        has_lazy[node] = true;
        return;
    }
    push_down(node, l, r);
    int mid = (l + r) / 2;
    range_assign(2 * node,     l,       mid, ql, qr, val);
    range_assign(2 * node + 1, mid + 1, r,   ql, qr, val);
    tree[node] = tree[2 * node] + tree[2 * node + 1];
}
```

**Трассировка:** $A=[1,2,3,4]$, assign $[1,3] = 7$.

- `tree[3]` [2,3]: внутри → `tree[3] = 7*2=14`, `lazy[3]=7`, `has_lazy[3]=true`.
- `tree[5]` [1,1]: внутри → `tree[5] = 7*1=7`, `lazy[5]=7`, `has_lazy[5]=true`.
- `tree[2]` [0,1]: пересчёт → `tree[2] = tree[4]+tree[5] = 1+7 = 8`.
- `tree[1]`: пересчёт → `8+14=22`.

Проверка: $1+7+7+7 = 22$. ✓

### Ответ

Для range assign: использовать флаг `has_lazy`; при push_down тег перезаписывает потомков (а не суммируется с ними). Сложность операций по-прежнему $O(\log n)$.

## Задача 9. Нахождение k-го нуля

**Условие.** Дан массив из нулей и единиц длины $n$. Поддержите операции: (1) toggle($i$) — инвертировать $A[i]$; (2) k_zero($k$) — найти позицию $k$-го нуля (по порядку слева) за $O(\log n)$.

### Решение

**Идея.** Дерево отрезков хранит в каждом узле количество нулей на соответствующем подотрезке. Тогда k-й ноль ищется спуском: если в левом поддереве не менее $k$ нулей, идём влево; иначе вычитаем их количество и идём вправо.

```cpp
int n;
int tree[4 * MAXN];  // tree[node] = кол-во нулей в [l,r]
int A[MAXN];

void build(int node, int l, int r) {
    if (l == r) { tree[node] = (A[l] == 0) ? 1 : 0; return; }
    int mid = (l + r) / 2;
    build(2 * node, l, mid);
    build(2 * node + 1, mid + 1, r);
    tree[node] = tree[2 * node] + tree[2 * node + 1];
}

void toggle(int node, int l, int r, int pos) {
    if (l == r) { A[pos] ^= 1; tree[node] = 1 - tree[node]; return; }
    int mid = (l + r) / 2;
    if (pos <= mid) toggle(2 * node, l, mid, pos);
    else toggle(2 * node + 1, mid + 1, r, pos);
    tree[node] = tree[2 * node] + tree[2 * node + 1];
}

int k_zero(int node, int l, int r, int k) {
    if (l == r) return l;  // нашли позицию
    int mid = (l + r) / 2;
    if (tree[2 * node] >= k) {
        return k_zero(2 * node, l, mid, k);  // k-й ноль в левом поддереве
    } else {
        return k_zero(2 * node + 1, mid + 1, r, k - tree[2 * node]);
    }
}
```

**Пример:** $A = [1, 0, 1, 0, 0, 1, 0]$, найти 3-й ноль.

- tree[1] = 4 (нули на позициях 1,3,4,6).
- Узел 1 [0,6]: tree[left=узел2, [0,3]] = 2 < 3 → идём вправо, $k=3-2=1$.
- Узел 3 [4,6]: tree[left=узел6, [4,5]] = 1 >= 1 → идём влево, $k=1$.
- Узел 6 [4,5]: tree[left=узел12, [4,4]] = 1 >= 1 → идём влево.
- Узел 12 [4,4]: лист, $A[4]=0$ → return 4.

Ответ: позиция 4. Проверка: нули на 1,3,4,6 — 3-й ноль на позиции 4. ✓

### Ответ

Спуск по дереву: в каждом узле сравниваем $k$ с числом нулей в левом потомке. $O(\log n)$.

## Задача 10. Слияние деревьев отрезков и 2D-задача

**Условие.** Дана матрица $n \times m$. Поддержите операцию: запрос суммы в подматрице $[r_1, r_2] \times [c_1, c_2]$ (без обновлений). Предложите два подхода: (1) через 2D-дерево отрезков; (2) через двумерные префиксные суммы. Сравните по памяти и времени запроса.

### Решение

**Подход 1: Двумерные префиксные суммы.**

Предвычислим $P[i][j] = \sum_{r=0}^{i-1}\sum_{c=0}^{j-1} A[r][c]$.

Формула включения-исключения:

$$
\text{sum}(r_1, r_2, c_1, c_2) = P[r_2+1][c_2+1] - P[r_1][c_2+1] - P[r_2+1][c_1] + P[r_1][c_1]
$$

- Построение: $O(nm)$.
- Запрос: $O(1)$.
- Обновление: $O(nm)$.
- Память: $O(nm)$.

**Подход 2: 2D дерево отрезков (дерево деревьев).**

Строим дерево отрезков по строкам. Каждый узел строкового дерева содержит своё дерево отрезков по столбцам.

- Построение: $O(nm)$ времени и памяти.
- Запрос: $O(\log n \cdot \log m)$.
- Point update: $O(\log n \cdot \log m)$.

```
Структурно: для каждого узла i строкового дерева
            храним дерево отрезков по столбцам с агрегатом соответствующих строк.
```

**Практические соображения:**

При $n, m = 10^3$: 2D-prefix суммы — $10^6$ памяти, $O(1)$ запрос, неприменимы при обновлениях. 2D дерево отрезков — $O(nm \log n \log m)$ памяти в наивной реализации, запрос $O(\log^2 n)$.

**Сравнение:**

| | Построение | Запрос | Обновление | Память |
|---|---|---|---|---|
| 2D prefix | $O(nm)$ | $O(1)$ | $O(nm)$ | $O(nm)$ |
| 2D seg tree | $O(nm)$ | $O(\log^2 n)$ | $O(\log^2 n)$ | $O(nm)$ |

При наличии обновлений — 2D дерево отрезков; без обновлений — двумерные префиксные суммы.

### Ответ

Без обновлений — 2D prefix ($O(1)$ запрос, $O(nm)$ память). С обновлениями — 2D дерево отрезков ($O(\log^2 n)$ запрос и обновление). Правильный выбор зависит от задачи.

---

<details><summary>Что тренируют эти задачи</summary>

1. Построение дерева отрезков вручную и чтение дерева по узлам.
2. Трассировка обновления и понимание, какие узлы меняются.
3. Адаптация агрегирующей функции (min вместо sum).
4. Ленивая пропагация range add: правильный расчёт тегов.
5. Формальный анализ сложности query через «4 узла на уровне».
6. Нестандартный агрегат (НОД) и выбор нейтрального элемента.
7. Применение дерева отрезков для подсчёта инверсий (алгоритмическая задача).
8. Ленивая пропагация range assign: флаг и перезапись тега.
9. Двоичный поиск по дереву (k-й ноль): нестандартное использование.
10. 2D-обобщение: выбор структуры данных в зависимости от условий задачи.

</details>
