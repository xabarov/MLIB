## Задача 1. Трассировка СНМ

Дан массив $\texttt{parent} = [0, 1, 2, 3, 4, 5]$ (каждый элемент — своя компонента). Выполните последовательно операции:

1. $\texttt{Union}(0, 1)$
2. $\texttt{Union}(2, 3)$
3. $\texttt{Union}(4, 5)$
4. $\texttt{Union}(1, 2)$
5. $\texttt{Union}(3, 5)$

Используйте объединение по рангу (начальный ранг всех = 0). Выпишите массив $\texttt{parent}$ и $\texttt{rank}$ после каждой операции. Являются ли $0$ и $5$ элементами одного множества?

### Решение

Начало: parent=[0,1,2,3,4,5], rank=[0,0,0,0,0,0].

**Union(0,1):** find(0)=0, find(1)=1. Ранги равны, 1 → под 0, rank[0]++.
- parent=[0,0,2,3,4,5], rank=[1,0,0,0,0,0]

**Union(2,3):** find(2)=2, find(3)=3. Ранги равны, 3 → под 2, rank[2]++.
- parent=[0,0,2,2,4,5], rank=[1,0,1,0,0,0]

**Union(4,5):** find(4)=4, find(5)=5. Ранги равны, 5 → под 4, rank[4]++.
- parent=[0,0,2,2,4,4], rank=[1,0,1,0,1,0]

**Union(1,2):** find(1)=0, find(2)=2. rank[0]=1, rank[2]=1 — равны, 2 → под 0, rank[0]++.
- parent=[0,0,0,2,4,4], rank=[2,0,1,0,1,0]

**Union(3,5):** find(3): parent[3]=2, parent[2]=0, так что find(3)=0. find(5)=4.
rank[0]=2 > rank[4]=1, значит 4 → под 0.
- parent=[0,0,0,2,0,4], rank=[2,0,1,0,1,0]

Теперь find(0)=0, find(5): parent[5]=4, parent[4]=0 → find(5)=0.

**Ответ:** Да, 0 и 5 в одном множестве (оба имеют корень 0).

### Ответ

После всех операций все 6 элементов принадлежат одному множеству с корнем 0. parent=[0,0,0,2,0,4].

## Задача 2. Вычисление префикс-функции

Вычислите префикс-функцию для строки $s = \texttt{"AABAABAAB"}$.

### Решение

$n = 9$, $\pi[0] = 0$ по определению.

**$i=1$, s[1]='A':** k=π[0]=0. s[1]='A' = s[0]='A' → k=1. π[1]=1.

**$i=2$, s[2]='B':** k=π[1]=1. s[2]='B' ≠ s[1]='A' → k=π[0]=0. s[2]='B' ≠ s[0]='A' → k=0. π[2]=0.

**$i=3$, s[3]='A':** k=π[2]=0. s[3]='A' = s[0]='A' → k=1. π[3]=1.

**$i=4$, s[4]='A':** k=π[3]=1. s[4]='A' = s[1]='A' → k=2. π[4]=2.

**$i=5$, s[5]='B':** k=π[4]=2. s[5]='B' = s[2]='B' → k=3. π[5]=3.

**$i=6$, s[6]='A':** k=π[5]=3. s[6]='A' = s[3]='A' → k=4. π[6]=4.

**$i=7$, s[7]='A':** k=π[6]=4. s[7]='A' = s[4]='A' → k=5. π[7]=5.

**$i=8$, s[8]='B':** k=π[7]=5. s[8]='B' = s[5]='B' → k=6. π[8]=6.

Результат: $\pi = [0, 1, 0, 1, 2, 3, 4, 5, 6]$.

Проверка: $\pi[8]=6$ означает, что префикс "AABAAБ" длины 6 равен суффиксу "AABAAБ" строки "AABAABAAB". Действительно: s[0..5]="AABAAB" = s[3..8]="AABAAB". ✓

### Ответ

$\pi = [0, 1, 0, 1, 2, 3, 4, 5, 6]$.

## Задача 3. Поиск образца с помощью КМП

Найдите все вхождения образца $P = \texttt{"ABA"}$ в текст $T = \texttt{"ABAABABAAB"}$ с помощью алгоритма КМП. Выпишите трассировку переменной $j$.

### Решение

Шаг 1: префикс-функция для "ABA".
- π[0]=0, π[1]=0, π[2]=1.

Шаг 2: поиск в тексте T="ABAABABAAB" (|T|=10).

| i | T[i] | j до | Действие | j после |
|---|---|---|---|---|
| 0 | A | 0 | A=P[0]='A' ✓ | 1 |
| 1 | B | 1 | B=P[1]='B' ✓ | 2 |
| 2 | A | 2 | A=P[2]='A' ✓ | 3 → **вхождение на позиции 0**, j=π[2]=1 |
| 3 | A | 1 | A=P[1]='B'? ✗ → j=π[0]=0; A=P[0]='A' ✓ | 1 |
| 4 | B | 1 | B=P[1]='B' ✓ | 2 |
| 5 | A | 2 | A=P[2]='A' ✓ | 3 → **вхождение на позиции 3**, j=π[2]=1 |
| 6 | B | 1 | B=P[1]='B' ✓ | 2 |
| 7 | A | 2 | A=P[2]='A' ✓ | 3 → **вхождение на позиции 5**, j=π[2]=1 |
| 8 | A | 1 | A=P[1]='B'? ✗ → j=π[0]=0; A=P[0]='A' ✓ | 1 |
| 9 | B | 1 | B=P[1]='B' ✓ | 2 |

### Ответ

Вхождения на позициях **0, 3, 5** (индексация с нуля).

## Задача 4. Операции на боре

Вставьте слова `["to", "tea", "ten", "in", "inn", "inn"]` в бор. Затем ответьте:

1. Сколько узлов в боре (не считая корень)?
2. Является ли `"te"` словом в боре?
3. Является ли `"te"` префиксом слова в боре?
4. Что вернёт `search("inn")` — и изменилась ли структура бора после второй вставки "inn"?

### Решение

Строим бор:

- "to": root → t → o[*]
- "tea": root → t → e → a[*]
- "ten": root → t → e → n[*]
- "in": root → i → n[*]
- "inn": root → i → n → n[*]
- "inn" (повтор): путь уже существует, is_end узла последнего 'n' уже true — ничего не меняется.

Узлы (не корень): t, o, e, a, n(после e), i, n(после i), n(после второго n) = **8 узлов**.

1. **8 узлов.**
2. `search("te")`: проходим root→t→e, узел 'e' не является терминальным → **нет**.
3. `starts_with("te")`: путь root→t→e существует → **да**.
4. `search("inn")` = **true**. Структура бора не изменилась: повторная вставка не добавляет новых узлов.

### Ответ

8 узлов, "te" — не слово, но префикс. "inn" найдено. Повторная вставка не меняет бор.

## Задача 5. Компоненты связности онлайн

Дан граф из $n=7$ вершин (0..6). Рёбра добавляются в следующем порядке: (0,1), (2,3), (1,4), (5,6), (3,5), (0,2).

После каждого ребра выведите количество компонент связности. Реализуйте через DSU.

### Решение

Начало: 7 компонент (каждая вершина сама по себе).

**(0,1):** find(0)=0, find(1)=1, разные → unite. Компонент: **6**.

**(2,3):** find(2)=2, find(3)=3, разные → unite. Компонент: **5**.

**(1,4):** find(1)=0 (корень компоненты {0,1}), find(4)=4, разные → unite. Компонент: **4**.

**(5,6):** find(5)=5, find(6)=6, разные → unite. Компонент: **3**.

**(3,5):** find(3)=2 (корень компоненты {2,3}), find(5)=5 (корень {5,6}), разные → unite. Компонент: **2**.

**(0,2):** find(0)=0 (корень {0,1,4}), find(2)=2 (корень {2,3,5,6}), разные → unite. Компонент: **1**.

### Ответ

После рёбер: 6, 5, 4, 3, 2, 1. В конце все вершины в одной компоненте.

## Задача 6. Цикловой тест в графе

Дан неориентированный граф: $V = \{0,1,2,3,4\}$, рёбра в порядке добавления: (0,1), (1,2), (2,3), (3,4), (4,1).

Определите, содержит ли граф цикл, используя DSU. Найдите ребро, добавление которого замыкает цикл.

### Решение

DSU: parent=[0,1,2,3,4], rank=[0,0,0,0,0].

**(0,1):** find(0)=0, find(1)=1, разные → unite(0,1). Цикла нет.

**(1,2):** find(1)=0, find(2)=2, разные → unite. Цикла нет.

**(2,3):** find(2)=0, find(3)=3, разные → unite. Цикла нет.

**(3,4):** find(3)=0, find(4)=4, разные → unite. Цикла нет.

**(4,1):** find(4)=0, find(1)=0 → **одна компонента!** Это ребро замыкает цикл.

Цикл: 1 → 2 → 3 → 4 → 1.

### Ответ

Граф содержит цикл. Ребро (4,1) замыкает цикл 1→2→3→4→1.

## Задача 7. Число различных подстрок через бор

Дан набор строк: "ab", "abc", "abcd", "b", "bc". Вставьте их в бор (на массиве из 26 символов). Подсчитайте суммарное количество узлов в боре (не считая корень). Чему равно это число и что оно означает?

### Решение

Строим бор:

- "ab": root → a → b[*]. Новые узлы: a, b (2 узла).
- "abc": root → a → b → c[*]. Новый узел: c (1 узел). Всего: 3.
- "abcd": root → a → b → c → d[*]. Новый узел: d. Всего: 4.
- "b": root → b[*]. Новый узел: b' (отдельная ветка). Всего: 5.
- "bc": root → b → c'[*]. Новый узел: c'. Всего: 6.

Итого: **6 узлов**.

Каждый узел соответствует уникальному непустому префиксу хотя бы одного из вставленных слов: "a", "ab", "abc", "abcd", "b", "bc". Число узлов в боре — это в точности количество **различных непустых префиксов** множества строк. (Не путать с количеством различных подстрок — это другая задача.)

### Ответ

6 узлов. Каждый узел соответствует одному из 6 уникальных префиксов: "a", "ab", "abc", "abcd", "b", "bc".

## Задача 8. Алгоритм Крускала на конкретном графе

Дан граф: $V=\{0,1,2,3,4\}$, рёбра с весами: (0,1,2), (0,3,6), (1,2,3), (1,3,8), (1,4,5), (2,4,7), (3,4,9).

Найдите минимальное остовное дерево с помощью алгоритма Крускала. Выпишите рёбра MST в порядке добавления и их суммарный вес.

### Решение

Сортируем рёбра по весу: (0,1,2), (1,2,3), (1,4,5), (0,3,6), (2,4,7), (1,3,8), (3,4,9).

DSU: 5 компонент.

**(0,1,w=2):** find(0)=0 ≠ find(1)=1 → добавляем. MST: {(0,1)}, компонент 4.

**(1,2,w=3):** find(1)=0, find(2)=2 → добавляем. MST: {(0,1),(1,2)}, компонент 3.

**(1,4,w=5):** find(1)=0, find(4)=4 → добавляем. MST: {(0,1),(1,2),(1,4)}, компонент 2.

**(0,3,w=6):** find(0)=0, find(3)=3 → добавляем. MST: {(0,1),(1,2),(1,4),(0,3)}, компонент 1.

Все $n-1 = 4$ ребра добавлены — останавливаемся.

Суммарный вес: $2 + 3 + 5 + 6 = 16$.

### Ответ

MST: рёбра (0,1), (1,2), (1,4), (0,3) с суммарным весом **16**.

## Задача 9. Количество слов с данным суффиксом

Дан словарь строк: ["apple", "maple", "staple", "table", "cable"]. Нужно для запроса `q = "ple"` найти количество слов, которые **заканчиваются** на `q`. Предложите алгоритм на основе бора. Ответьте на запрос вручную.

### Решение

**Идея**: вставим в бор все строки в **обратном** порядке символов. Тогда «заканчивается на q» превращается в «начинается с reverse(q)» — и это обычный запрос по префиксу.

Обратим строки:
- "apple" → "elppa"
- "maple" → "elpam"
- "staple" → "elpats"
- "table" → "elbat"
- "cable" → "elbac"

Запрос: reverse("ple") = "elp".

Вставляем в бор: "elppa", "elpam", "elpats", "elbat", "elbac".

Узлы на пути "elp": root → e → l → p (общая часть для "elppa", "elpam", "elpats").

Из узла p расходятся: ветка для "pa" (→"elppa"), "am" (→"elpam"), "ats" (→"elpats").

Три слова начинаются с "elp", значит три слова оригинального словаря заканчиваются на "ple": **"apple", "maple", "staple"**.

Проверка: "table" заканчивается на "ble", "cable" — на "ble". Не подходят. ✓

### Ответ

3 слова: "apple", "maple", "staple". Ключевой приём: вставить строки в бор в обратном порядке и искать prefix = reverse(suffix).

## Задача 10. Синтез: КМП + СНМ для группировки строк по образцу

Даны $k$ строк $s_1, \ldots, s_k$. Будем считать строки $s_i$ и $s_j$ *родственными*, если $s_i$ является подстрокой $s_j$ или наоборот. Необходимо сгруппировать строки в компоненты связности отношения родственности (транзитивное замыкание).

Дано: $s_1=\texttt{"AB"}$, $s_2=\texttt{"ABABC"}$, $s_3=\texttt{"C"}$, $s_4=\texttt{"CABABD"}$, $s_5=\texttt{"XY"}$.

1. Для каждой пары определите родственность через КМП.
2. Постройте компоненты связности через СНМ.

### Решение

Всего $\binom{5}{2} = 10$ пар. Для каждой запускаем КМП в обоих направлениях.

**Проверяем пары:**

- (s1,s2): "AB" ⊂ "ABABC"? Да (позиции 0,2). **Родственны.**
- (s1,s3): "AB" ⊂ "C"? Нет. "C" ⊂ "AB"? Нет. Не родственны.
- (s1,s4): "AB" ⊂ "CABABD"? Да (позиция 2). **Родственны.**
- (s1,s5): "AB" ⊂ "XY"? Нет. "XY" ⊂ "AB"? Нет. Не родственны.
- (s2,s3): "ABABC" ⊂ "C"? Нет. "C" ⊂ "ABABC"? Да (позиция 4). **Родственны.**
- (s2,s4): "ABABC" ⊂ "CABABD"? Проверяем: C-A-B-A-B-D vs ABABC. Нет. "CABABD" ⊂ "ABABC"? Нет. Не родственны.
- (s2,s5): нет в обе стороны. Не родственны.
- (s3,s4): "C" ⊂ "CABABD"? Да (позиция 0). **Родственны.**
- (s3,s5): нет. Не родственны.
- (s4,s5): нет. Не родственны.

**Рёбра родственности**: (1,2), (1,4), (2,3), (3,4).

**СНМ на индексах {1,2,3,4,5}:**

- Union(1,2): {1,2}, {3}, {4}, {5}
- Union(1,4): find(1)=find(4)? find(4)=4 ≠ find(1)=1 → unite → {1,2,4}, {3}, {5}
- Union(2,3): find(2)=1, find(3)=3 → unite → {1,2,3,4}, {5}
- Union(3,4): find(3)=1, find(4)=1 → уже в одной компоненте.

**Результат:**

- Компонента 1: {s1, s2, s3, s4} — все связаны через отношение родственности.
- Компонента 2: {s5} — изолирована.

### Ответ

Две компоненты: **{AB, ABABC, C, CABABD}** и **{XY}**. Алгоритм: $O(k^2 \cdot L)$ где $L$ — максимальная длина строки (КМП для каждой пары) плюс $O(k \cdot \alpha(k))$ для СНМ.

<details><summary>Что тренируют эти задачи</summary>

1. Ручная трассировка DSU с рангами — понимание механики объединения.
2. Вычисление префикс-функции по шагам — обязательный навык для экзамена.
3. Поиск всех вхождений через КМП — понимание прыжка j=π[j-1].

5. Онлайн-компоненты связности — классическое применение DSU.
6. Обнаружение цикла — ключевой приём: цикл ↔ Find(u)=Find(v).
7. Счёт узлов бора как счёт уникальных префиксов — концептуальное понимание.
8. Крускал + DSU — связка для MST, часто встречается на олимпиадах.
9. Запрос на суффикс через обратный бор — нестандартное применение.
10. Синтез КМП и DSU для транзитивного замыкания — задача олимпийского уровня.
