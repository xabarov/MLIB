## Задача 1. Существование топологической сортировки

Для каждого из следующих графов определите, существует ли топологическая сортировка. Если да — приведите её; если нет — укажите цикл.

**Граф A:** вершины $\{1,2,3,4\}$, рёбра: $1 \to 2$, $1 \to 3$, $2 \to 4$, $3 \to 4$.

**Граф B:** вершины $\{1,2,3\}$, рёбра: $1 \to 2$, $2 \to 3$, $3 \to 1$.

**Граф C:** вершины $\{1,2,3,4,5\}$, рёбра: $1 \to 3$, $2 \to 3$, $3 \to 4$, $4 \to 5$, $2 \to 5$.

### Решение

**Граф A.** Проверяем наличие циклов. In-degrees: $d[1]=0$, $d[2]=1$, $d[3]=1$, $d[4]=2$. Запускаем Кана:
- Очередь: $\{1\}$. Берём 1, in-deg[2]--, in-deg[3]--. Очередь: $\{2, 3\}$.
- Берём 2, in-deg[4]--. Очередь: $\{3\}$.
- Берём 3, in-deg[4]--. In-deg[4]=0. Очередь: $\{4\}$.
- Берём 4. Результат: $[1, 2, 3, 4]$.

Все 4 вершины в результате → цикла нет. Топологический порядок: **1, 2, 3, 4** (также корректен порядок 1, 3, 2, 4).

**Граф B.** In-degrees: $d[1]=1$, $d[2]=1$, $d[3]=1$. Ни у одной вершины in-degree не равен 0! Алгоритм Кана вернёт пустой результат. Цикл: $1 \to 2 \to 3 \to 1$. Топологической сортировки не существует.

**Граф C.** In-degrees: $d[1]=0$, $d[2]=0$, $d[3]=2$, $d[4]=1$, $d[5]=2$. Очередь: $\{1,2\}$.
- Берём 1: in-deg[3]--=1. Берём 2: in-deg[3]--=0, in-deg[5]--=1.
- Очередь: $\{3\}$. Берём 3: in-deg[4]--=0.
- Берём 4: in-deg[5]--=0. Берём 5. Результат: $[1,2,3,4,5]$.

Топологический порядок: **1, 2, 3, 4, 5**.

### Ответ
- A: топологическая сортировка существует, например $[1,2,3,4]$.
- B: топологической сортировки нет, цикл $1 \to 2 \to 3 \to 1$.
- C: топологическая сортировка существует, например $[1,2,3,4,5]$.

## Задача 2. Трассировка DFS-алгоритма

Дан граф с вершинами $\{A, B, C, D, E\}$ и рёбрами: $A \to B$, $A \to C$, $B \to D$, $C \to D$, $D \to E$.

Выполните DFS-топологическую сортировку начиная с вершины A (соседей обходить в алфавитном порядке). Заполните таблицу времён входа ($t_{in}$) и выхода ($t_{out}$) для каждой вершины. Восстановите топологический порядок.

### Решение

Трассируем DFS (таймер начинается с 1):

| Шаг | Действие | Таймер |
|---|---|---|
| 1 | Вход в A | $t_{in}[A]=1$ |
| 2 | Вход в B (первый сосед A) | $t_{in}[B]=2$ |
| 3 | Вход в D (первый сосед B) | $t_{in}[D]=3$ |
| 4 | Вход в E (сосед D) | $t_{in}[E]=4$ |
| 5 | Выход из E | $t_{out}[E]=5$ |
| 6 | Выход из D | $t_{out}[D]=6$ |
| 7 | Выход из B | $t_{out}[B]=7$ |
| 8 | Вход в C (следующий сосед A) | $t_{in}[C]=8$ |
| 9 | D уже BLACK — пропускаем | — |
| 10 | Выход из C | $t_{out}[C]=9$ |
| 11 | Выход из A | $t_{out}[A]=10$ |

Итоговая таблица:

| Вершина | $t_{in}$ | $t_{out}$ |
|---|---|---|
| A | 1 | 10 |
| B | 2 | 7 |
| C | 8 | 9 |
| D | 3 | 6 |
| E | 4 | 5 |

Сортируем по убыванию $t_{out}$: $A(10) > B(7) > C(9)$... Правильный порядок: $A(10), C(9), B(7), D(6), E(5)$.

Топологический порядок: **A, C, B, D, E**.

Проверка: для каждого ребра предшественник стоит раньше потомка.
- $A \to B$: A перед B ✓
- $A \to C$: A перед C ✓
- $B \to D$: B перед D ✓
- $C \to D$: C перед D ✓
- $D \to E$: D перед E ✓

### Ответ
Топологический порядок: $A, C, B, D, E$. Таблица времён: $t_{out} = [A:10, C:9, B:7, D:6, E:5]$.

## Задача 3. Количество топологических сортировок

Подсчитайте количество различных топологических сортировок DAG с вершинами $\{1,2,3,4\}$ и рёбрами $1 \to 3$, $2 \to 3$, $3 \to 4$.

### Решение

Вершина 4 всегда стоит последней (зависит от 3, которая зависит от 1 и 2).
Вершина 3 стоит на третьей позиции (после 1 и 2, перед 4).
Вершины 1 и 2 не связаны рёбрами — их можно расставить в любом порядке из двух.

Все корректные топологические сортировки:
1. $[1, 2, 3, 4]$
2. $[2, 1, 3, 4]$

Итого: **2** различные топологические сортировки.

**Альтернативный подсчёт через формулу.** У нас 2 независимых «стартовых» вершины (1 и 2), затем обязательно 3, затем 4. Количество перестановок независимых вершин на начальных позициях: $2! = 2$.

### Ответ
2 топологические сортировки: $[1,2,3,4]$ и $[2,1,3,4]$.

## Задача 4. Обнаружение цикла

Реализуйте функцию `bool has_cycle(int n, vector<vector<int>>& adj)`, которая возвращает `true`, если в ориентированном графе есть цикл. Алгоритм должен работать за $O(V+E)$.

Проверьте на входе: $n=4$, рёбра: $0\to1$, $1\to2$, $2\to3$, $3\to1$.

### Решение

Используем трёхцветный DFS. Вершина GRAY означает, что она находится в текущем пути DFS. Ребро, ведущее в GRAY-вершину, является обратным и означает цикл.

```cpp
#include <vector>

const int WHITE = 0, GRAY = 1, BLACK = 2;

bool dfs_cycle(int v, std::vector<int>& color,
               const std::vector<std::vector<int>>& adj) {
    color[v] = GRAY;
    for (int to : adj[v]) {
        if (color[to] == GRAY) return true;   // back edge = цикл
        if (color[to] == WHITE) {
            if (dfs_cycle(to, color, adj)) return true;
        }
    }
    color[v] = BLACK;
    return false;
}

bool has_cycle(int n, const std::vector<std::vector<int>>& adj) {
    std::vector<int> color(n, WHITE);
    for (int v = 0; v < n; ++v) {
        if (color[v] == WHITE) {
            if (dfs_cycle(v, color, adj)) return true;
        }
    }
    return false;
}
```

**Трассировка на примере** ($n=4$, рёбра $0\to1$, $1\to2$, $2\to3$, $3\to1$):

| Шаг | v | color[v] | Действие |
|---|---|---|---|
| 1 | 0 | GRAY | Вход в 0 |
| 2 | 1 | GRAY | Вход в 1 (из 0) |
| 3 | 2 | GRAY | Вход в 2 (из 1) |
| 4 | 3 | GRAY | Вход в 3 (из 2) |
| 5 | — | — | Сосед 3→1: color[1]=GRAY → цикл! |

Функция возвращает `true`. Цикл: $1 \to 2 \to 3 \to 1$.

**Почему `bool visited[]` не подходит?** Если вершина $v$ уже посещена и `visited[v]=true`, мы не можем отличить: она завершила обработку (BLACK, цикла нет) или всё ещё в стеке (GRAY, цикл есть). Трёхцветная раскраска решает эту проблему.

### Ответ
Функция `has_cycle` возвращает `true`. Цикл: $1 \to 2 \to 3 \to 1$.

## Задача 5. Лексикографически минимальная топологическая сортировка

Дан DAG с $n$ вершинами. Найдите лексикографически минимальную топологическую сортировку (то есть такую, которая минимальна при побуквенном сравнении).

**Пример:** $n=4$, рёбра $1\to2$, $3\to2$, $3\to4$. Найти лексикографически минимальный порядок.

### Решение

**Идея.** Модифицируем алгоритм Кана: вместо обычной очереди используем **приоритетную очередь (min-heap)**. Тогда среди всех вершин с in-degree=0 мы всегда будем выбирать вершину с наименьшим номером.

```cpp
#include <vector>
#include <queue>

std::vector<int> lex_min_topo(int n,
                               const std::vector<std::vector<int>>& adj) {
    std::vector<int> in_degree(n + 1, 0);
    for (int u = 1; u <= n; ++u)
        for (int v : adj[u])
            in_degree[v]++;

    // min-heap вместо обычной очереди
    std::priority_queue<int, std::vector<int>, std::greater<int>> pq;
    for (int v = 1; v <= n; ++v)
        if (in_degree[v] == 0)
            pq.push(v);

    std::vector<int> result;
    while (!pq.empty()) {
        int u = pq.top(); pq.pop();
        result.push_back(u);
        for (int v : adj[u]) {
            if (--in_degree[v] == 0)
                pq.push(v);
        }
    }
    return result;
}
```

**Трассировка для примера** ($n=4$, рёбра $1\to2$, $3\to2$, $3\to4$, вершины 1..4):

In-degrees: $d[1]=0$, $d[2]=2$, $d[3]=0$, $d[4]=1$.

| Шаг | Heap | Берём | Действие | In-deg обновлён |
|---|---|---|---|---|
| 0 | {1, 3} | — | Начало | — |
| 1 | {1, 3} | 1 | Добавляем 1 в результат | in-deg[2]-- → 1 |
| 2 | {3} | 3 | Добавляем 3 в результат | in-deg[2]--→0, in-deg[4]--→0 |
| 3 | {2, 4} | 2 | Добавляем 2 в результат | — |
| 4 | {4} | 4 | Добавляем 4 в результат | — |

Результат: $[1, 3, 2, 4]$.

**Сложность.** $O((V+E) \log V)$ из-за операций с heap (против $O(V+E)$ для обычного Кана).

**Проверка.** Другие корректные топосортировки: $[3, 1, 2, 4]$, $[1, 3, 4, 2]$, $[3, 1, 4, 2]$. Лексикографически наименьшая — $[1, 3, 2, 4]$, так как $1 < 3$ на первой позиции. ✓

### Ответ
Лексикографически минимальная топологическая сортировка: $[1, 3, 2, 4]$. Алгоритм: Кан с min-heap, сложность $O((V+E) \log V)$.

## Задача 6. Длиннейший путь в DAG

Дан взвешенный DAG:
- Вершины: $\{0, 1, 2, 3, 4, 5\}$
- Рёбра: $0\to1$ (вес 2), $0\to2$ (вес 4), $1\to3$ (вес 3), $2\to3$ (вес 1), $2\to4$ (вес 5), $3\to5$ (вес 2), $4\to5$ (вес 1)

Найдите длиннейший путь от вершины 0 до вершины 5.

### Решение

**Шаг 1: топологическая сортировка.**

In-degrees: $d[0]=0$, $d[1]=1$, $d[2]=1$, $d[3]=2$, $d[4]=1$, $d[5]=2$.

Алгоритм Кана: начинаем с вершины 0.
- Берём 0: убавляем in-deg[1] и in-deg[2]. Оба = 0.
- Берём 1: убавляем in-deg[3]→1.
- Берём 2: убавляем in-deg[3]→0, in-deg[4]→0.
- Берём 3: убавляем in-deg[5]→1.
- Берём 4: убавляем in-deg[5]→0.
- Берём 5.

Топологический порядок: $[0, 1, 2, 3, 4, 5]$.

**Шаг 2: DP — длиннейший путь до каждой вершины.**

$dp[v] = \max_{u \to v}(dp[u] + w(u,v))$, $dp[0] = 0$.

| Вершина | Откуда | dp[v] |
|---|---|---|
| 0 | — | 0 |
| 1 | от 0: $dp[0]+2=2$ | 2 |
| 2 | от 0: $dp[0]+4=4$ | 4 |
| 3 | от 1: $dp[1]+3=5$; от 2: $dp[2]+1=5$ | 5 |
| 4 | от 2: $dp[2]+5=9$ | 9 |
| 5 | от 3: $dp[3]+2=7$; от 4: $dp[4]+1=10$ | **10** |

**Шаг 3: восстановление пути.**

Длиннейший путь до 5 длиной 10 достигается через вершину 4 ($dp[4]+1=10$). Вершина 4 достигнута через вершину 2 ($dp[2]+5=9$). Вершина 2 — из вершины 0.

Путь: $0 \to 2 \to 4 \to 5$, суммарный вес: $4 + 5 + 1 = 10$.

### Ответ
Длиннейший путь: $0 \to 2 \to 4 \to 5$ длиной **10**.

## Задача 7. Подсчёт числа путей

Дан DAG с вершинами $\{0,1,2,3,4\}$ и рёбрами $0\to1$, $0\to2$, $1\to3$, $2\to3$, $1\to4$, $3\to4$. Найдите количество различных путей от вершины 0 до вершины 4.

### Решение

Используем DP: $cnt[v]$ = количество путей от 0 до $v$.

**Инициализация:** $cnt[0] = 1$ (путь из 0 в 0 — тривиальный), $cnt[v] = 0$ для $v \neq 0$.

**Топологическая сортировка** (Кан): $[0, 1, 2, 3, 4]$.

**Рекуррентность:** при обработке ребра $u \to v$: $cnt[v] += cnt[u]$.

| Обрабатываем | Ребро | cnt до | cnt после |
|---|---|---|---|
| 0 | $0\to1$ | cnt[1]=0 | cnt[1]=1 |
| 0 | $0\to2$ | cnt[2]=0 | cnt[2]=1 |
| 1 | $1\to3$ | cnt[3]=0 | cnt[3]=1 |
| 1 | $1\to4$ | cnt[4]=0 | cnt[4]=1 |
| 2 | $2\to3$ | cnt[3]=1 | cnt[3]=2 |
| 3 | $3\to4$ | cnt[4]=1 | cnt[4]=3 |

Ответ: $cnt[4] = 3$.

**Проверка — перечислим все пути:**
1. $0 \to 1 \to 3 \to 4$
2. $0 \to 1 \to 4$
3. $0 \to 2 \to 3 \to 4$

Итого 3 пути. ✓

```cpp
#include <vector>
#include <queue>

int count_paths(int n, int src, int dst,
                const std::vector<std::vector<int>>& adj) {
    std::vector<int> in_degree(n, 0);
    for (int u = 0; u < n; ++u)
        for (int v : adj[u])
            in_degree[v]++;

    std::queue<int> q;
    for (int v = 0; v < n; ++v)
        if (in_degree[v] == 0) q.push(v);

    std::vector<long long> cnt(n, 0);
    cnt[src] = 1;

    while (!q.empty()) {
        int u = q.front(); q.pop();
        for (int v : adj[u]) {
            cnt[v] += cnt[u];
            if (--in_degree[v] == 0)
                q.push(v);
        }
    }
    return cnt[dst];
}
```

### Ответ
Количество путей от 0 до 4: **3**.

## Задача 8. Проверка корректности расписания

Вам дан список из $n$ курсов (пронумерованных $1..n$) и список пар $(a, b)$, означающих «курс $a$ нужно пройти до курса $b$». Напишите алгоритм, который определяет, возможно ли пройти все курсы, и если да — возвращает корректный порядок их прохождения.

**Это задача LeetCode #210 «Course Schedule II» — классика ШАД.**

**Пример:** $n=4$, зависимости: $(1,2)$, $(2,3)$, $(1,4)$, $(3,4)$. Найти порядок прохождения.

### Решение

Задача напрямую сводится к топологической сортировке: вершины — курсы, ребро $a \to b$ означает «$a$ должен быть пройден до $b$».

Цикл в графе зависимостей = невозможность пройти все курсы (circular dependency).

```cpp
#include <vector>
#include <queue>

std::vector<int> find_order(int n,
                             const std::vector<std::pair<int,int>>& prerequisites) {
    // prerequisites[i] = (b, a) означает "пройти a до b"
    // Строим ребро a -> b
    std::vector<std::vector<int>> adj(n + 1);
    std::vector<int> in_degree(n + 1, 0);

    for (auto [b, a] : prerequisites) {
        adj[a].push_back(b);
        in_degree[b]++;
    }

    std::queue<int> q;
    for (int v = 1; v <= n; ++v)
        if (in_degree[v] == 0) q.push(v);

    std::vector<int> order;
    while (!q.empty()) {
        int u = q.front(); q.pop();
        order.push_back(u);
        for (int v : adj[u]) {
            if (--in_degree[v] == 0)
                q.push(v);
        }
    }

    if ((int)order.size() < n) return {}; // Цикл — невозможно
    return order;
}
```

**Трассировка для примера** ($n=4$, зависимости $(1,2)$, $(2,3)$, $(1,4)$, $(3,4)$):

Рёбра: $2\to1$, $3\to2$, $4\to1$, $4\to3$.

Стоп — интерпретируем $(a,b)$ как «$a$ нужно до $b$», значит ребро $a \to b$:
Рёбра: $1\to2$, $2\to3$, $1\to4$, $3\to4$.

In-degrees: $d[1]=0$, $d[2]=1$, $d[3]=1$, $d[4]=2$.

- Очередь: $\{1\}$. Берём 1: in-deg[2]--=0, in-deg[4]--=1.
- Очередь: $\{2\}$. Берём 2: in-deg[3]--=0.
- Очередь: $\{3\}$. Берём 3: in-deg[4]--=0.
- Очередь: $\{4\}$. Берём 4.

Результат: $[1, 2, 3, 4]$. Все 4 курса — проходимы.

**Что если добавить зависимость $(2, 1)$ к примеру?** Тогда рёбра $1\to2$ и $2\to1$ образуют цикл. Алгоритм вернёт пустой список — курсы пройти невозможно.

### Ответ
Порядок прохождения курсов: $[1, 2, 3, 4]$. Алгоритм — топологическая сортировка (Кан), $O(V+E)$.

## Задача 9. Минимальное количество «рёбер разрыва» для устранения всех циклов

Дан ориентированный граф $G$. Назовём множество рёбер $F$ *минимальным обратным множеством рёбер* (minimum feedback arc set), если удаление рёбер из $F$ делает граф ациклическим, и $|F|$ минимально.

Докажите, что задача нахождения feedback arc set является NP-трудной (без полного формального доказательства: объясните, к чему она сводится). Предложите эффективную эвристику.

### Решение

**Сводимость к NP-трудной задаче.** Задача нахождения минимального feedback arc set (MFAS) является NP-трудной: она эквивалентна задаче *Maximum Acyclic Subgraph*, которая NP-трудна. Это доказывается сведением от задачи MAX-CUT: если бы мы умели решать MFAS за полиномиальное время, мы могли бы решать MAX-CUT.

Интуиция: в худшем случае граф может содержать $\Theta(V^2)$ рёбер, образующих сложно переплетённые циклы, и нет известного полиномиального алгоритма выбора минимального числа рёбер для разрыва всех циклов.

**Эффективная эвристика — метод «топологической сортировки»:**

Идея: нашим «естественным» порядком вершин является произвольная перестановка. Ребро «идёт вперёд», если $\pi(u) < \pi(v)$, и «назад» иначе. Минимизируем количество обратных рёбер.

```cpp
#include <vector>
#include <algorithm>

// Жадная эвристика: GR (greedy algorithm)
// Повторяем: находим исток (в очередь с in-deg=0) или сток (out-deg=0)
// Добавляем к соответствующему концу результата
std::vector<int> greedy_fas_heuristic(int n,
                                       const std::vector<std::vector<int>>& adj) {
    std::vector<int> in_deg(n, 0), out_deg(n, 0);
    for (int u = 0; u < n; ++u) {
        out_deg[u] = adj[u].size();
        for (int v : adj[u]) in_deg[v]++;
    }

    std::vector<int> left, right;
    std::vector<bool> removed(n, false);

    for (int step = 0; step < n; ++step) {
        // Добавляем все стоки в right (справа)
        for (int v = 0; v < n; ++v)
            if (!removed[v] && out_deg[v] == 0)
                right.push_back(v), removed[v] = true;
        // Добавляем все истоки в left (слева)
        for (int v = 0; v < n; ++v)
            if (!removed[v] && in_deg[v] == 0)
                left.push_back(v), removed[v] = true;
        // Если остались вершины — берём с max(out_deg - in_deg)
        int best = -1;
        for (int v = 0; v < n; ++v)
            if (!removed[v])
                if (best == -1 ||
                    out_deg[v] - in_deg[v] > out_deg[best] - in_deg[best])
                    best = v;
        if (best != -1) {
            left.push_back(best);
            removed[best] = true;
        }
    }

    // Объединяем: left + right (reversed)
    std::reverse(right.begin(), right.end());
    for (int v : right) left.push_back(v);
    return left; // Порядок вершин; обратные рёбра = FAS
}
```

**Гарантия качества эвристики.** Алгоритм «GR» гарантирует, что количество прямых рёбер в полученном порядке не менее $m/2$ (где $m$ — число рёбер). Следовательно, количество обратных рёбер не превышает $m/2$. Оптимальное значение MFAS может быть существенно меньше, но эта эвристика — эффективная и практически применимая.

### Ответ
Задача MFAS NP-трудна (сводится от MAX-CUT / Maximum Acyclic Subgraph). Жадная эвристика GR даёт гарантию не более $m/2$ обратных рёбер, что является 2-приближением для максимального ациклического подграфа.

## Задача 10. Топологическая сортировка с ограничениями (синтез)

Дан DAG и набор дополнительных ограничений вида «вершина $u$ должна идти непосредственно перед вершиной $v$» (то есть в итоговом топологическом порядке между $u$ и $v$ нет других вершин). Постройте топологическую сортировку, удовлетворяющую всем ограничениям, или определите, что это невозможно.

**Пример:** $n=5$, рёбра DAG: $1\to3$, $2\to4$, $3\to5$, $4\to5$. Ограничение: вершина 3 сразу после вершины 2 (то есть $2, 3$ — последовательные элементы).

### Решение

**Анализ сложности задачи.** Это существенно более сложная задача, чем стандартная топологическая сортировка. Введём понятие «цепочки»: если $u$ должна идти сразу перед $v$, мы «склеиваем» $u$ и $v$ в один суперузел $[u, v]$.

**Алгоритм (сжатие цепочек):**

1. Для каждой пары «немедленной последовательности» $(u, v)$ объединяем их в суперузел. Суперузел представляет последовательный блок вершин.

2. Обновляем граф: все рёбра, входящие в $u$ или $v$, теперь входят в суперузел; все рёбра, исходящие из $u$ или $v$, исходят из суперузла.

3. Проверяем, что объединение не создаёт противоречий (суперузел не может иметь ребро сам в себя).

4. Запускаем топологическую сортировку на сжатом графе. Каждый суперузел разворачиваем в исходном порядке.

**Разбор примера:**

Ограничение: 3 сразу после 2. Создаём суперузел $S = [2, 3]$.

Обновлённые рёбра:
- $1 \to S$ (было $1\to3$, а 3 теперь в $S$, но 2 тоже в $S$, и входящее ребро 2 — нет входящих из других вершин)
- $S \to 5$ (было $3\to5$)
- $2\to4$ становится $S\to4$

Граф суперузлов: вершины $\{1, S, 4, 5\}$, рёбра: $1\to S$, $S\to5$, $S\to4$, $4\to5$.

Проверка: нет цикла в суперузле (2 и 3 не связаны ребром 3→2).

Топосортировка суперузлов: $[1, S, 4, 5]$.

Разворачиваем $S = [2, 3]$:

Финальный порядок: **$[1, 2, 3, 4, 5]$**.

**Проверка:**
- $1\to3$: 1 перед 3 ✓
- $2\to4$: 2 перед 4 ✓
- $3\to5$: 3 перед 5 ✓
- $4\to5$: 4 перед 5 ✓
- Ограничение: 2 непосредственно перед 3 (позиции 2 и 3 подряд) ✓

**Когда задача неразрешима?**
- Ограничение «$u$ сразу перед $v$» конфликтует с другим ограничением «$w$ сразу перед $v$» (два предшественника на немедленную позицию).
- Ограничение требует, чтобы $u$ шла сразу перед $v$, но $u$ и $v$ уже разделены обязательной вершиной по рёбрам DAG.
- Склеивание создаёт цикл в суперузловом графе.

```cpp
#include <vector>
#include <unordered_map>
#include <queue>

// Упрощённая версия: объединение цепочек через Union-Find,
// с последующей топосортировкой суперузлов
struct Chain {
    int head, tail;               // первая и последняя вершина цепочки
    std::vector<int> sequence;    // упорядоченный список вершин
};

std::vector<int> topo_with_constraints(
    int n,
    const std::vector<std::pair<int,int>>& edges,
    const std::vector<std::pair<int,int>>& immediate  // (u,v): u сразу перед v
) {
    // Для каждой вершины: чей она "next" и "prev"
    std::vector<int> next(n + 1, -1), prev_v(n + 1, -1);
    for (auto [u, v] : immediate) {
        if (next[u] != -1 || prev_v[v] != -1) return {}; // конфликт
        next[u] = v;
        prev_v[v] = u;
    }

    // Строим цепочки
    std::vector<int> chain_id(n + 1, -1);
    std::vector<Chain> chains;

    for (int v = 1; v <= n; ++v) {
        if (prev_v[v] == -1) {  // голова цепочки
            Chain c;
            int cur = v;
            while (cur != -1) {
                c.sequence.push_back(cur);
                chain_id[cur] = chains.size();
                cur = next[cur];
            }
            c.head = c.sequence.front();
            c.tail = c.sequence.back();
            chains.push_back(c);
        }
    }

    int m = chains.size();
    std::vector<std::vector<int>> cadj(m);
    std::vector<int> cin_deg(m, 0);

    for (auto [u, v] : edges) {
        int cu = chain_id[u], cv = chain_id[v];
        if (cu == cv) continue; // внутреннее ребро цепочки
        cadj[cu].push_back(cv);
        cin_deg[cv]++;
    }

    // Топосортировка цепочек
    std::queue<int> q;
    for (int i = 0; i < m; ++i)
        if (cin_deg[i] == 0) q.push(i);

    std::vector<int> result;
    int processed = 0;
    while (!q.empty()) {
        int ci = q.front(); q.pop();
        processed++;
        for (int v : chains[ci].sequence)
            result.push_back(v);
        for (int cj : cadj[ci])
            if (--cin_deg[cj] == 0)
                q.push(cj);
    }

    if (processed < m) return {}; // цикл
    return result;
}
```

### Ответ
Порядок $[1, 2, 3, 4, 5]$ удовлетворяет всем ограничениям. Алгоритм: сжатие цепочек из ограничений «немедленной последовательности», затем топологическая сортировка суперузлов, $O(V + E)$.
