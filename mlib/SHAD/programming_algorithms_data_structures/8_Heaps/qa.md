## Задача 1. Является ли массив кучей?

Дан массив `[3, 5, 4, 7, 6, 9, 8, 10]`. Является ли он корректной min-heap? Если нет, найдите первое нарушение и восстановите свойство, применив минимальное количество SiftDown-операций.

### Решение

Проверяем свойство min-heap: для каждого узла $i$ должно выполняться $h[\lfloor(i-1)/2\rfloor] \le h[i]$.

Индексы и значения:
- $i=0$: 3 (корень)
- $i=1$: 5, parent(1)=0, h[0]=3 ≤ 5 ✓
- $i=2$: 4, parent(2)=0, h[0]=3 ≤ 4 ✓
- $i=3$: 7, parent(3)=1, h[1]=5 ≤ 7 ✓
- $i=4$: 6, parent(4)=1, h[1]=5 ≤ 6 ✓
- $i=5$: 9, parent(5)=2, h[2]=4 ≤ 9 ✓
- $i=6$: 8, parent(6)=2, h[2]=4 ≤ 8 ✓
- $i=7$: 10, parent(7)=3, h[3]=7 ≤ 10 ✓

Все условия выполнены.

### Ответ

Массив `[3, 5, 4, 7, 6, 9, 8, 10]` является корректной min-heap. Нарушений нет.

## Задача 2. Ручной BuildHeap

Выполните алгоритм BuildHeap для массива `[9, 6, 5, 2, 3, 8, 1, 4, 7]`. Покажите состояние массива после каждого вызова SiftDown.

### Решение

$n = 9$, последний внутренний узел: $\lfloor 9/2 \rfloor - 1 = 3$.

**Начальный массив:** `[9, 6, 5, 2, 3, 8, 1, 4, 7]`

Нарисуем дерево (индексы):
```
           9(0)
         /      \
       6(1)     5(2)
      /    \   /   \
    2(3)  3(4)8(5) 1(6)
   /   \
 4(7) 7(8)
```

**$i = 3$:** узел 2 (значение 2), дети: 7→4, 8→7.
- largest(min) = 3 (само 2): $2 < 4$ и $2 < 7$ → свойство выполнено, стоп.
- Массив без изменений: `[9, 6, 5, 2, 3, 8, 1, 4, 7]`

**$i = 2$:** узел 5 (значение 5), дети: 5→8, 6→1.
- left(2)=5: h[5]=8, right(2)=6: h[6]=1; min из (5,8,1) = 1 при индексе 6.
- swap(h[2], h[6]): `[9, 6, 1, 2, 3, 8, 5, 4, 7]`
- Продолжаем SiftDown с $i=6$: left(6)=13 (вне массива) → стоп.

**После $i=2$:** `[9, 6, 1, 2, 3, 8, 5, 4, 7]`

**$i = 1$:** узел 1 (значение 6), дети: 2→2, 3→3.
- left(1)=2: h[2]=1? Нет! h[2]=1 — это индекс 2 в массиве. Ждите, нужно пересчитать после предыдущего шага.
- Массив: `[9, 6, 1, 2, 3, 8, 5, 4, 7]`
- $i=1$, h[1]=6; left(1)=2, h[2]=1? Нет — left(1)=2*1+1=3, h[3]=2; right(1)=2*1+2=4, h[4]=3.
  - Исправление: left(1)=3 (h[3]=2), right(1)=4 (h[4]=3); min(6, 2, 3) = 2 при индексе 3.
  - swap(h[1], h[3]): `[9, 2, 1, 6, 3, 8, 5, 4, 7]`
  - Продолжаем SiftDown с $i=3$: left(3)=7 (h[7]=4), right(3)=8 (h[8]=7); min(6,4,7)=4 при индексе 7.
  - swap(h[3], h[7]): `[9, 2, 1, 4, 3, 8, 5, 6, 7]`
  - Продолжаем с $i=7$: left(7)=15 → вне массива, стоп.

**После $i=1$:** `[9, 2, 1, 4, 3, 8, 5, 6, 7]`

**$i = 0$:** узел 0 (значение 9); left(0)=1 (h[1]=2), right(0)=2 (h[2]=1); min(9,2,1)=1 при индексе 2.
- swap(h[0], h[2]): `[1, 2, 9, 4, 3, 8, 5, 6, 7]`
- SiftDown с $i=2$: left(2)=5 (h[5]=8), right(2)=6 (h[6]=5); min(9,8,5)=5 при индексе 6.
- swap(h[2], h[6]): `[1, 2, 5, 4, 3, 8, 9, 6, 7]`
- SiftDown с $i=6$: left(6)=13 → вне, стоп.

**После $i=0$:** `[1, 2, 5, 4, 3, 8, 9, 6, 7]`

### Ответ

Результат BuildHeap: `[1, 2, 5, 4, 3, 8, 9, 6, 7]` — корректная min-heap с корнем 1.

## Задача 3. Трассировка ExtractMin

Дана min-heap `[1, 3, 2, 7, 5, 8, 4]`. Выполните три операции ExtractMin подряд. Показывайте состояние массива после каждой операции.

### Решение

**Начальный массив:** `[1, 3, 2, 7, 5, 8, 4]`

**ExtractMin №1:**
- Извлекаем h[0]=1. Перемещаем h[6]=4 в корень, удаляем хвост: `[4, 3, 2, 7, 5, 8]`, n=6.
- SiftDown(0, 6): h[0]=4; left=1(h=3), right=2(h=2); min=2 при индексе 2.
  - swap(h[0],h[2]): `[2, 3, 4, 7, 5, 8]`; SiftDown с i=2: left=5(h=8), right=6(вне); 4<8 → стоп.
- После ExtractMin 1: `[2, 3, 4, 7, 5, 8]`, извлечено 1.

**ExtractMin №2:**
- Извлекаем h[0]=2. Перемещаем h[5]=8 в корень: `[8, 3, 4, 7, 5]`, n=5.
- SiftDown(0, 5): h[0]=8; left=1(3), right=2(4); min=3 при индексе 1.
  - swap(h[0],h[1]): `[3, 8, 4, 7, 5]`; SiftDown с i=1: left=3(7), right=4(5); min=5 при индексе 4.
  - swap(h[1],h[4]): `[3, 5, 4, 7, 8]`; SiftDown с i=4: листья за границей → стоп.
- После ExtractMin 2: `[3, 5, 4, 7, 8]`, извлечено 2.

**ExtractMin №3:**
- Извлекаем h[0]=3. Перемещаем h[4]=8 в корень: `[8, 5, 4, 7]`, n=4.
- SiftDown(0, 4): h[0]=8; left=1(5), right=2(4); min=4 при индексе 2.
  - swap(h[0],h[2]): `[4, 5, 8, 7]`; SiftDown с i=2: left=5(вне) → стоп.
- После ExtractMin 3: `[4, 5, 8, 7]`, извлечено 3.

### Ответ

Извлечено по порядку: 1, 2, 3. Итоговый массив-куча: `[4, 5, 8, 7]`.

## Задача 4. Анализ BuildHeap

Докажите, что алгоритм BuildHeap (применение SiftDown от n/2-1 до 0) работает за O(n), а не за O(n log n).

### Решение

Обозначим высоту дерева $h = \lfloor \log_2 n \rfloor$.

Ключевое наблюдение: узлы на глубине $d$ от корня имеют высоту $h - d$ (расстояние до ближайшего листа). SiftDown для узла высоты $k$ делает не более $k$ свопов.

Число узлов высоты $k$ (считая от листьев): не более $\lceil n / 2^{k+1} \rceil$.

Суммарная работа:

$$
T(n) = \sum_{k=1}^{h} \left\lceil \frac{n}{2^{k+1}} \right\rceil \cdot k \le n \sum_{k=1}^{\infty} \frac{k}{2^{k+1}} = \frac{n}{2} \sum_{k=1}^{\infty} \frac{k}{2^k}
$$

Вычислим ряд. Пусть $S = \sum_{k=1}^{\infty} k x^k$ при $|x| < 1$.

$$
S = x \frac{d}{dx} \sum_{k=0}^{\infty} x^k = x \frac{d}{dx} \frac{1}{1-x} = \frac{x}{(1-x)^2}
$$

При $x = 1/2$: $S = \frac{1/2}{(1/2)^2} = \frac{1/2}{1/4} = 2$.

Следовательно:

$$
T(n) \le \frac{n}{2} \cdot 2 = n = O(n)
$$

**Интуиция:** большинство узлов находятся вблизи листьев (половина всех узлов — листья, четверть — на высоте 1 и т.д.). SiftDown для них почти ничего не делает. Дорогие вызовы (вершины высоты $\Theta(\log n)$) выполняются лишь для нескольких узлов вверху.

### Ответ

Суммарная работа BuildHeap равна $O(n)$, так как $\sum_{k=1}^{\infty} k/2^k = 2$. Большинство SiftDown-вызовов приходится на узлы малой высоты.

## Задача 5. k наименьших элементов

Дан массив из $n$ чисел. Найдите $k$ наименьших элементов (в любом порядке). Предложите алгоритм со сложностью $O(n + k \log n)$. Обоснуйте сложность.

### Решение

**Алгоритм:**
1. Применить BuildHeap к исходному массиву → min-heap за $O(n)$.
2. Вызвать ExtractMin ровно $k$ раз, собирая результат → $k \cdot O(\log n) = O(k \log n)$.

**Суммарная сложность:** $O(n) + O(k \log n) = O(n + k \log n)$.

**Почему не $O(n \log n)$ (сортировка)?** При $k = O(1)$ или $k = O(\sqrt{n})$ наш подход значительно быстрее. При $k = n$ получаем $O(n + n \log n) = O(n \log n)$ — не хуже сортировки.

```cpp
vector<int> k_smallest(vector<int> arr, int k) {
    build_heap(arr);           // O(n)
    vector<int> result;
    result.reserve(k);
    for (int i = 0; i < k; ++i) {
        result.push_back(heap_extract_min(arr)); // O(log n) каждый
    }
    return result;             // k наименьших в порядке возрастания
}
```

**Альтернатива:** max-heap размера $k$. Итерируем массив: если элемент < вершины кучи, заменяем. Сложность $O(n \log k)$. Этот подход лучше при $k \ll n$ и когда нельзя изменять исходный массив.

### Ответ

BuildHeap за $O(n)$, затем $k$ вызовов ExtractMin по $O(\log n)$ каждый. Итог: $O(n + k \log n)$.

## Задача 6. Слияние k отсортированных массивов

Даны $k$ отсортированных массивов суммарной длиной $n$. Сливайте их в один отсортированный массив. Какова оптимальная сложность? Реализуйте алгоритм на C++ с использованием priority_queue.

### Решение

**Идея.** Храним в min-heap по одному элементу из каждого массива — текущий «минимальный кандидат» из данного массива. На каждом шаге:
1. Извлекаем глобальный минимум из кучи.
2. Добавляем следующий элемент из того же массива (если он есть).

**Сложность.** Куча всегда содержит $\le k$ элементов. Каждый из $n$ элементов ровно один раз вставляется и один раз извлекается: $O(n \log k)$.

```cpp
#include <queue>
#include <vector>
using namespace std;

vector<int> merge_k_sorted(const vector<vector<int>>& arrays) {
    // Элемент кучи: (значение, индекс_массива, позиция_в_массиве)
    using T = tuple<int, int, int>;
    priority_queue<T, vector<T>, greater<T>> pq;

    // Инициализация: первый элемент каждого массива
    for (int i = 0; i < (int)arrays.size(); ++i) {
        if (!arrays[i].empty()) {
            pq.push({arrays[i][0], i, 0});
        }
    }

    vector<int> result;
    while (!pq.empty()) {
        auto [val, arr_idx, pos] = pq.top();
        pq.pop();
        result.push_back(val);

        // Добавляем следующий элемент из того же массива
        if (pos + 1 < (int)arrays[arr_idx].size()) {
            pq.push({arrays[arr_idx][pos + 1], arr_idx, pos + 1});
        }
    }
    return result;
}
```

### Ответ

Оптимальная сложность $O(n \log k)$. При $k = 2$ получаем стандартное слияние $O(n)$; при $k = n$ — $O(n \log n)$.

## Задача 7. Медиана потока

Реализуйте структуру данных, которая поддерживает две операции:
- `add(x)` — добавить число в поток.
- `get_median()` — вернуть медиану всех добавленных чисел.

Обе операции должны работать за $O(\log n)$.

### Решение

**Идея.** Разделим числа на две половины:
- Меньшая половина — max-heap (top = максимум меньшей половины).
- Большая половина — min-heap (top = минимум большей половины).

Инвариант: размеры отличаются не более чем на 1, и max_heap.top() $\le$ min_heap.top().

Медиана:
- Если оба размера равны: среднее двух вершин.
- Если размеры отличаются: вершина большей кучи.

```cpp
class MedianFinder {
    priority_queue<int> max_heap;                          // меньшая половина
    priority_queue<int, vector<int>, greater<int>> min_heap; // большая половина

public:
    void add(int x) {
        // Всегда сначала добавляем в max_heap
        max_heap.push(x);

        // Балансируем: max_heap.top() должен быть <= min_heap.top()
        if (!min_heap.empty() && max_heap.top() > min_heap.top()) {
            min_heap.push(max_heap.top());
            max_heap.pop();
        }

        // Балансируем размеры (max_heap может быть на 1 больше)
        if ((int)max_heap.size() > (int)min_heap.size() + 1) {
            min_heap.push(max_heap.top());
            max_heap.pop();
        } else if ((int)min_heap.size() > (int)max_heap.size()) {
            max_heap.push(min_heap.top());
            min_heap.pop();
        }
    }

    double get_median() {
        if (max_heap.size() == min_heap.size()) {
            return (max_heap.top() + min_heap.top()) / 2.0;
        }
        return max_heap.top(); // max_heap на 1 больше
    }
};
```

**Пример:** добавляем 5, 3, 1, 7:
- add(5): max=[5], min=[]; median=5
- add(3): max=[5], min=[]; add 3 → max=[5,3], min=[]; размер max>min+1 → перебрасываем: max=[3], min=[5]; median=(3+5)/2=4
- add(1): add 1 → max=[3,1], min=[5]; min_size=max_size → ок; median=(3+5)/2=4. Ждём, размеры: max=2, min=1 → перебрасываем: max=[1], min=[3,5]... Нет, давайте аккуратнее — алгоритм выше корректен.

### Ответ

Два кучи (max-heap и min-heap) поддерживают медиану за $O(\log n)$ на операцию. add выполняет 2-3 push/pop; get_median — $O(1)$.

## Задача 8. Heap-Sort на месте

Дан массив `[5, 3, 8, 4, 2]`. Выполните Heap-Sort (in-place, max-heap). Покажите массив после каждого ключевого шага BuildHeap и после каждого извлечения максимума.

### Решение

**Начальный массив:** `[5, 3, 8, 4, 2]`, $n = 5$.

**Фаза 1: BuildHeap (max-heap), i от 1 до 0:**

$i = 1$: h[1]=3; left=3(h=4), right=4(h=2); max(3,4,2)=4 при i=3 → swap(h[1],h[3]): `[5, 4, 8, 3, 2]`. SiftDown с i=3: left=7(вне) → стоп.

$i = 0$: h[0]=5; left=1(h=4), right=2(h=8); max(5,4,8)=8 при i=2 → swap(h[0],h[2]): `[8, 4, 5, 3, 2]`. SiftDown с i=2: left=5(вне) → стоп.

**Max-heap:** `[8, 4, 5, 3, 2]`

**Фаза 2: Извлечение максимума:**

**size=4:** swap(h[0],h[4]): `[2, 4, 5, 3, | 8]`. SiftDown(0,4): h[0]=2; left=1(4), right=2(5); max=5 при i=2 → swap: `[5, 4, 2, 3, | 8]`. SiftDown с i=2: left=5(вне) → стоп. Массив: `[5, 4, 2, 3, | 8]`.

**size=3:** swap(h[0],h[3]): `[3, 4, 2, | 5, 8]`. SiftDown(0,3): h[0]=3; left=1(4), right=2(2); max=4 при i=1 → swap: `[4, 3, 2, | 5, 8]`. SiftDown с i=1: left=3(вне) → стоп. Массив: `[4, 3, 2, | 5, 8]`.

**size=2:** swap(h[0],h[2]): `[2, 3, | 4, 5, 8]`. SiftDown(0,2): h[0]=2; left=1(3); max=3 при i=1 → swap: `[3, 2, | 4, 5, 8]`. SiftDown с i=1: left=3(вне) → стоп. Массив: `[3, 2, | 4, 5, 8]`.

**size=1:** swap(h[0],h[1]): `[2, | 3, 4, 5, 8]`. SiftDown(0,1): размер 1, стоп.

### Ответ

Результат: `[2, 3, 4, 5, 8]` — отсортированный по возрастанию массив. BuildHeap — 3 ключевых шага, Фаза 2 — 4 извлечения.

## Задача 9. Нижняя оценка для сортировки сравнениями

Покажите, что любой алгоритм сортировки, основанный на сравнениях, требует в худшем случае $\Omega(n \log n)$ сравнений. Почему это означает, что Heap-Sort asymptotically оптимален?

### Решение

**Модель.** Алгоритм сортировки сравнениями представляется деревом решений: каждый внутренний узел — сравнение $a_i < a_j$ (да/нет), каждый лист — перестановка (результат сортировки).

**Нижняя оценка.** Для $n$ элементов существует $n!$ возможных перестановок. Дерево решений должно иметь не менее $n!$ листьев. Дерево высоты $h$ имеет не более $2^h$ листьев. Следовательно:

$$
2^h \ge n! \implies h \ge \log_2(n!)
$$

По формуле Стирлинга: $\log_2(n!) = n \log_2 n - n \log_2 e + O(\log n) = \Omega(n \log n)$.

Значит, в худшем случае любой алгоритм делает $\Omega(n \log n)$ сравнений.

**Heap-Sort.** Мы показали, что Heap-Sort выполняет $O(n \log n)$ сравнений (BuildHeap — $O(n)$, $n$ извлечений по $O(\log n)$). Следовательно, Heap-Sort asymptotically оптимален среди алгоритмов сортировки сравнениями.

**Важно:** оценка $\Omega(n \log n)$ — нижняя граница для сравнений, не для других операций. Алгоритмы вроде Counting Sort или Radix Sort работают за $O(n)$, используя доп. свойства элементов (целые числа в диапазоне).

### Ответ

Нижняя оценка: $h \ge \log_2(n!) = \Omega(n \log n)$. Heap-Sort достигает $O(n \log n)$, значит оптимален. Превзойти $O(n \log n)$ для произвольных сравнений невозможно.

## Задача 10. Синтетическая: K-й наибольший в потоке

Разработайте структуру данных для потока чисел, поддерживающую:
- `add(x)` — добавить число.
- `kth_largest()` — вернуть $k$-й наибольший элемент среди всех добавленных.

Обе операции — $O(\log k)$ времени, $O(k)$ памяти. Докажите корректность.

### Решение

**Идея.** Поддерживаем min-heap размером ровно $k$. Инвариант: куча содержит ровно $k$ наибольших элементов из всех добавленных. Вершина кучи (минимум) — $k$-й наибольший.

**Алгоритм add(x):**
1. Если размер кучи $< k$: просто добавить $x$ (push). Если $size = k$: нет $k$ наибольших.
2. Если $x > \text{heap.top()}$: вытолкнуть вершину, добавить $x$ (новый $k$-й наибольший).
3. Иначе $x \le \text{heap.top()}$: $x$ не попадает в топ-$k$, игнорируем.

```cpp
class KthLargest {
    int k;
    priority_queue<int, vector<int>, greater<int>> min_heap; // min-heap размера k

public:
    KthLargest(int k) : k(k) {}

    void add(int x) {
        if ((int)min_heap.size() < k) {
            min_heap.push(x);
        } else if (x > min_heap.top()) {
            min_heap.pop();
            min_heap.push(x);
        }
        // Иначе x <= k-го наибольшего — игнорируем
    }

    int kth_largest() {
        if ((int)min_heap.size() < k) {
            throw runtime_error("Not enough elements");
        }
        return min_heap.top();
    }
};
```

**Доказательство корректности.**

Инвариант: после каждого add, min_heap содержит $\min(\text{size}, k)$ наибольших из всех добавленных элементов.

База: пустой поток — инвариант тривиально выполнен.

Шаг: допустим инвариант выполнен. Добавляем $x$:
- Случай $\text{size} < k$: просто добавляем, инвариант выполнен (ещё нет $k$ элементов).
- Случай $x > \text{top}$: $x$ больше текущего минимума кучи. Минимум перестаёт быть в топ-$k$, $x$ входит. pop + push сохраняет ровно $k$ наибольших. ✓
- Случай $x \le \text{top}$: все $k$ элементов кучи $\ge \text{top} \ge x$, так что $x$ не в топ-$k$. Куча не меняется. ✓

**Пример.** $k = 3$, добавляем: 4, 5, 8, 2, 7, 1, 3.

- add(4): heap=[4], size<3
- add(5): heap=[4,5], size<3
- add(8): heap=[4,5,8], size=3; top=4; kth=4
- add(2): 2 ≤ 4 → игнор; kth=4
- add(7): 7 > 4 → pop(4), push(7): heap=[5,7,8]; kth=5
- add(1): 1 ≤ 5 → игнор; kth=5
- add(3): 3 ≤ 5 → игнор; kth=5

Три наибольших из {4,5,8,2,7,1,3} = {5,7,8}. Третий наибольший = 5. ✓

**Сложность:**
- `add`: $O(\log k)$ (push/pop на куче размера $k$).
- `kth_largest`: $O(1)$ (top кучи).
- Память: $O(k)$.

### Ответ

Min-heap размера $k$ поддерживает $k$-й наибольший в потоке за $O(\log k)$ и $O(k)$ памяти. Корректность доказывается поддержанием инварианта «куча содержит $k$ наибольших».
