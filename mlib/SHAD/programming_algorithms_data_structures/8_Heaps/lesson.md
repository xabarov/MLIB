# Лекция 8: Кучи и сортировка кучей

![Кучи и сортировка кучей](assets/hero_heaps.png)

Структура данных «куча» — один из фундаментальных инструментов алгоритмиста: она стоит за алгоритмом Дейкстры, кодированием Хаффмана, планировщиками задач в операционных системах и, конечно, за одним из немногих сортировочных алгоритмов с гарантированным $O(n \log n)$ и работой на месте. На вступительном экзамене в ШАД задачи на кучи встречаются регулярно — либо напрямую (реализуй ExtractMin), либо косвенно (спроектируй структуру данных с нужными временными характеристиками). Эта лекция строится на понимании деревьев из лекций 1–4 и сортировок из лекции 5, завершая картину эффективных структур данных.

**Главная линия лекции:**

$$
\text{Очередь с приоритетом} \to \text{Бинарная куча (массив)} \to \text{SiftDown / SiftUp} \to \text{Insert / ExtractMin} \to \text{BuildHeap за } O(n) \to \text{Heap-Sort}
$$

**Как читать эту лекцию:**
- Разделы 1–2: понять абстракцию и представление в памяти — без этого всё остальное бессмысленно.
- Разделы 3–5: ключевые операции; прочитать код, закрыть и написать по памяти.
- Раздел 6: разобрать анализ $O(n)$ — его часто просят доказать на экзамене.
- Раздел 7: Heap-Sort; важно понять, почему нужен **максимальная** куча для сортировки по возрастанию.
- Разделы 8–12: практика, типичные ошибки, вопросы.

## 1. Очередь с приоритетом — абстрактный тип данных

**Определение.** Очередь с приоритетом (Priority Queue) — абстрактный тип данных, поддерживающий следующие операции:

- **Insert(x)** — добавить элемент $x$.
- **GetMin()** — вернуть минимальный элемент (не удаляя).
- **ExtractMin()** — извлечь и вернуть минимальный элемент.

Иногда добавляют **DecreaseKey(x, k)** (уменьшить ключ элемента) и **Delete(x)**.

**Почему это важно?** Многие «жадные» алгоритмы нуждаются в структуре, которая быстро отдаёт наименьший элемент из динамически изменяющегося набора:

| Алгоритм | Зачем нужна очередь с приоритетом |
|---|---|
| Дейкстра (кратчайшие пути) | Всегда обрабатывать вершину с минимальным расстоянием |
| Хаффман (сжатие) | Объединять два дерева с минимальными весами |
| Планировщик событий | Выполнять событие с ближайшим временем |
| Сортировка слиянием k лент | Выбирать минимальный из k текущих элементов |

**Наивные реализации и их сложность:**

| Реализация | Insert | ExtractMin | GetMin |
|---|---|---|---|
| Несортированный массив | $O(1)$ | $O(n)$ | $O(n)$ |
| Отсортированный массив | $O(n)$ | $O(1)$ | $O(1)$ |
| Бинарная куча | $O(\log n)$ | $O(\log n)$ | $O(1)$ |

Бинарная куча достигает баланса между всеми операциями.

## 2. Бинарная куча: дерево и массив

**Определение.** Бинарная минимальная куча (min-heap) — это **полное бинарное дерево**, в котором для каждого узла выполняется **свойство кучи**:

$$
\text{key}(\text{parent}) \le \text{key}(\text{child}) \quad \text{для всех узлов}
$$

**Полное бинарное дерево** — все уровни заполнены, кроме, возможно, последнего, который заполняется слева.

**Хранение в массиве.** Полное бинарное дерево удобно хранить в массиве (нумерация с 0):

$$
\text{parent}(i) = \left\lfloor \frac{i-1}{2} \right\rfloor, \quad \text{left}(i) = 2i+1, \quad \text{right}(i) = 2i+2
$$

**Пример.** Куча `[1, 3, 5, 10, 4]`:

```
        1          ← индекс 0
       / \
      3   5        ← индексы 1, 2
     / \
    10   4         ← индексы 3, 4
```

Массив: `[1, 3, 5, 10, 4]`

Соответствие:
- узел 0 (значение 1): левый потомок — индекс 1 (значение 3), правый — индекс 2 (значение 5)
- узел 1 (значение 3): левый потомок — индекс 3 (значение 10), правый — индекс 4 (значение 4)
- узел 2 (значение 5): потомков нет (индексы 5 и 6 вне массива)

**Проверка свойства кучи:**
- $1 \le 3$ ✓, $1 \le 5$ ✓
- $3 \le 10$ ✓, $3 \le 4$ ✓

**Минимум** всегда находится в корне (индекс 0): GetMin() за $O(1)$.

```cpp
// Вспомогательные функции индексов
int parent(int i) { return (i - 1) / 2; }
int left(int i)   { return 2 * i + 1;   }
int right(int i)  { return 2 * i + 2;   }

// Проверка свойства min-heap
bool is_heap(const vector<int>& h) {
    int n = h.size();
    for (int i = 1; i < n; ++i) {
        if (h[parent(i)] > h[i]) return false;
    }
    return true;
}
```

## 3. SiftDown — просеивание вниз

**Назначение.** SiftDown восстанавливает свойство кучи для узла $i$, при условии что оба его поддерева уже являются кучами. Используется в ExtractMin и BuildHeap.

**Алгоритм:**
1. Найти минимального из трёх: узел $i$, левый потомок, правый потомок.
2. Если минимальный — не сам узел $i$, поменять местами узел $i$ с минимальным потомком.
3. Рекурсивно применить SiftDown к потомку, куда переместился узел.
4. Остановиться, когда узел $\le$ обоих потомков или достигнут лист.

**Сложность:** $O(\log n)$ — высота дерева.

```cpp
void sift_down(vector<int>& h, int i, int n) {
    // n — текущий размер кучи (h может быть больше)
    while (true) {
        int smallest = i;
        int l = 2 * i + 1;
        int r = 2 * i + 2;

        if (l < n && h[l] < h[smallest]) smallest = l;
        if (r < n && h[r] < h[smallest]) smallest = r;

        if (smallest == i) break;   // свойство кучи выполнено

        swap(h[i], h[smallest]);
        i = smallest;               // продолжаем просеивание вниз
    }
}
```

**Разбор примера.** ExtractMin из кучи `[1, 3, 5, 10, 4]`:

Шаг 1. Корень (минимум) сохраняем: min = 1. Перемещаем последний элемент в корень:
`[4, 3, 5, 10]` (размер n = 4)

Шаг 2. SiftDown(0, 4):
- $i=0$, значение 4; left = 1 (значение 3), right = 2 (значение 5)
- smallest = 1 (значение 3 < 4)
- swap(h[0], h[1]): `[3, 4, 5, 10]`, $i = 1$

Шаг 3. SiftDown продолжается с $i=1$, значение 4:
- left = 3 (значение 10), right = 4 (вне массива, так как n=4)
- $10 > 4$, smallest = 1: условие $\text{smallest} == i$ → выход

Результат: `[3, 4, 5, 10]` — корректная куча, min = 3.

## 4. SiftUp — просеивание вверх

**Назначение.** SiftUp восстанавливает свойство кучи для узла $i$, когда его значение могло стать **меньше** родителя. Используется в Insert и DecreaseKey.

**Алгоритм:**
1. Пока узел $i$ не корень и $h[i] < h[\text{parent}(i)]$:
   - Поменять $h[i]$ и $h[\text{parent}(i)]$.
   - Перейти к родителю.

**Сложность:** $O(\log n)$.

```cpp
void sift_up(vector<int>& h, int i) {
    while (i > 0) {
        int p = (i - 1) / 2;   // индекс родителя
        if (h[p] <= h[i]) break; // свойство кучи выполнено
        swap(h[i], h[p]);
        i = p;                   // поднимаемся к родителю
    }
}
```

**Разбор примера.** Insert(2) в кучу `[1, 3, 5, 10, 4]`:

Шаг 1. Добавляем в конец: `[1, 3, 5, 10, 4, 2]`, $i = 5$.

Шаг 2. SiftUp(5):
- parent(5) = 2, значение 5; h[5] = 2 < 5 → swap: `[1, 3, 2, 10, 4, 5]`, $i = 2$

Шаг 3. parent(2) = 0, значение 1; h[2] = 2 ≥ 1 → выход.

Результат: `[1, 3, 2, 10, 4, 5]` — корректная куча.

## 5. Insert и ExtractMin

### Insert

**Идея:** добавить в конец массива, затем восстановить свойство кучи снизу вверх.

$$
\text{Insert}(x): \quad h.\text{push\_back}(x) \to \text{SiftUp}(\text{size}-1)
$$

```cpp
void heap_insert(vector<int>& h, int x) {
    h.push_back(x);
    sift_up(h, (int)h.size() - 1);
}
```

### ExtractMin

**Идея:** минимум в корне. Заменяем корень последним элементом (чтобы не создавать «дыры»), уменьшаем размер, восстанавливаем кучу сверху вниз.

$$
\text{ExtractMin}: \quad \text{save } h[0] \to h[0] = h[\text{last}] \to \text{pop\_back} \to \text{SiftDown}(0)
$$

```cpp
int heap_extract_min(vector<int>& h) {
    if (h.empty()) throw runtime_error("Heap is empty");

    int min_val = h[0];          // сохраняем минимум
    h[0] = h.back();             // корень ← последний элемент
    h.pop_back();                // удаляем последний

    if (!h.empty()) {
        sift_down(h, 0, (int)h.size());
    }
    return min_val;
}
```

**Полный пример работы с кучей:**

```cpp
#include <iostream>
#include <vector>
#include <stdexcept>
using namespace std;

// ... (sift_up, sift_down, heap_insert, heap_extract_min как выше)

int main() {
    vector<int> h;
    heap_insert(h, 4);
    heap_insert(h, 10);
    heap_insert(h, 3);
    heap_insert(h, 5);
    heap_insert(h, 1);
    // h = [1, 4, 3, 10, 5]  (или другой корректный вариант)

    cout << heap_extract_min(h) << "\n"; // 1
    cout << heap_extract_min(h) << "\n"; // 3
    cout << heap_extract_min(h) << "\n"; // 4
    return 0;
}
```

**Сложность обеих операций:** $O(\log n)$ — не более $\lfloor \log_2 n \rfloor$ свопов (высота дерева).

## 6. BuildHeap за $O(n)$

**Задача.** Дан произвольный массив из $n$ элементов. Построить из него корректную кучу **за $O(n)$**, а не за $O(n \log n)$.

**Ключевое наблюдение.** Листья (индексы от $\lfloor n/2 \rfloor$ до $n-1$) уже являются корректными кучами размера 1. Достаточно применить SiftDown ко всем внутренним узлам, начиная с последнего ($\lfloor n/2 \rfloor - 1$) и до корня (0).

```cpp
void build_heap(vector<int>& h) {
    int n = (int)h.size();
    // Начинаем с последнего внутреннего узла
    for (int i = n / 2 - 1; i >= 0; --i) {
        sift_down(h, i, n);
    }
}
```

**Пример.** BuildHeap из `[4, 10, 3, 5, 1]`:

- $n = 5$, последний внутренний узел: $5/2 - 1 = 1$

Шаг $i=1$: узел 1 (значение 10), дети: 3 (5), 4 (1)
- min(10, 5, 1) = 1, swap(h[1], h[4]): `[4, 1, 3, 5, 10]`
- SiftDown продолжается с $i=4$: лист, стоп.

Шаг $i=0$: узел 0 (значение 4), дети: 1 (1), 2 (3)
- min(4, 1, 3) = 1, swap(h[0], h[1]): `[1, 4, 3, 5, 10]`
- SiftDown продолжается с $i=1$: дети 3 (5), 4 (10); $4 \le 5, 4 \le 10$ → стоп.

Результат: `[1, 4, 3, 5, 10]` — корректная min-heap.

**Почему $O(n)$, а не $O(n \log n)$?**

На уровне $k$ от листьев (высота узла равна $k$) находится не более $\lceil n / 2^{k+1} \rceil$ узлов. SiftDown для каждого такого узла занимает $O(k)$. Суммарное время:

$$
T(n) = \sum_{k=0}^{\lfloor \log n \rfloor} \left\lceil \frac{n}{2^{k+1}} \right\rceil \cdot O(k) = O\!\left( n \sum_{k=0}^{\infty} \frac{k}{2^k} \right) = O(n \cdot 2) = O(n)
$$

Используем факт: $\displaystyle\sum_{k=0}^{\infty} \frac{k}{2^k} = 2$.

Вывод: **BuildHeap работает за $O(n)$**, несмотря на то что $n/2$ вызовов SiftDown. Большинство SiftDown вызывается на узлах вблизи листьев с малой высотой.

## 7. Heap-Sort

**Идея.** Heap-Sort использует максимальную кучу (max-heap: каждый родитель $\ge$ потомков) для сортировки массива по возрастанию in-place.

**Почему max-heap?** Максимальный элемент стоит в корне. Извлекаем его в конец массива, уменьшаем размер кучи — и так n раз. Получаем отсортированный массив.

**Фаза 1: BuildHeap** ($O(n)$)
Строим max-heap из всего массива.

**Фаза 2: Извлечение** ($O(n \log n)$)
Повторяем $n-1$ раз:
- Поменять корень (максимум) с последним элементом кучи.
- Уменьшить размер кучи на 1.
- SiftDown для нового корня.

```cpp
// SiftDown для MAX-кучи
void sift_down_max(vector<int>& h, int i, int n) {
    while (true) {
        int largest = i;
        int l = 2 * i + 1;
        int r = 2 * i + 2;

        if (l < n && h[l] > h[largest]) largest = l;
        if (r < n && h[r] > h[largest]) largest = r;

        if (largest == i) break;

        swap(h[i], h[largest]);
        i = largest;
    }
}

void heap_sort(vector<int>& h) {
    int n = (int)h.size();

    // Фаза 1: построить max-heap
    for (int i = n / 2 - 1; i >= 0; --i) {
        sift_down_max(h, i, n);
    }

    // Фаза 2: извлекать максимум в конец
    for (int size = n - 1; size > 0; --size) {
        swap(h[0], h[size]);        // максимум → в конец
        sift_down_max(h, 0, size);  // восстановить кучу для оставшихся
    }
}
```

**Трассировка Heap-Sort на `[4, 2, 7, 1, 3]`:**

BuildHeap (max-heap):
- $i=1$: узел 2 (значение 2), дети: 1 (1), 3 (3) → swap(h[1], h[4]??) Давайте аккуратнее.
  - $l=3$ (h[3]=1), $r=4$ (h[4]=3); largest = 4 (h[4]=3 > h[1]=2) → swap: `[4, 3, 7, 1, 2]`
- $i=0$: узел 0 (h[0]=4), $l=1$ (h[1]=3), $r=2$ (h[2]=7); largest = 2 → swap: `[7, 3, 4, 1, 2]`

Max-heap: `[7, 3, 4, 1, 2]`

Фаза 2:
- size=4: swap(h[0],h[4]) → `[2, 3, 4, 1, | 7]`; SiftDown(0,4): 2→4 → `[4, 3, 2, 1, | 7]`
- size=3: swap(h[0],h[3]) → `[1, 3, 2, | 4, 7]`; SiftDown(0,3): 1→3 → `[3, 1, 2, | 4, 7]`
- size=2: swap(h[0],h[2]) → `[2, 1, | 3, 4, 7]`; SiftDown(0,2): 2>1 → стоп
- size=1: swap(h[0],h[1]) → `[1, | 2, 3, 4, 7]`; SiftDown(0,1): размер 1, стоп

Результат: `[1, 2, 3, 4, 7]` ✓

**Характеристики Heap-Sort:**

| Свойство | Значение |
|---|---|
| Время (худшее) | $O(n \log n)$ |
| Время (среднее) | $O(n \log n)$ |
| Время (лучшее) | $O(n \log n)$ |
| Память | $O(1)$ (in-place) |
| Устойчивость | Нет |
| Адаптивность | Нет |

Heap-Sort имеет лучшие гарантии в худшем случае, чем QuickSort ($O(n^2)$), но на практике медленнее из-за плохой кэш-локальности (обращения к далёким элементам массива).

## 8. `std::priority_queue` в C++

В C++ стандартная библиотека предоставляет `priority_queue<T>` в заголовке `<queue>`.

**По умолчанию — max-heap** (наибольший элемент сверху).

```cpp
#include <queue>
#include <vector>
#include <functional>
using namespace std;

// MAX-куча (по умолчанию)
priority_queue<int> max_pq;
max_pq.push(3);
max_pq.push(1);
max_pq.push(4);
cout << max_pq.top() << "\n";  // 4
max_pq.pop();
cout << max_pq.top() << "\n";  // 3

// MIN-куча: используем greater<int> как компаратор
priority_queue<int, vector<int>, greater<int>> min_pq;
min_pq.push(3);
min_pq.push(1);
min_pq.push(4);
cout << min_pq.top() << "\n";  // 1
min_pq.pop();
cout << min_pq.top() << "\n";  // 3
```

**Сложность операций `std::priority_queue`:**

| Операция | Сложность |
|---|---|
| `push(x)` | $O(\log n)$ |
| `top()` | $O(1)$ |
| `pop()` | $O(\log n)$ |
| `size()`, `empty()` | $O(1)$ |

**Для пар и структур** (по умолчанию сравнение лексикографическое):

```cpp
// Задача: очередь с приоритетом пар (расстояние, вершина) для Дейкстры
priority_queue<pair<int,int>,
               vector<pair<int,int>>,
               greater<pair<int,int>>> dijkstra_pq;

dijkstra_pq.push({10, 1}); // (dist=10, vertex=1)
dijkstra_pq.push({3,  2}); // (dist=3,  vertex=2)

auto [dist, v] = dijkstra_pq.top(); // dist=3, v=2
```

**Важно:** `std::priority_queue` не поддерживает DecreaseKey напрямую. В Дейкстре это обходят повторным добавлением с проверкой посещённости.

## 9. Типичные ошибки

**Ошибка 1: Off-by-one в формулах индексов.**
При нумерации с 0: `left(i) = 2*i+1`, `right(i) = 2*i+2`, `parent(i) = (i-1)/2`.
При нумерации с 1: `left(i) = 2*i`, `right(i) = 2*i+1`, `parent(i) = i/2`.
Смешение двух систем приводит к трудноуловимым ошибкам: SiftDown обращается не к тем элементам.

**Ошибка 2: Использование min-heap для Heap-Sort по возрастанию.**
С min-heap корень всегда минимум. Если класть его в конец, получится массив, отсортированный по убыванию — не по возрастанию. Для сортировки по возрастанию нужна max-heap (или перевернуть результат).

**Ошибка 3: BuildHeap начиная с 0 вместо n/2-1.**
```cpp
// НЕВЕРНО: O(n log n), не O(n)
for (int i = 0; i < n; ++i) sift_down(h, i, n); // SiftDown на листьях бессмысленен

// ВЕРНО: O(n)
for (int i = n/2 - 1; i >= 0; --i) sift_down(h, i, n);
```
Первый вариант корректен в итоге, но теряет преимущество $O(n)$.

**Ошибка 4: Неверная граница в SiftDown после ExtractMin.**
После перемещения последнего элемента в корень размер кучи уменьшается на 1. Если передать `n` вместо `n-1`, SiftDown «увидит» удалённый элемент и может нарушить порядок.
```cpp
h[0] = h.back();
h.pop_back();
// Теперь h.size() уже уменьшился — передаём актуальный размер:
sift_down(h, 0, (int)h.size()); // ВЕРНО
```

**Ошибка 5: Забыть проверить пустоту перед ExtractMin.**
Вызов `h[0]` или `h.back()` на пустом векторе — неопределённое поведение. Всегда проверяйте `h.empty()`.

**Ошибка 6: Нарушение инварианта при DecreaseKey вручную.**
Если уменьшили элемент `h[i]` напрямую (без структуры данных), нужно вызвать `sift_up(h, i)`, а не `sift_down` — значение стало меньше, значит оно может нарушить свойство кучи только вверх.

- **Знать наизусть** формулы индексов (parent, left, right) и уметь записать их для нумерации как с 0, так и с 1.
- **Реализовать с нуля** SiftDown, SiftUp, Insert, ExtractMin, BuildHeap, HeapSort на C++ без подсказок за 15–20 минут.
- **Доказать** сложность BuildHeap за $O(n)$ — стандартный вопрос на теорию. Уметь записать сумму $\sum k/2^k$ и объяснить, почему она сходится.
- **Объяснить разницу** между min-heap и max-heap и зачем в Heap-Sort нужна именно max-heap.
- **Объяснить** почему Heap-Sort не стабилен и как это проявляется на примере.

- **Применять** очередь с приоритетом в задачах: k-th largest element, merge k sorted arrays, Dijkstra.
- **Анализировать** задачи на сложность: когда BuildHeap + n×ExtractMin даёт $O(n \log n)$ и как это соотносится с сортировкой.
- **Понимать** ограничения: priority_queue не поддерживает произвольный поиск и DecreaseKey за $O(\log n)$ без дополнительных структур.

## 11. Итог

Бинарная куча — элегантная структура данных, объединяющая компактность массива с логарифмическим временем ключевых операций. Её сила — в двух ключевых инсайтах: (1) полное бинарное дерево идеально кодируется в массиве без указателей; (2) восстановление свойства кучи требует $O(\log n)$ шагов, следуя одному пути от узла до листа или корня.

Heap-Sort демонстрирует, как из структуры данных рождается алгоритм сортировки с оптимальным гарантированным временем $O(n \log n)$ и без дополнительной памяти. Для ШАД важно не только знать реализацию, но и понимать анализ $O(n)$ для BuildHeap — этот результат неинтуитивен и часто проверяется на экзамене. Освоив кучи, вы получаете фундамент для алгоритмов на графах (Дейкстра, Прим) и задач типа «найти k наибольших элементов».
