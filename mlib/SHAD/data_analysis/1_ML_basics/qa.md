## Задача 1. Классификация или регрессия?

Для каждой из перечисленных задач укажите тип (классификация / регрессия / ранжирование / кластеризация) и обоснуйте выбор $\mathcal{Y}$:

a) Предсказать, откажет ли жёсткий диск в течение следующих 30 дней.  
b) Предсказать, сколько дней осталось до отказа жёсткого диска.  
c) По запросу пользователя упорядочить 1000 товаров в интернет-магазине.  
d) Разбить клиентов банка на группы по поведению без каких-либо меток.

### Решение

a) **Бинарная классификация**: $\mathcal{Y} = \{0, 1\}$. Ответ — дискретный бинарный признак «откажет / не откажет».

b) **Регрессия**: $\mathcal{Y} = [0, +\infty)$. Ответ — непрерывное число (количество дней).

c) **Ранжирование**: $\mathcal{Y}$ — порядок объектов (permutation). Нужно выдать список, отсортированный по убыванию релевантности запросу.

d) **Кластеризация**: нет $y_i$. Задача — найти скрытую структуру в данных без учителя.

### Ответ

a) Классификация, б) Регрессия, в) Ранжирование, г) Кластеризация.

## Задача 2. Формализация задачи

Дана задача: по фотографии лица предсказать возраст человека (целое число лет от 0 до 120).

Запишите формальную постановку: $\mathcal{X}$, $\mathcal{Y}$, предложите $\mathcal{L}$, объясните выбор.

### Решение

- $\mathcal{X}$ = множество пикселей изображения, $x \in \mathbb{R}^{H \times W \times 3}$ (RGB-тензор).
- $\mathcal{Y} = \{0, 1, \ldots, 120\}$ — возраст в годах.

Задача формально дискретная, но:
- **Вариант 1 — регрессия**: $\mathcal{Y} = [0, 120]$, $\mathcal{L} = |\hat y - y|$ (MAE) или $(\hat y - y)^2$ (MSE). Подходит, если важна точность.
- **Вариант 2 — классификация**: $\mathcal{Y} = \{0, \ldots, 120\}$, $\mathcal{L} = \mathbf{1}[\hat y \ne y]$. Теряем порядковую информацию (ошибка «30 вместо 31» и «30 вместо 90» одинаково штрафуется).

MAE предпочтительнее MSE, так как менее чувствителен к выбросам (сильно состарившиеся лица).

### Ответ

Рекомендуется формулировать как **регрессию** с $\mathcal{L} = |\hat y - y|$ (MAE), так как важна порядковая структура ответов.

## Задача 3. Data leakage

Разработчик строит модель для предсказания дефолта клиента. Он загрузил датасет, применил `StandardScaler` ко всем данным, затем разбил на train/test. Тест показал accuracy = 0.94. Позже выяснилось, что реальная точность 0.71. Что пошло не так?

### Решение

**Утечка данных** (data leakage) при масштабировании. `StandardScaler.fit()` вычисляет $\mu$ и $\sigma$ **по всей выборке**, включая тестовую. Значит, тестовые примеры неявно влияли на обучение модели через нормировочные статистики.

Правильный порядок:

```python
from sklearn.preprocessing import StandardScaler
from sklearn.model_selection import train_test_split

# Сначала — разбивка
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

# Затем — fit ТОЛЬКО по train
scaler = StandardScaler()
X_train = scaler.fit_transform(X_train)
X_test  = scaler.transform(X_test)   # только transform, без fit
```

Тест стал "нечестным": модель косвенно "видела" тестовые данные во время обучения.

### Ответ

Data leakage из-за масштабирования до разбивки. Нужно: сначала `train_test_split`, потом `scaler.fit_transform(X_train)` и `scaler.transform(X_test)`.

## Задача 4. Дисбаланс классов

В датасете 10 000 транзакций, из которых 9 700 — нормальные, 300 — мошеннические. Модель предсказывает «нормальная» для всех объектов. Какова accuracy? Почему это плохая модель?

### Решение

$$\text{accuracy} = \frac{9700}{10000} = 0.97$$

Модель-тривиалка достигает 97% точности, ничему не научившись. При этом:
- Все 300 мошеннических транзакций пропущены (recall по классу «мошенничество» = 0).
- Бизнес-потери — прямые убытки.

Accuracy — **плохая метрика при дисбалансе классов**. Нужно использовать:
- **F1-score** по минорному классу.
- **ROC-AUC** (площадь под ROC-кривой).
- **PR-AUC** (площадь под precision-recall кривой) — особенно при сильном дисбалансе.

```python
from sklearn.metrics import f1_score, roc_auc_score

y_pred_trivial = [0] * len(y_test)
print(f1_score(y_test, y_pred_trivial, pos_label=1))       # → 0.0
print(roc_auc_score(y_test, y_pred_trivial))                # → 0.5
```

### Ответ

Accuracy = 0.97, но F1 = 0.0 — модель бесполезна. Нужна метрика, учитывающая дисбаланс (F1, ROC-AUC, PR-AUC).

## Задача 5. Supervised vs unsupervised

В компании есть 1 миллион фотографий собак. Меток породы нет. Придумайте, как можно использовать МО без учителя, а затем — как привлечь учителя для улучшения результата.

### Решение

**Без учителя:**
- Обучить автоэнкодер для получения компактного представления (embedding) фотографий.
- Применить K-Means или HDBSCAN к эмбеддингам — получим кластеры похожих собак (возможно, разные породы).

**Привлечение учителя (semi-supervised):**
- Разметить вручную небольшую выборку (например, 1000 фото) → получить $\{(x_i, \text{порода}_i)\}$.
- Обучить классификатор на размеченных + псевдоразметке кластеров → transfer learning.

Этот подход экономит дорогую разметку, используя структуру нелабелированных данных.

### Ответ

Кластеризация (без учителя) + transfer learning с небольшой разметкой (semi-supervised) позволяет создать классификатор пород без полной разметки всего датасета.

## Задача 6. Выбор функции потерь

Вы строите модель прогноза спроса на продукты в супермаркете. Ошибка в сторону занижения (нехватка товара) обходится в 3× дороже, чем ошибка в сторону завышения (излишки). Как учесть это в функции потерь?

### Решение

Стандартный MSE или MAE симметричны — не разделяют недооценку и переоценку. Нужно **асимметричную** потерю.

**Quantile Loss (пинбол-потеря)** с квантилем $\tau$:

$$\mathcal{L}_\tau(\hat y, y) = \begin{cases} \tau \cdot (y - \hat y) & y \geq \hat y \\ (1 - \tau) \cdot (\hat y - y) & y < \hat y \end{cases}$$

Если недооценка в 3 раза дороже — выбираем $\tau = 0.75$: модель будет смещена вверх, занижение штрафуется сильнее.

```python
# Quantile regression в sklearn через GradientBoostingRegressor
from sklearn.ensemble import GradientBoostingRegressor

model = GradientBoostingRegressor(loss="quantile", alpha=0.75)
model.fit(X_train, y_train)
```

### Ответ

Quantile loss с $\tau = 0.75$: штраф за недооценку = $\tau$, за переоценку = $1 - \tau$. При соотношении 3:1 → $\tau = 3/(3+1) = 0.75$.

## Задача 7. Матрица «объект–признак»

Дан датасет жилья:

| Адрес | Площадь | Этаж | Район | Цена |
|---|---|---|---|---|
| ул. Ленина, 5 | 45 | 3 | Центр | 6.2 |
| пр. Мира, 12 | 80 | 10 | Спальный | 9.8 |

a) Что является объектом, что — признаками, что — целевым признаком?  
b) Какие признаки вещественные, категориальные, порядковые?  
c) Почему нельзя напрямую подать "Адрес" в линейную регрессию?

### Решение

a) **Объект** — строка (одна квартира). **Признаки** — Площадь, Этаж, Район. **Целевой признак** — Цена.

b)
- **Вещественные**: Площадь, Цена.
- **Порядковый** (или вещественный): Этаж — числовой, но интерпретируется как порядок.
- **Категориальный**: Район (нет естественного порядка). Адрес тоже категориальный.

c) Линейная регрессия работает с $\mathbb{R}^d$. "Центр" / "Спальный" — строки, их нельзя умножить на вес. Нужно кодирование: One-Hot Encoding (создаём бинарные признаки `is_center`, `is_sleeping_area`), или Target Encoding, или Ordinal Encoding — в зависимости от задачи.

### Ответ

Объект = строка/квартира, признаки = числовые атрибуты, цель = цена. "Адрес" и "Район" — категориальные, требуют кодирования перед подачей в линейную модель.

## Задача 8. Поиск задачи под данные

У вас есть логи веб-сайта: для каждого пользователя — история просмотренных страниц за последние 30 дней. Придумайте три разные задачи МО (по одной на классификацию, регрессию и кластеризацию) для этих данных.

### Решение

**Классификация**: предсказать, купит ли пользователь что-нибудь в течение следующих 7 дней ($\mathcal{Y} \in \{0, 1\}$). Обучается на исторических данных, где $y_i$ — факт покупки.

**Регрессия**: предсказать сумму покупки пользователя в следующие 7 дней ($\mathcal{Y} \in [0, +\infty)$). Помогает приоретизировать рекламный бюджет.

**Кластеризация**: разбить пользователей на сегменты по паттернам просмотра (без меток). Кластеры могут соответствовать «ищущим скидки», «casual browsers», «high-intent покупателям».

### Ответ

Три задачи МО на одних данных: (1) купит ли? — классификация; (2) сколько потратит? — регрессия; (3) какой тип пользователя? — кластеризация.

## Задача 9. Разбивка на выборки при временных данных

Дан временной ряд: ежедневные продажи за 3 года. Вы хотите предсказывать продажи на 7 дней вперёд. Почему нельзя использовать случайный `train_test_split`? Как правильно разбить данные?

### Решение

Случайный `train_test_split` даёт **data leakage через время**: в train попадают данные из будущего относительно некоторых тестовых точек. Модель «подглядывает» в будущее.

**Правильный подход** — **temporal split**: первые 2.5 года → train, последние 6 месяцев → test.

Для hyperparameter tuning — **walk-forward validation**:

```
Train: [Jan–Dec 2023] | Val: [Jan 2024]
Train: [Jan 2023–Jan 2024] | Val: [Feb 2024]
...
```

В sklearn: `TimeSeriesSplit`:

```python
from sklearn.model_selection import TimeSeriesSplit

tscv = TimeSeriesSplit(n_splits=5)
for train_idx, val_idx in tscv.split(X):
    X_tr, X_val = X[train_idx], X[val_idx]
    # train and evaluate
```

### Ответ

Случайная разбивка нарушает временную причинность. Нужен temporal split (прошлое → train, будущее → test) и `TimeSeriesSplit` для кросс-валидации.

## Задача 10 (Синтез). Проектирование ML-системы

Вы строите систему рекомендаций для стриминговой платформы. Дано: 5 млн пользователей, 100 тыс. фильмов, история просмотров пользователей (85% пользователей посмотрели < 5 фильмов — «холодный старт»).

a) Какую задачу МО решает рекомендательная система?  
b) Как формализовать $\mathcal{X}$, $\mathcal{Y}$, $\mathcal{L}$?  
c) Как бороться с «холодным стартом»?  
d) Как оценить качество на тесте без data leakage?

### Решение

**a)** Ранжирование: для данного пользователя упорядочить 100 тыс. фильмов по убыванию прогнозируемого рейтинга / вероятности просмотра.

**b)** Формализация:
- $\mathcal{X}$ = (user_id, item_id) + пользовательские и айтемные фичи (жанр, рейтинг, история).
- $\mathcal{Y}$ = вероятность клика / просмотра ∈ [0,1] (pointwise), или порядок (listwise).
- $\mathcal{L}$ = binary cross-entropy (pointwise), или NDCG-based loss (listwise).

**c)** Холодный старт:
- Для новых пользователей — использовать демографические признаки, популярные фильмы.
- Для новых фильмов — контентные признаки (жанр, актёры, описание).
- Гибридная система: коллаборативная фильтрация + контентные признаки.

**d)** Разбивка без leakage:
- **По времени**: обучаем на истории до даты $T$, тестируем на поведении после $T$.
- Не включать тестовые взаимодействия в матрицу user-item при обучении.
- Метрики: Recall@K, NDCG@K на отложенных пользовательских сессиях.

### Ответ

Рекомендации — задача ранжирования. Холодный старт решается гибридом (CF + контент). Оценка — temporal split по дате взаимодействий, метрика Recall@K или NDCG@K.
