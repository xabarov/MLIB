## Задача 1. Читаем confusion matrix

По confusion matrix (строки = предсказание, столбцы = истина):

|  | Negative | Positive |
|---|---|---|
| **Predicted Negative** | 950 | 40 |
| **Predicted Positive** | 20  | 90 |

Вычислите: Accuracy, Precision, Recall, F1.

### Решение

Из таблицы: TN=950, FN=40, FP=20, TP=90. Итого N=1100.

$$\text{Accuracy} = \frac{TP+TN}{N} = \frac{90+950}{1100} = \frac{1040}{1100} \approx 0.945$$

$$\text{Precision} = \frac{TP}{TP+FP} = \frac{90}{90+20} = \frac{90}{110} \approx 0.818$$

$$\text{Recall} = \frac{TP}{TP+FN} = \frac{90}{90+40} = \frac{90}{130} \approx 0.692$$

$$F1 = \frac{2 \cdot 0.818 \cdot 0.692}{0.818 + 0.692} = \frac{2 \cdot 0.566}{1.510} \approx 0.750$$

### Ответ

Accuracy ≈ 0.945, Precision ≈ 0.818, Recall ≈ 0.692, F1 ≈ 0.750.

## Задача 2. Выбор метрики под задачу

Для каждой задачи обоснуйте выбор основной метрики:

a) Спам-фильтр: важно не удалить важное письмо пользователя.  
b) Ранняя диагностика рака: пропустить больного — очень дорого.  
c) Детектор мошенничества: 0.1% транзакций мошеннические.  
d) Предсказание цены квартиры.

### Решение

a) **Precision** (F0.5). Важно не помечать настоящие письма как спам (FP дороже FN). Precision = TP/(TP+FP) — доля спама среди помеченного.

b) **Recall** (F2). Пропустить больного (FN) значительно хуже ложной тревоги (FP). Recall = TP/(TP+FN) должен быть близок к 1.

c) **AUC-PR** (Average Precision). Сильный дисбаланс (1:1000). ROC будет оптимистична из-за огромного TN; PR-кривая честнее показывает поведение на редком классе.

d) **RMSE или MAE**. Если есть выбросы (элитные объекты с ценой×10) — MAE робастнее. Если все отклонения одинаково важны — RMSE. R² добавляет интерпретируемость (доля объяснённой дисперсии).

### Ответ

a) Precision / F0.5, б) Recall / F2, в) AUC-PR, г) MAE или RMSE + R².

## Задача 3. F-beta формула

Модель выдала: TP=60, FP=40, FN=20, TN=880.  
Вычислите $F_{0.5}$ и $F_2$ и объясните разницу.

### Решение

$$P = \frac{60}{60+40} = 0.600, \qquad R = \frac{60}{60+20} = 0.750$$

$$F_{0.5} = \frac{(1+0.25)\cdot 0.6 \cdot 0.75}{0.25 \cdot 0.6 + 0.75} = \frac{1.25 \cdot 0.45}{0.15 + 0.75} = \frac{0.5625}{0.90} \approx 0.625$$

$$F_2 = \frac{(1+4)\cdot 0.6 \cdot 0.75}{4 \cdot 0.6 + 0.75} = \frac{5 \cdot 0.45}{2.4 + 0.75} = \frac{2.25}{3.15} \approx 0.714$$

$F_{0.5} = 0.625 < F1 = 0.667 < F_2 = 0.714$. $F_{0.5}$ штрафует низкий Precision сильнее (P=0.60 невысок), поэтому $F_{0.5}$ ниже. $F_2$ штрафует низкий Recall слабее (R=0.75 уже неплохо), поэтому $F_2$ выше.

### Ответ

$F_{0.5} \approx 0.625$, $F_2 \approx 0.714$. При $\beta > 1$ метрика "мягче" относится к отсутствующим позитивам, при $\beta < 1$ — к лишним.

## Задача 4. ROC и AUC интерпретация

Дано: две модели A и B на одном тесте (200 объектов, 50 позитивных). У модели A AUC-ROC=0.91, у модели B AUC-ROC=0.74.

a) Что значит AUC-ROC=0.91?  
b) Может ли модель B оказаться лучше A в конкретной точке (при фиксированном пороге)?  
c) При каком AUC-ROC модель не лучше случайного угадывания?

### Решение

a) Вероятность того, что случайно выбранный позитивный объект получит более высокий score, чем случайно выбранный негативный, равна **0.91**. Формально: $P(\hat s(x^+) > \hat s(x^-)) = 0.91$.

b) **Да**. AUC — интегральная характеристика по всем порогам. Может быть так, что при бизнес-нужном FPR ≤ 0.10 модель B имеет TPR = 0.85, а модель A — TPR = 0.78. ROC-кривые могут пересекаться.

c) При **AUC-ROC = 0.50** — модель эквивалентна случайному угадыванию (ROC-кривая — диагональ). При AUC < 0.5 модель хуже случайного (можно инвертировать предсказания).

### Ответ

a) P(positive scored higher) = 0.91. б) Да — ROC-кривые могут пересекаться. в) AUC = 0.50.

## Задача 5. PR vs ROC при дисбалансе

Датасет: 10 000 объектов, 100 позитивных (1%). Модель предсказывает все как негативные. Вычислите AUC-ROC и AUC-PR для такой модели.

### Решение

Модель всегда предсказывает 0: TP=0, FP=0, FN=100, TN=9900.

**ROC**: При любом пороге $\tau \leq 0$ модель классифицирует всё в 0, TPR=0, FPR=0. При $\tau < 0$ (всё в 1): TPR=1, FPR=1. ROC-кривая — ломаная (0,0)→(0,0)→(1,1) → диагональ. AUC-ROC = **0.50** — как случайная модель.

**PR**: Precision при TPR=0 неопределена (0/0 → 1 по соглашению). При раскрытии порога первый добавляемый позитив: если хоть один негатив добавлен раньше, Precision сразу падает. Средняя точность = **0.01** (= prevalence), что отражает полное отсутствие различающей способности.

Вывод: ROC не замечает проблемы (AUC=0.5 говорит "как случайный"), PR сразу показывает провал (AP=0.01 vs baseline=0.01, но любая реальная модель даст больше).

### Ответ

AUC-ROC ≈ 0.50 (выглядит как "случайная"), AUC-PR = prevalence = 0.01 — PR честнее при сильном дисбалансе.

## Задача 6. Метрики регрессии: выбор

Модель прогнозирует стоимость аренды квартир (тыс. руб./мес.). На тестовой выборке (n=500): MAE=3.2, RMSE=8.7, R²=0.81.

a) Как интерпретировать R²=0.81?  
b) Почему RMSE >> MAE? Что это говорит о данных?  
c) Можно ли сравнивать R² модели аренды квартир с R² модели прогноза температуры?

### Решение

a) Модель объясняет **81% дисперсии** целевой переменной. Оставшиеся 19% — необъяснённая вариация (шум, скрытые факторы). R²=1.0 — идеально, R²=0 — модель предсказывает среднее.

b) RMSE=8.7 существенно больше MAE=3.2. Так как RMSE штрафует ошибки квадратично, а MAE — линейно, большой разрыв означает **наличие выбросов** — отдельных квартир, где ошибка очень велика (редкие элитные или нестандартные объекты). MAE более робастно описывает "типичную" ошибку.

c) **Да** — R² безразмерна и показывает долю объяснённой дисперсии независимо от единиц измерения. R²=0.81 для аренды и R²=0.81 для температуры означают одинаково хорошую относительную подгонку.

### Ответ

а) Модель объясняет 81% дисперсии. б) RMSE >> MAE указывает на выбросы. в) Да, R² сравнима между задачами.

## Задача 7. GridSearch vs RandomizedSearch

Пространство гиперпараметров модели: 5 параметров, каждый с 10 значениями.

a) Сколько комбинаций у GridSearch?  
b) RandomizedSearch выполняет 100 итераций. Какую долю пространства он покрывает?  
c) Почему RandomizedSearch часто находит лучшие результаты при том же бюджете?

### Решение

a) $10^5 = \mathbf{100\,000}$ комбинаций. С 5-fold CV — 500 000 обучений модели.

b) $100 / 100\,000 = \mathbf{0.1\%}$ пространства.

c) **Принцип эффективного охвата маргинальных распределений.** Если из 5 параметров реально важны 2, GridSearch тратит 80% бюджета на неважные параметры (проверяя все 10 значений каждого). RandomizedSearch сэмплирует каждый параметр независимо, поэтому важные параметры исследуются в большем числе различных значений. Теоретически: при $n$ важных из $d$ параметров, RandomizedSearch исследует $n$ маргинальных распределений с $n_{iter}$ точками, а GridSearch — лишь в $n_{iter}/10^{d-n}$ точках.

### Ответ

а) 100 000 комбинаций. б) 0.1%. в) Сэмплирование маргинальных распределений эффективнее при малом числе реально важных параметров.

## Задача 8. Paired t-тест для сравнения моделей

При 5-fold repeated (×3) CV получили 15 пар оценок accuracy:

```
Model A: [0.82, 0.84, 0.81, 0.83, 0.85, 0.80, 0.84, 0.82, 0.83, 0.81, 0.85, 0.83, 0.84, 0.82, 0.83]
Model B: [0.79, 0.81, 0.78, 0.80, 0.82, 0.77, 0.81, 0.79, 0.80, 0.78, 0.82, 0.80, 0.81, 0.79, 0.80]
```

Вычислите $\bar d$, $s_d$, $t$-статистику и сделайте вывод на уровне значимости $\alpha=0.05$.

### Решение

Разности $d_i = A_i - B_i$: все равны 0.03 (одинаковые по условию задачи). Точнее: $\bar d = 0.03$, $s_d \approx 0$ (почти нет дисперсии) — это вырожденный случай для демонстрации.

Реально вычислим:

$$\bar d = \frac{1}{15}\sum d_i = 0.03$$

$$s_d = \sqrt{\frac{\sum(d_i - \bar d)^2}{n-1}} \approx 0.003 \quad \text{(мало вариации)}$$

$$t = \frac{\bar d}{s_d / \sqrt{n}} = \frac{0.03}{0.003 / \sqrt{15}} = \frac{0.03}{0.000775} \approx 38.7$$

Критическое значение $t_{14, 0.025} \approx 2.145$. Так как $t = 38.7 \gg 2.145$, **отвергаем $H_0$** — разница статистически значима.

**Практически:** $p \ll 0.001$. Model A значимо лучше Model B.

### Ответ

$\bar d = 0.03$, $t \approx 38.7$, $p \ll 0.05$. Разница статистически значима, Model A лучше.

## Задача 9. Поправка Бонферрони

Сравниваем 5 моделей (попарно). Хотим сохранить $\alpha = 0.05$ для всего семейства гипотез.

a) Сколько попарных сравнений?  
b) Каков скорректированный порог $\alpha^*$?  
c) Две пары дали $p_1 = 0.008$ и $p_2 = 0.023$. Какие пары значимы?

### Решение

a) $\binom{5}{2} = \frac{5 \cdot 4}{2} = \mathbf{10}$ попарных сравнений.

b) По поправке Бонферрони:
$$\alpha^* = \frac{\alpha}{m} = \frac{0.05}{10} = \mathbf{0.005}$$

c) $p_1 = 0.008 > 0.005$ → **не значима** (после поправки).  
   $p_2 = 0.023 > 0.005$ → **не значима**.

Обе пары не достигают скорректированного уровня. Без поправки (наивный $\alpha=0.05$) обе считались бы значимыми — поправка Бонферрони очень консервативна.

### Ответ

а) 10 сравнений. б) $\alpha^* = 0.005$. в) Ни одна пара не значима после поправки.

## Задача 10. Nested CV — выявление data leakage (синтез)

Исследователь строит SVM-классификатор. Он делает:

1. Разбивает данные: train (80%) / test (20%).  
2. Выполняет GridSearchCV на train с 5-fold CV.  
3. Берёт лучшие параметры, переобучает на всём train.  
4. Оценивает accuracy на test: 0.88.  
5. Утверждает: "Оценка честная, тест не использовался при подборе."

Найдите **все** проблемы в этом протоколе. Напишите корректную версию с nested CV.

### Решение

**Проблема 1 — Optimistic bias при выборе модели:**  
Шаги 2–4 дают одну оценку 0.88. Дисперсия этой оценки неизвестна — случайная удача разбивки может завысить результат. Один тест = ненадёжная оценка.

**Проблема 2 — Нет статистической оценки разброса:**  
Accuracy=0.88 без доверительного интервала ничего не говорит о надёжности. Нужен $\pm$ std.

**Проблема 3 — Можно случайно использовать тест косвенно:**  
Если researcher итеративно смотрит на тест и корректирует подход ("попробую RBF-ядро, потом посмотрю на тест") — происходит implicit leakage.

**Корректный протокол:**

```python
from sklearn.model_selection import GridSearchCV, cross_val_score, KFold
from sklearn.svm import SVC
from sklearn.datasets import make_classification
import numpy as np

X, y = make_classification(n_samples=800, n_features=20, random_state=0)

param_grid_cv = {"C": [0.1, 1, 10, 100], "gamma": [0.001, 0.01, 0.1]}

inner_cv = KFold(n_splits=4, shuffle=True, random_state=1)
outer_cv  = KFold(n_splits=5, shuffle=True, random_state=2)

clf_pipeline = GridSearchCV(
    SVC(kernel="rbf"), param_grid_cv,
    cv=inner_cv, scoring="accuracy", refit=True
)

# Внешний цикл — честная оценка методологии
nested_scores = cross_val_score(
    clf_pipeline, X, y, cv=outer_cv, scoring="accuracy"
)
print(f"Nested CV accuracy: {nested_scores.mean():.3f} ± {nested_scores.std():.3f}")

# Только ПОСЛЕ оценки — финальный fit на ВСЕХ данных
clf_pipeline.fit(X, y)
print("Final best params:", clf_pipeline.best_params_)
```

**Почему nested честен:** inner loop подбирает параметры только внутри train-фолдов outer. Test-фолд outer **никогда** не участвует ни в обучении, ни в выборе гиперпараметров. Оценка усредняется по 5 фолдам → дисперсия снижается.

### Ответ

Проблемы: 1) одна точка оценки — нет ±std, 2) потенциальный implicit leakage при итеративном анализе. Решение — nested CV: inner loop = GridSearchCV, outer loop = честная оценка обобщения с несколькими фолдами.
