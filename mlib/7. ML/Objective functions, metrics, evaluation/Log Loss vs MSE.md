# Log Loss vs MSE для логистической регрессии

**Вопрос (EN)**: For logistic regression, why is log loss recommended over MSE (mean squared error)?

## Ответ

### Log loss (cross-entropy) для классификации

\[
L = -\sum_i \left[ y_i \log(p_i) + (1-y_i) \log(1-p_i) \right]
\]

### Почему log loss предпочтительнее MSE

1. **Выпуклость** — log loss выпуклая, MSE для вероятностей — нет. SGD с log loss даёт устойчивую оптимизацию.

2. **Градиенты** — при \(p \to 0\) или \(p \to 1\) градиент log loss остаётся информативным; MSE затухает и обучение замедляется.

3. **Интерпретация** — log loss соответствует максимизации правдоподобия модели.

4. **Совместимость с вероятностями** — логистическая регрессия выдаёт вероятности; log loss естественно штрафует уверенные неверные предсказания.

5. **MSE для классификации** — при нелинейности (sigmoid) MSE невыпукла и может давать плохие градиенты на краях.
