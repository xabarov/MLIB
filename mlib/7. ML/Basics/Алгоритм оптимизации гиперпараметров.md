# Алгоритм оптимизации гиперпараметров

**Вопрос (EN)**: Explain algorithm for tuning hyperparameters. Why is hyperparameter tuning important?

## Ответ

### Почему важна настройка гиперпараметров

Гиперпараметры (learning rate, batch size, глубина, регуляризация) не обучаются градиентным спуском — их нужно выбирать вручную или поиском. Неверный выбор ведёт к переобучению, расходимости или слабому качеству модели. Правильная настройка кардинально влияет на итоговое качество.

### Основные алгоритмы настройки

1. **Grid Search (поиск по сетке)**  
   Полный перебор всех комбинаций из заданных значений. Гарантирует нахождение лучшей конфигурации в сетке, но экспоненциально дорог с числом гиперпараметров.

2. **Random Search (случайный поиск)**  
   Случайная выборка конфигураций из заданных диапазонов. Эффективнее Grid Search при большом числе гиперпараметров: если немногие из них существенно влияют на качество, Random Search быстрее находит хорошие значения.

3. **Bayesian Optimization (байесовская оптимизация)**  
   Строит суррогатную модель (обычно Gaussian Process или TPE) зависимости качества от гиперпараметров. Выбирает следующую точку с балансом exploration/exploitation через функцию приобретения (acquisition function). Эффективна при дорогостоящих оценках (каждый эксперимент — долгое обучение).

4. **Hyperband / ASHA**  
   Ранняя остановка слабых конфигураций: обучение многих вариантов на коротком числе эпох, лучшие продолжают обучение. Сочетает random search с адаптивным распределением бюджета.

5. **Population-Based Training (PBT)**  
   Популяция моделей обучается параллельно; слабые наследуют веса и гиперпараметры от сильных с мутацией. Позволяет адаптировать гиперпараметры по ходу обучения.

6. **Эволюционные алгоритмы**  
   Мутация и скрещивание конфигураций, отбор лучших по метрике. Применяются при поиске архитектур (NAS).

### Практические инструменты

| Инструмент | Подход |
|---|---|
| Optuna | Bayesian (TPE), Hyperband, CMA-ES |
| Ray Tune | Hyperband, Population-Based Training, интеграция с Optuna |
| Hyperopt | TPE (Tree-structured Parzen Estimator) |
| scikit-learn GridSearchCV / RandomizedSearchCV | Grid / Random search |
| Weights & Biases Sweeps | Grid, Random, Bayes |

### Совет

Random search часто выгоднее Grid search при >3 гиперпараметрах. Bayesian optimization — при долгих экспериментах (дорогостоящий objective). Hyperband — при наличии промежуточных метрик.
