# Утечка признаков (Feature Leakage)

**Вопрос (EN)**: What are causes of feature leakage? Why does normalization help prevent it? How do you detect feature leakage?

## Ответ

### Причины feature leakage

1. **Признаки, доступные только после события** — например, «выдан ли кредит» для модели одобрения кредита.
2. **Агрегаты по целевой переменной** — среднее по группе, куда входит целевой объект.
3. **Использование target в препроцессинге** — scaling/normalization по всей выборке, включая test.
4. **Дублирование target** — ID, дата, производные от метки.
5. **Временная утечка** — будущие значения при прогнозе на прошлое.

### Почему normalization помогает

При нормализации по train (fit на train, transform на train и test):
- Тестовые данные обрабатываются так же, как в production.
- Исключается использование глобальной статистики по всей выборке (включая test) при расчёте параметров нормализации.

**Ошибка:** fit на всём датасете — в параметры normalization попадает информация из test.

### Как обнаружить leakage

1. **Анализ признаков** — проверка, доступны ли они в момент предсказания.
2. **Permutation importance** — если удаление «подозрительного» признака сильно роняет качество, возможна утечка.
3. **Корреляция с target** — слишком высокая корреляция может указывать на leakage.
4. **Доменный анализ** — знание предметной области и порядка появления данных.
