# Достаточно ли данных для обучения модели

**Вопрос (EN)**: How do you know you've collected enough samples to train your ML model?

## Ответ

Жёсткого правила нет, но есть эвристики и практики.

### Эвристики

1. **Правило 10×** — не менее 10 примеров на параметр/признак. Для логистической регрессии с 100 фичами — тысячи примеров.

2. **Правило по классам** — для классификации: hundreds–thousands примеров на класс (для сложных задач — больше).

3. **Learning curves** — строить train/val по объёму данных. Если val ещё явно улучшается с ростом данных, имеет смысл собирать больше.

4. **Power analysis** — для A/B-тестов оценивать размер выборки для заданной мощности и эффекта.

### Практические индикаторы

- **Стабилизация метрик** — добавление данных почти не меняет val метрику.
- **Сходимость** — модель перестаёт переобучаться при тех же гиперпараметрах.
- **Variance оценок** — bootstrap/кросс-валидация дают устойчивые оценки.

### Важно

«Достаточно» зависит от сложности задачи, числа признаков, дисбаланса классов и требуемой точности. Редко можно обойтись десятками примеров; для глубоких моделей и сложных задач нужны миллионы.
