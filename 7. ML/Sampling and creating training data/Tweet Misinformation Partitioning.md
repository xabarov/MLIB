# Разбиение данных для модели misinformation

**Вопрос (EN)**: 100K labeled tweets over 24 months. Random shuffle, 80% train, 10% valid, 10% test. What might be the problem with this partitioning?

## Ответ

### Проблемы при случайном разбиении

1. **Temporal leakage**  
   Твиты из разных периодов перемешаны. Модель может опираться на временные паттерны (новый сленг, события), которые не воспроизводятся в будущем. В production будут более новые данные.

2. **Автокорреляция**  
   Твиты близкие по времени часто похожи (одна новость, один хэштег). При random split соседние твиты могут оказаться и в train, и в test → завышенная оценка качества.

3. **Разный характер данных**  
   Контент и стиль misinformation меняются со временем. Модель, обученная на старых примерах, может хуже работать на новых.

### Рекомендации

- **Temporal split**: train на более ранних месяцах (например, первые 18), valid на следующих 3, test на последних 3.
- **Stratified temporal split**: стратификация по классам и по времени.
- **Walk-forward**: периодическое переобучение и тест на следующий период.
