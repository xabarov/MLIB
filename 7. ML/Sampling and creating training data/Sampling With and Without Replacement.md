# Выборка с возвращением и без возвращения

**Вопрос (EN)**: What is the difference between sampling with vs. without replacement? Name an example of when you would use one rather than the other?

## Ответ

### Выборка с возвращением (with replacement)

Элемент после выбора возвращается в генеральную совокупность и может быть выбран снова. Вероятность выбора каждого элемента одинакова на каждом шаге.

**Примеры:**
- **Бутстрэп** — построение множества бутстрэп-выборок для доверительных интервалов, оценки дисперсии.
- **Bagging, Random Forest** — каждый дерево обучается на своей бутстрэп-выборке.
- Оценки при малом объёме данных.

### Выборка без возвращения (without replacement)

Элемент после выбора исключается из совокупности и не может быть выбран повторно.

**Примеры:**
- **Train/test split** — объекты не должны попадать и в train, и в test.
- **Cross-validation** — разные фолды не пересекаются.
- Случайная выборка для аннотирования, когда дубликаты не нужны.

### Важное свойство бутстрэпа

При выборке с возвращением объёмом \(n\) из \(n\) объектов примерно 63.2% уникальных строк попадают в выборку, 36.8% — не попадают ни разу.
