# Проблемы при развёртывании больших ML-моделей

**Вопрос (EN)**: What problems might we run into when deploying large machine learning models?

## Ответ

### Производительность и ресурсы

1. **Время инференса** — большие модели могут быть слишком медленными для online-сервиса при ограниченных latency SLA.
2. **Память** — модели в миллиарды параметров не помещаются на одну GPU/CPU, нужны распределённая загрузка и inference.
3. **Вычислительные затраты** — высокий cost на инференс и хранение.

### Масштабирование

4. **Распределённый inference** — требуется разбиение модели или запросов на несколько устройств.
5. **Пропускная способность** — при большом RPS нужны кэширование, батчинг, оптимизация запросов.

### Технические сложности

6. **Численная стабильность** — риск overflow/underflow при смешанной точности или больших весах.
7. **Версионирование** — управление версиями больших артефактов (checkpoint, конфиг, зависимости).
8. **A/B тестирование** — сравнение тяжёлой и более лёгкой модели при ограничениях по ресурсам.

### Операционные риски

9. **Надёжность** — сбой одного узла может затрагивать весь inference.
10. **Развёртывание и откат** — долгая загрузка модели, сложности с blue-green deployment.
11. **Мониторинг** — отслеживание качества, drift и производительности в production.

### Решения

- Квантизация, дистилляция, pruning.
- Тензорный и pipeline-параллелизм.
- Кэширование, батчинг, асинхронный inference.
- Смешанная точность (FP16/BF16).
- Специализированные сервисы (Triton, ONNX Runtime и т.п.).
